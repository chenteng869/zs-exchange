## 1\. 移动根目录临时文件到 \_trash/



建议用更安全的方式，不直接写多个可能不存在的通配符参数。



```PowerShell
Write-Host "=== Q03-7.0b move root tmp files to _trash ===" -ForegroundColor Cyan

New-Item -ItemType Directory -Force "_trash/q03-7-prepush" | Out-Null

$patterns = @(
  "__tmp_q03_*.ps1",
  "__tmp_q03_*.js",
  "__tmp_q03_*.cmd",
  "__tmp_q03_*.md",
  "__tmp_dump.ps1"
)

foreach ($pattern in $patterns) {
  Get-ChildItem -Path "." -File -Filter $pattern -ErrorAction SilentlyContinue |
    ForEach-Object {
      Move-Item -LiteralPath $_.FullName -Destination "_trash/q03-7-prepush/" -Force
    }
}

Write-Host ""
Write-Host "### remaining root tmp files" -ForegroundColor Yellow
Get-ChildItem -Path "." -File -Filter "__tmp_*" -ErrorAction SilentlyContinue | Select-Object Name

Write-Host ""
Write-Host "### status tmp check" -ForegroundColor Yellow
git --no-pager status --short | Where-Object { $_ -match "__tmp_" }
```



通过条件：



```Plain Text
remaining root tmp files：无输出
status tmp check：无输出，或只剩 _trash/ 下且被忽略不显示
```

## 3\. 扩展 \.gitignore，防止根目录 \_\_tmp\_\* 再次堆积



建议只加根目录规则，不加全局递归规则：



```Plain Text
/__tmp_*
```



原因：



```Plain Text
/__tmp_* 只忽略仓库根目录临时文件
不会误伤子目录内可能有意义的文件
```



执行：



```PowerShell
Write-Host "=== Q03-7.0b update gitignore for root tmp files ===" -ForegroundColor Cyan

$rule = "/__tmp_*"

if (-not (Select-String -Path ".gitignore" -Pattern "^\s*/__tmp_\*" -Quiet)) {
  Add-Content -Path ".gitignore" -Value ""
  Add-Content -Path ".gitignore" -Value "# Agent/session temporary files"
  Add-Content -Path ".gitignore" -Value $rule
}

Write-Host ""
Write-Host "### gitignore tail" -ForegroundColor Yellow
Get-Content ".gitignore" | Select-Object -Last 12

Write-Host ""
Write-Host "### gitignore diff" -ForegroundColor Yellow
git --no-pager diff -- .gitignore
```



如果 diff 只包含：



```Plain Text
# Agent/session temporary files
/__tmp_*
```



则提交：



```PowerShell
git add -- .gitignore
git commit -m "chore(gitignore): ignore root temporary agent files"
```



提交后验证：



```PowerShell
Write-Host "=== verify gitignore commit ===" -ForegroundColor Cyan

git --no-pager log --oneline -n 4
git --no-pager status --short | Where-Object { $_ -match "__tmp_" }
```

# 是否还要求全仓 clean 才 push？



这里我修正之前的绝对说法。



理想状态当然是：



```Plain Text
push 前全仓 clean
```



但现在 Q03\-7\.0a 已经证明：



```Plain Text
565+ dirty 是历史遗留专项
不属于本轮已提交的 24 个 commit
也不影响 fast-forward push
```



所以更现实、更稳的策略是：



```Plain Text
先把本轮新增临时污染物清掉
把需要留档的 Q03 报告提交
加 .gitignore 防复发
然后允许 push 26 个治理完成 commit
```



不建议为了追求全仓 clean，在 push 前强行处理 565\+ 历史 dirty。那会把 Q03\-7 推送动作拖进新的大专项，风险反而更高。
