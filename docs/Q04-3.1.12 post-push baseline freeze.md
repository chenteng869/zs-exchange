## 4. Push transport fix

Git 默认 HTTP/2 协议栈在当前网络环境下会 RST/挂起。

Effective push/fetch template：

```powershell
$env:GIT_TERMINAL_PROMPT = "0"
git -c credential.helper= `
    -c "credential.https://github.com.helper=!'C:\Program Files\GitHub CLI\gh.exe' auth git-credential" `
    -c http.version=HTTP/1.1 `
    push origin main
```

修复链路：

1. `gh auth setup-git` — 将 `gh` 接管为 github.com 的 credential helper
2. `-c http.version=HTTP/1.1` — 强制 HTTP/1.1 绕过 HTTP/2 协议栈 RST
3. `$env:GIT_TERMINAL_PROMPT = "0"` — 防止 manager helper 等待输入

**约束**：当前不写入 `git config --global http.version HTTP/1.1`，仅在命令级使用 `-c` 注入。仓库级 `git config --local http.version HTTP/1.1` 应作为单独决定处理，不与 Q04-3.1 混在一起。
