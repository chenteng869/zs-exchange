# 0\. 本部分覆盖范围



本部分继续定义生态增长、商城、财务、税务、风控、审批、链上与报表接口：



```Plain Text
1. Referral Reward API：推荐人 10% 奖励接口
2. Team Reward API：团队 5/3/2 服务奖励接口
3. Node Reward API：节点 3/3/2/2 运营奖励接口
4. DAppX Mall API：全球商城接口
5. Finance API：财务账本接口
6. Tax API：税务接口
7. Risk API：风控接口
8. Approval API：审批接口
9. Audit API：审计接口
10. Blockchain API：链上资产接口
11. Report API：报表接口
```



核心要求：



```Plain Text
奖励必须来源于真实订单
团队和节点必须绑定服务记录
财务必须可追踪
税务必须可记录
风控必须可冻结
审批必须可执行
审计必须不可篡改
链上链下必须可对账
```
