### 工业级完整版 / 生产级数据库底稿



---



# 0\. 数据库设计定位



本数据库设计不是 MVP 版本，不是临时版本，也不是普通商城版本。



本版本按照以下标准设计：



```Plain Text
交易所级账户系统
金融级财务账本
链上资产级积分系统
多角色奖励结算系统
全球节点运营系统
风控审计系统
税务合规系统
智能合约对账系统
```



数据库必须支持：



```Plain Text
多用户
多角色
多主体
多币种
多地区
多税率
多规则版本
多审批流程
多资产账本
多链上资产
全流程审计
```



---



# 1\. 数据库总体分库设计



工业级系统建议按业务域分库或逻辑 schema 管理。



推荐 PostgreSQL 多 schema 结构：



```Plain Text
identity_schema：用户身份库
permission_schema：权限库
product_schema：商品权益库
order_schema：订单支付库
revenue_schema：收入分配库
asset_schema：资产账本库
points_schema：积分库
tpoints_schema：tFJ369 库
nft_schema：NFT 权益库
power_schema：生态算力库
release_schema：释放库
reward_schema：推荐 / 团队 / 节点奖励库
mall_schema：DAppX 商城库
finance_schema：财务库
tax_schema：税务库
risk_schema：风控库
approval_schema：审批库
audit_schema：审计库
blockchain_schema：链上资产库
report_schema：报表库
```



如果第一版使用单库，也必须按表名前缀区分业务域。

# 2\. 全局数据库规范



## 2\.1 主键规范



所有表使用：



```Plain Text
id：UUID / ULID
```



建议：



```Plain Text
ULID
```



原因：



```Plain Text
全局唯一
按时间有序
适合分布式系统
适合日志和账本查询
```



---



## 2\.2 业务编号规范



所有核心业务表必须有业务编号：



```Plain Text
user_no
order_no
payment_no
refund_no
allocation_no
ledger_no
reward_no
settlement_no
approval_no
risk_event_no
tx_no
```



编号格式示例：



```Plain Text
USR20250101000001
ORD20250101000001
PAY20250101000001
RFN20250101000001
LED20250101000001
```



---



## 2\.3 金额与精度



所有金额、积分、Token、算力字段统一：



```Plain Text
decimal(36, 18)
```



法币展示：



```Plain Text
保留 2 位
```



但数据库不能只存 2 位。



禁止使用：



```Plain Text
float
double
```



---



## 2\.4 通用字段



核心表必须包含：



```Plain Text
id
business_no
status
created_at
updated_at
deleted_at
created_by
updated_by
version
metadata
```



账本类表不允许软删除：



```Plain Text
ledger 表不可 deleted_at
audit 表不可 deleted_at
finance 表不可 deleted_at
tax 表不可 deleted_at
blockchain transaction 表不可 deleted_at
```



---



## 2\.5 枚举字段策略



工业级系统建议：



```Plain Text
状态字段使用 varchar + 应用层枚举
```



不要过度依赖数据库 enum。



原因：



```Plain Text
后续状态可能扩展
跨语言服务更灵活
迁移成本更低
```

# 24\. 数据一致性要求



## 24\.1 订单一致性



必须满足：



```Plain Text
订单应付金额 = 支付成功金额
支付成功金额 = 分账总金额 + 税务处理差额
分账总金额 = 财务流水分账金额
```



---



## 24\.2 资产一致性



必须满足：



```Plain Text
资产账户余额 = 资产流水累计结果
冻结余额 <= 可用余额 + 冻结余额
冲正必须有原始流水
```



---



## 24\.3 奖励一致性



必须满足：



```Plain Text
推荐奖励 = 订单金额 × 规则比例
团队奖励 = 订单金额 × 对应层级比例
节点奖励 = 订单金额 × 节点等级比例
奖励发放金额 = 奖励金额 - 税额 - 扣减金额
```



---



## 24\.4 链上一致性



必须满足：



```Plain Text
链下 Token 发放记录 = 链上 Transfer 记录
链下 NFT 铸造记录 = 链上 Mint 记录
链下领取记录 = 链上 Claimed 记录
金库链下余额 = 链上钱包余额
```
