# 第 17 阶段：DAppX 全球商城



---



## 17\.1 阶段目标



DAppX 全球商城是福建老酒 369 全球生态的消费闭环。



它的作用不是普通电商，而是承接：



```Plain Text
FJ369 Points 消费
cFJ369 消费
tFJ369 消费
FJ369 Token 消费
NFT 权益兑换
AI 工具兑换
虚拟盘积分消费
商城券消费
酒类权益兑换
活动门票兑换
数字资产兑换
```



DAppX 必须让生态权益有地方使用，不能让积分、Token、NFT 只停留在账户里。



---



## 17\.2 商城模式



DAppX 必须支持两种模式：



### 自营模式



平台自己销售商品：



```Plain Text
平台确认商品收入
平台承担商品成本
平台负责履约
平台负责售后
```



### 平台模式



第三方商户入驻：



```Plain Text
商户销售商品
平台收取服务费
商户负责履约
平台负责监管和结算
```

## 18\.3 后台权限模型

```Plain Text

```

```Plain Text

```



ABAC：



```Plain Text
按地区
按金额
按业务类型
按风险等级
按审批级别
按时间
按用户状态
```



例子：



```Plain Text

超过 10,000 USD 必须财务主管 + 超级管理员双审。

风控管理员可以冻结用户；
但解冻必须二级审批。

运营人员可以创建活动；
但不能修改奖励比例。
```

## 19\.6 Release Claim 合约



释放领取建议采用 Merkle Proof。



流程：



```Plain Text
链下计算用户释放额度
生成 Merkle Tree
Merkle Root 上链
用户提交 Proof 领取
合约校验 Proof
合约发放 Token
记录 claimed 状态
链下同步 tx_hash
```



优点：



```Plain Text
Gas 成本低
计算灵活
链下可风控
链上可验证
```
