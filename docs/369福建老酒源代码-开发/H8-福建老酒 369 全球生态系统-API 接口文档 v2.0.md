# 1\. API 总体架构



## 1\.1 API 分层



系统 API 分为 5 类：



```Plain Text
1. Public API
   面向未登录用户，例如商品展示、注册、登录、公告。

2. User API
   面向普通用户，例如下单、支付、积分查询、NFT 查询、奖励查询。

3. Admin API
   面向后台管理员，例如审核、冻结、分账、奖励审批、风控处理。

4. Internal API
   服务之间内部调用，例如订单触发分账、积分发放、奖励生成。

5. Webhook API
   外部系统回调，例如支付回调、KYC 回调、链上事件回调。
```

# 2\. API 通用规范



## 2\.1 请求头



所有需要登录的接口必须带：



```HTTP
Authorization: Bearer 
```



推荐附加：



```HTTP
X-Request-Id: 全局请求 ID
X-Device-Id: 设备 ID
X-Client-Type: web / app / admin
X-Client-Version: 客户端版本
X-Language: zh-CN / en-US
X-Timezone: Asia/Shanghai
```



涉及幂等的写入接口必须带：



```HTTP
Idempotency-Key: 唯一幂等键
```



涉及后台高危操作建议带：



```HTTP
X-Approval-Id: 审批单 ID
```



---



## 2\.2 统一返回格式



成功：



```JSON
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {},
  "request_id": "req_123456789"
}
```



失败：



```JSON
{
  "success": false,
  "code": "ORDER_NOT_FOUND",
  "message": "Order not found",
  "data": null,
  "request_id": "req_123456789"
}
```



---



## 2\.3 分页格式



请求参数：



```Plain Text
page
page_size
```



示例：



```HTTP
GET /api/v1/orders?page=1&page_size=20
```



返回：



```JSON
{
  "success": true,
  "code": "OK",
  "message": "success",
  "data": {
    "items": [],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total": 1000,
      "total_pages": 50
    }
  }
```

---



## 2\.4 金额格式



所有金额字段返回字符串，避免精度丢失：



```JSON
{
  "amount": "369.000000000000000000",
  "currency": "USD"
}
```



禁止前端假设金额是 number。

