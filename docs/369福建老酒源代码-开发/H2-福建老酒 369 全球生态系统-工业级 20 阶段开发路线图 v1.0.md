工业级 20 阶段开发路线图 v1\.0

对，这里必须改。  

原来的 17 个阶段是“功能开发顺序”，但不是工业级交付顺序。



工业级系统不能按“先做账户、再做订单、再做积分”这种普通项目方式推进。  

应该改成：



```Plain Text
平台底座先行
规则引擎先行
账本系统先行
风控审计先行
财务税务先行
再做业务模块
最后接链上
```



因为福建老酒 369 不是普通商城，而是：



```Plain Text
交易所级账户
金融级分账
链上积分
Token / NFT
多角色奖励
生态算力释放
全球税务合规
```



所以阶段要改成 **工业级 20 阶段交付路线图**。

# 第 13 阶段：生态释放计算系统



释放系统必须基于快照，不允许实时乱算。



流程：



```Plain Text
生成释放池
生成用户算力快照
计算全网有效算力
计算用户占比
计算用户释放额度
套用个人上限
套用剩余额度
套用风控系数
生成释放记录
审批
开放领取
```



公式：



```Plain Text
用户释放 =
释放池 × 用户有效算力 ÷ 全网有效算力
```



实际：



```Plain Text
min(计算释放, 个人上限, 剩余额度)
```
