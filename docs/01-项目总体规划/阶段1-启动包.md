### 5.3 应急预案

#### 预案 1: K8s 集群搭建失败

**触发**: T1.1.1 在 7/1 仍未完成
**行动**:
1. 立即切换到阿里云 ACK Pro
2. 7/2 完成基础集群（5 节点）
3. 7/3-7/10 渐进式扩容到 13 节点
4. W3 末验收

**影响**: 多 1 周延后 W3 末节点验收；培训不受影响

#### 预案 2: Fabric 链码无法部署

**触发**: T1.2.9 链码测试失败
**行动**:
1. 切换到 FISCO-BCOS（Java 链码）
2. 后端 #2 已有 Java 基础，1 周内掌握
3. 重新设计链码接口
4. W5 末完成部署

**影响**: W6 监控告警需重写 Fabric 集成；多 3-5 天

#### 预案 3: 中间件部署阻塞

**触发**: 任一中间件 smoke test 失败
**行动**:
1. 立即隔离问题中间件
2. 启动备用方案（如 Redis 不可用 → 切换到 KeyDB）
3. 24 小时内恢复
4. 失败 1 个不影响整体进度

**影响**: 个别中间件延后 1-2 天；W5 末全部就位不变

#### 预案 4: 培训考试未通过

**触发**: 7/3 考试 < 80 分
**行动**:
1. 7/4-7/10 重考周（弹性时间）
2. 架构师 + mentor 1v1 辅导
3. 7/10 补考
4. 通过率 100% 才能进入 W4

**影响**: W4-W5 中间件部署延后 1 周
