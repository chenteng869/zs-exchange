### 3.2 风险等级矩阵

```
        低影响    中影响    高影响
高概率:  中        中        高
中概率:  低        中        高
低概率:  低        中        中
```

### 3.3 风险处理流程

```
1. 风险识别（任何角色）→ 飞书 #zap-risk 频道
   ↓
2. 评估（PM + 架构师）→ 概率 + 影响 + 等级
   ↓
3. 缓解措施制定 → 责任人 + 截止时间
   ↓
4. 监控（每日站会）→ 状态更新
   ↓
5. 升级（如升级）→ CTO / SteerCo
   ↓
6. 关闭（已解决或接受）→ 关闭记录
```

### 3.4 风险状态定义

| 状态 | 含义 |
|------|------|
| **待发生** | 风险已识别，但未进入处理阶段 |
| **监控中** | 已制定缓解措施，持续监控 |
| **已发生** | 风险已触发，进入应急 |
| **已缓解** | 缓解措施生效 |
| **已解决** | 风险已消除 |
| **接受** | 评估后决定不处理 |

### 11.2 模板使用规则

1. **必须使用模板**: 所有交付物按模板编写

- 任何人对模板有建议，提交 PR 到 `工作模板.md`
