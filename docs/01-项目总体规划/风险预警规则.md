## 🎯 一、风险预警总览

### 1.1 风险监控原则

```
┌────────────────────────────────────────────────────────┐
│                   风险预警生命周期                       │
│                                                        │
│   识别 → 评估 → 登记 → 监控 → 触发 → 通知 → 升级 → 关闭  │
│    ↑                                              ↓    │
│    └──────────── 复盘改进 ←────────────────────────┘    │
└────────────────────────────────────────────────────────┘
```

### 1.2 风险等级定义

| 等级 | 标识 | 风险评分 | 响应时效 | 通知范围 |
|------|------|----------|----------|----------|
| 🔴 P0 | 严重 | 500-900 | 立即 (≤15min) | 全员 + 总监 + CEO |
| 🟠 P1 | 高 | 200-499 | 24 小时内 | PMO + 负责人 + 总监 |
| 🟡 P2 | 中 | 50-199 | 本周内 | 负责人 + PM |
| 🟢 P3 | 低 | <50 | 持续监控 | 负责人 |

**风险评分公式**：`风险评分 = 等级权重 × 影响系数 × 概率系数`
- 等级权重：P0=100, P1=50, P2=20, P3=5
- 影响系数：高=3, 中=2, 低=1
- 概率系数：高=3, 中=2, 低=1

## 📋 二、10 项风险详细规则

### R-01：智能合约安全漏洞

| 字段 | 内容 |
|------|------|
| **风险描述** | 智能合约存在漏洞，可能导致用户资金被盗或合约被攻击 |
| **风险类别** | 技术 / 安全 |
| **等级** | 🔴 P0（严重） |
| **影响** | 资金损失、平台信誉崩塌、监管处罚 |
| **概率** | 中（已发生过多起行业事件） |
| **数据源** | Mythril、Slither、MythX、Certik 审计报告 |
| **监控指标** | 高危漏洞数、合约调用异常、链上异常交易 |
| **自动触发条件** | 1. 静态扫描发现 High/Critical 漏洞<br>2. 链上出现异常大额转账<br>3. 合约被暂停/升级异常<br>4. Audit 报告未通过<br>5. 重入攻击特征检测到 |
| **负责人** | 周敏（区块链） |
| **应对策略** | 1. 部署前完成 Certik/SlowMist 审计<br>2. 实施多签和时间锁<br>3. 异常监控 + 紧急暂停<br>4. Bug Bounty 计划 |
| **应急预案** | 1. 立即暂停出金<br>2. 启动应急小组（15min 内）<br>3. 公开披露 + 用户通知<br>4. 链上资产迁移到多签冷钱包 |
| **通知渠道** | 飞书群（ZS-P0-告警）+ 短信 + 邮件 + 电话 |
| **升级路径** | 周敏 → 李娜（架构）→ 项目总监 → CEO（30min 未响应） |
| **关闭条件** | 1. 漏洞修复并通过复测<br>2. 审计机构出具修复证明<br>3. 已上链用户无资产损失<br>4. 48 小时观察期无异常 |
| **关闭审批人** | 周敏 + 李娜 + 安全负责人 |

## 🔔 三、自动触发条件详细配置

### 3.1 Prometheus 告警规则（YAML）

```yaml
# prometheus-alerts.yml
groups:
  - name: zs-ai-risk-alerts
    interval: 30s
    rules:
      
      # R-01 智能合约安全
      - alert: SmartContractCriticalVuln
        expr: contract_vuln_critical_count > 0
        for: 0m
        labels:
          severity: P0
          risk_id: R-01
        annotations:
          summary: "🚨 智能合约 Critical 漏洞"
          description: "检测到 {{ $value }} 个 Critical 漏洞"
      
      - alert: ContractAbnormalTx
        expr: increase(contract_large_transfer_amount[5m]) > 1000000
        for: 2m
        labels:
          severity: P0
          risk_id: R-01
        annotations:
          summary: "🚨 合约异常大额转账"
      
      # R-02 AI 模型
      - alert: AIModelDrawdown
        expr: ai_strategy_daily_pnl < -5
        for: 0m
        labels:
          severity: P1
          risk_id: R-02
      
      - alert: AIModelConfidence
        expr: ai_model_confidence < 0.6
        for: 10m
        labels:
          severity: P1
          risk_id: R-02
      
      # R-05 第三方 API
      - alert: ThirdPartyAPIErrorRate
        expr: sum(rate(api_errors_total[5m])) / sum(rate(api_calls_total[5m])) > 0.05
        for: 3m
        labels:
          severity: P2
          risk_id: R-05
      
      - alert: ThirdPartyAPILatency
        expr: histogram_quantile(0.99, api_request_duration_seconds) > 3
        for: 5m
        labels:
          severity: P2
          risk_id: R-05
      
      # R-06 性能
      - alert: MatchingEngineTPS
        expr: rate(matching_engine_tps[1m]) < 8000
        for: 5m
        labels:
          severity: P2
          risk_id: R-06
      
      - alert: MatchingEngineLatency
        expr: histogram_quantile(0.99, matching_latency_seconds) > 0.04
        for: 3m
        labels:
          severity: P2
          risk_id: R-06
      
      - alert: HighCPUUsage
        expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[2m])) * 100) > 80
        for: 5m
        labels:
          severity: P2
          risk_id: R-06
      
      # R-07 跨链桥
      - alert: BridgeFailureRate
        expr: sum(rate(bridge_tx_failures[5m])) / sum(rate(bridge_tx_total[5m])) > 0.03
        for: 5m
        labels:
          severity: P2
```

## ✅ 六、风险关闭流程

### 6.1 关闭步骤

```
1. 确认根因已消除
     ↓
2. 验证修复效果
     ↓
3. 观察期（24-48h 无异常）
     ↓
4. 提交关闭申请（Jira Risk 状态改为 "Resolved"）
     ↓
5. 复盘报告（Post-Mortem）
     ↓
6. PMO + 负责人审批
     ↓
7. 状态改为 "Closed" + 归档
     ↓
8. 更新风险知识库
```
