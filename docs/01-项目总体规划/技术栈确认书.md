## 一、4 层架构总览

```
┌────────────────────────────────────────────────────────────────────┐
│                       第 1 层: 接入层 (Access)                     │
│  移动 App (iOS/Android) │ 小程序 (微信/抖音) │ Web (Admin/Portal)  │
│  飞书 / 钉钉 / 邮件 / 短信 │ REST / GraphQL / WebSocket            │
├────────────────────────────────────────────────────────────────────┤
│                       第 2 层: 业务层 (Business)                    │
│  API 网关 (Spring Cloud Gateway) │ 统一鉴权 (JWT) │ 限流 (Sentinel)│
│  智能体服务 (OpenClaw) │ 工作流服务 (n8n) │ BPM 服务 (Flowable)    │
│  存证服务 (NestJS) │ DID 服务 │ 获客服务 │ 管理后台                │
├────────────────────────────────────────────────────────────────────┤
│                       第 3 层: 数据层 (Data)                        │
│  PostgreSQL │ Redis │ MongoDB │ Kafka │ MinIO │ Milvus            │
│  ClickHouse │ InfluxDB │ Hyperledger Fabric                       │
├────────────────────────────────────────────────────────────────────┤
│                    第 4 层: 基础设施层 (Infra)                      │
│  K8s 1.29 │ Docker 24 │ Helm │ ArgoCD │ Prometheus │ Grafana      │
│  Loki │ ELK │ Jaeger │ Vault │ Cert-Manager                        │
└────────────────────────────────────────────────────────────────────┘
```

### 1.1 架构原则

1. **云原生优先**: 所有服务容器化，K8s 编排
2. **微服务 + 领域驱动**: 业务能力下沉到独立服务
3. **异步优先**: 跨服务调用优先使用 Kafka 异步消息
4. **可观测性**: 全链路 TraceID + 指标 + 日志
5. **安全内建**: 零信任、密钥管理、漏洞扫描

## 七、关键技术决策（ADR 概要）

### ADR-001: K8s 自建 vs ACK

**决策**: **自建 K8s 集群**（生产环境）
**理由**:
- 避免云厂商锁定
- 完整控制（自定义 Operator / 资源）
- 团队具备 K8s 运维能力
**备选**: 阿里云 ACK（开发 / 测试环境）
**影响**: 需要 3 master + 10 worker = 13 节点；增加运维成本 0.5 FTE
**签字**: CTO + 架构师 + 运维

### ADR-002: AI 主力模型

**决策**: **GPT-5（海外）+ 通义千问（国内）** 双主力
**理由**:
- 海外业务用 GPT-5 综合能力最强
- 国内业务用通义千问合规 + 性价比
- 降级路由用 Claude / DeepSeek
**备选**: 全部用 GPT-5（成本高、合规风险）
**影响**: AI 网关需要实现智能路由 + 降级
**签字**: CTO + 架构师 + 后端 #1

### ADR-003: 区块链框架

**决策**: **Hyperledger Fabric 2.5**
**理由**:
- Go 链码（团队熟悉）
- 国密算法支持（通过 BCCSP 扩展）
- 联盟治理（适合国链数据联盟）
**备选**: FISCO-BCOS（Java 链码）
**影响**: 链码语言限定 Go；增加 0.5 FTE 区块链学习成本
**签字**: CTO + 架构师 + 后端 #2

### ADR-004: 移动端技术

**决策**: **React Native 0.74**（iOS + Android 统一）
**理由**:
- 单一代码库，节省 50% 移动端成本
- 性能满足需求（非游戏 / 非复杂动画）
- 团队具备 React 经验
**备选**: Flutter（学习曲线）/ 原生（成本高 2x）
**影响**: 复杂动画场景性能有限（必要时用原生模块）
**签字**: CTO + PM + 前端 #2

### ADR-005: 工作流引擎

**决策**: **n8n 1.x (Queue Mode)**
**理由**:
- 可视化（产品 / 运营可维护）
- 200+ 集成节点（覆盖 90% 场景）
- 队列模式支持水平扩展
**备选**: Airflow（Python 强但学习曲线）/ Temporal（强但 Java）
**影响**: 复杂逻辑需编写自定义节点
**签字**: CTO + 架构师 + 后端 #3

### ADR-006: BPM 引擎

**决策**: **Flowable 7.x (商业版)**
**理由**:
- BPMN 2.0 标准（业内主流）
- 商业版支持（避免开源协议风险）
- 国密算法支持
**备选**: Activiti（社区版功能受限）/ Camunda（贵 3x）
**影响**: 商业版授权费 ~5 万 / 年
**签字**: CTO + 架构师 + PM

### ADR-007: 向量数据库

**决策**: **Milvus 2.3**
**理由**:
- 性能领先（ANN-Benchmarks）
- 水平扩展
- 生态成熟（与 LangChain 集成好）
**备选**: Pinecone（SaaS 锁定）/ Qdrant（规模小）
**影响**: 运维复杂度（需 etcd + MinIO + Pulsar）
**签字**: CTO + 架构师 + 后端 #1

### ADR-008: OLAP 引擎

**决策**: **ClickHouse 23.12**
**理由**:
- 写入性能 + 查询性能双优
- 列存压缩（节省 50% 磁盘）
- 国产化生态成熟
**备选**: StarRocks（生态较小）/ Doris（兼容 MySQL）
**影响**: 需要学习 ClickHouse SQL 方言
**签字**: CTO + 架构师 + 后端 #2

### 8.2 安全约束

| 项 | 要求 |
|----|------|
| 密码学 | 国密 SM2/SM3/SM4（区块链） + 国际 AES-256（数据库） |

| 传输 | 全站 HTTPS（TLS 1.3）+ HSTS |
| 存储 | 敏感数据 AES-256 加密（KMS 管理密钥） |
| 审计 | 全量操作日志 + 1 年保留 |
| 漏洞 | 0 高危漏洞（OWASP Top 10 通过） |

| 项 | 适用法规 |
|----|----------|
