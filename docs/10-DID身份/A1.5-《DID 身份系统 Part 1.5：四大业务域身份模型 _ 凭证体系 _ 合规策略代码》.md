# A1\.5\-《DID 身份系统 Part 1\.5：四大业务域身份模型 / 凭证体系 / 合规策略代码》

明白。那 DID 这次不能只做“通用 DID/VC 工具包”，必须从第一天就按 **四大业务域共用身份底座 \+ 分域合规策略 \+ 分域凭证模型 \+ 分域风控授权** 来设计。



这四个场景的 DID 需求完全不同：



```Plain Text
1. 加密数字货币交易所
   重点：KYC、AML、地址归属、交易权限、VIP、合规地区限制、制裁名单、链上地址绑定

2. 全球跨境电商平台
   重点：买家/卖家身份、企业认证、物流地址证明、税务凭证、信用评级、履约证明、跨境合规

3. 博彩
   重点：年龄验证、地区限制、责任博彩、自我排除、反洗钱、投注限额、黑名单、设备/钱包风控

4. 金融公司
   重点：强 KYC、合格投资者、风险承受能力、授信、收入证明、企业 UBO、合规审计、监管可验证
```



所以 DID 身份系统从现在开始必须升级为：



```Plain Text
DID Core
  + Domain Identity Profile
  + Domain Credential Schema
  + Domain Compliance Policy
  + Domain Verification Rule
  + Domain Revocation Rule
  + Domain Privacy Rule
  + Domain Audit Rule
```

# 1\. 为什么必须提前做四域模型



如果先做通用 DID，后面再接业务，会出现严重问题：



## 1\.1 交易所风险



如果 DID 没有提前考虑：



```Plain Text
KYC Level
AML Screening
Sanction Check
Travel Rule
Wallet Ownership
Risk Country
Trading Permission
Withdrawal Permission
```



后面交易系统会缺少身份依据。



---



## 1\.2 跨境电商风险



如果 DID 没有提前考虑：



```Plain Text
卖家企业认证
税务身份
收货地址证明
物流履约
商品合规
跨境清关身份
```



后面电商平台无法可信判断卖家、买家和履约能力。



---



## 1\.3 博彩风险



博彩身份比普通 KYC 更敏感，必须提前支持：



```Plain Text
年龄验证
国家/地区限制
自我排除
责任博彩
投注限额
黑名单
成瘾风险
AML
```



否则一上线就有合规事故。



---



## 1\.4 金融风险



金融身份必须支持更强合规：



```Plain Text
合格投资者
风险测评
收入证明
资产证明
授信评级
企业 UBO
PEP / Sanction
监管审计
```



否则 DID 只能做展示，不能做金融准入。

# 8\. Domain Policy Service



## `core/domain/did-domain-policy.service.ts`



```TypeScript
import {
  DIDBusinessDomain,
  DIDDomainContext,
  DIDDomainDecision,
} from './did-domain.types';
import { DomainComplianceRule } from './did-domain-compliance.types';
import { DIDDomainIdentityProfile } from './did-domain-profile.types';

export interface DomainPolicyRepository {
  listRules(input: {
    domain: DIDBusinessDomain;
    action: string;
    countryCode?: string;
    jurisdiction?: string;
  }): Promise;
}

export class InMemoryDomainPolicyRepository implements DomainPolicyRepository {
  constructor(
    private readonly rules: DomainComplianceRule[] = [],
  ) {}

  async listRules(input: {
    domain: DIDBusinessDomain;
    action: string;
    countryCode?: string;
    jurisdiction?: string;
  }): Promise {
    return this.rules.filter((rule) => {
      if (!rule.enabled) return false;
      if (rule.domain !== input.domain) return false;
      if (rule.action !== input.action) return false;

      if (rule.countryCode && input.countryCode && rule.countryCode !== input.countryCode) {
        return false;
      }

      if (rule.jurisdiction && input.jurisdiction && rule.jurisdiction !== input.jurisdiction) {
        return false;
      }

      return true;
    });
  }
}

export class DIDDomainPolicyService {
  constructor(
    private readonly repo: DomainPolicyRepository,
  ) {}

  async evaluate(input: {
    context: DIDDomainContext;
    profile: DIDDomainIdentityProfile;
    action: string;
  }): Promise {
    const rules = await this.repo.listRules({
      domain: input.context.domain,
      action: input.action,
      countryCode: input.context.countryCode,
      jurisdiction: input.context.jurisdiction,
    });

    if (input.profile.status !== 'active') {
      return {
        allowed: false,
        action: 'deny',
        riskLevel: 'blocked',
        reasons: [`PROFILE_STATUS_${input.profile.status.toUpperCase()}`],
      };
    }

    const missingCredentials: string[] = [];
    const expiredCredentials: string[] = [];
    const revokedCredentials: string[] = [];

    for (const rule of rules) {
      if (rule.blockedCountries?.includes(input.context.countryCode ?? '')) {
        return {
          allowed: false,
          action: 'block',
          riskLevel: 'blocked',
          reasons: ['COUNTRY_BLOCKED'],
          metadata: {
            ruleId: rule.ruleId,
          },
        };
      }

      if (
        rule.allowedCountries &&
        input.context.countryCode &&
        !rule.allowedCountries.includes(input.context.countryCode)
      ) {
        return {
          allowed: false,
          action: 'block',
          riskLevel: 'blocked',
          reasons: ['COUNTRY_NOT_ALLOWED'],
          metadata: {
            ruleId: rule.ruleId,
          },
        };
      }

      for (const requirement of rule.requiredCredentials) {
        if (!requirement.required) continue;

        const credential = input.profile.credentials.find(
          (item) => item.type === requirement.type,
        );

        if (!credential) {
          missingCredentials.push(requirement.type);
          continue;
        }

        if (credential.revoked && !requirement.allowRevoked) {
          revokedCredentials.push(requirement.type);
        }

        if (
          credential.expiresAt &&
          credential.expiresAt  0) {
      return {
        allowed: false,
        action: 'deny',
        riskLevel: 'blocked',
        reasons: ['REVOKED_CREDENTIALS'],
        revokedCredentials,
      };
    }

    if (expiredCredentials.length > 0) {
      return {
        allowed: false,
        action: 'require_reverification',
        riskLevel: 'high',
        reasons: ['EXPIRED_CREDENTIALS'],
        expiredCredentials,
      };
    }

    if (missingCredentials.length > 0) {
      return {
        allowed: false,
        action: 'require_more_credentials',
        riskLevel: 'medium',
        reasons: ['MISSING_REQUIRED_CREDENTIALS'],
        missingCredentials,
      };
    }

    return {
      allowed: true,
      action: 'allow',
      riskLevel: input.profile.riskLevel,
      reasons: [],
    };
  }
}
```

# 14\. Domain Permission Service



业务系统调用这个服务判断用户能不能执行某动作。



## `core/domain/did-domain-permission.service.ts`



```TypeScript
import {
  DIDBusinessDomain,
  DIDDomainContext,
  DIDDomainDecision,
} from './did-domain.types';
import { DIDDomainIdentityProfile } from './did-domain-profile.types';
import { DIDDomainPolicyService } from './did-domain-policy.service';

export interface DomainIdentityProfileRepository {
  getProfile(input: {
    domain: DIDBusinessDomain;
    userId: string;
  }): Promise;
}

export class DIDDomainPermissionService {
  constructor(
    private readonly profileRepo: DomainIdentityProfileRepository,
    private readonly policy: DIDDomainPolicyService,
  ) {}

  async canPerform(input: {
    context: DIDDomainContext;
    userId: string;
    action: string;
  }): Promise {
    const profile = await this.profileRepo.getProfile({
      domain: input.context.domain,
      userId: input.userId,
    });

    if (!profile) {
      return {
        allowed: false,
        action: 'require_more_credentials',
        riskLevel: 'medium',
        reasons: ['DOMAIN_PROFILE_NOT_FOUND'],
      };
    }

    return this.policy.evaluate({
      context: input.context,
      profile,
      action: input.action,
    });
  }
}
```

# 15\. 内存 Profile Repository



后续换数据库。



## `core/domain/in-memory-domain-profile.repository.ts`



```TypeScript
import { DIDBusinessDomain } from './did-domain.types';
import { DIDDomainIdentityProfile } from './did-domain-profile.types';
import { DomainIdentityProfileRepository } from './did-domain-permission.service';

export class InMemoryDomainIdentityProfileRepository
  implements DomainIdentityProfileRepository {
  private readonly profiles = new Map();

  async getProfile(input: {
    domain: DIDBusinessDomain;
    userId: string;
  }): Promise {
    return this.profiles.get(this.key(input.domain, input.userId)) ?? null;
  }

  async upsert(profile: DIDDomainIdentityProfile): Promise {
    this.profiles.set(this.key(profile.domain, profile.userId), profile);
  }

  private key(domain: DIDBusinessDomain, userId: string): string {
    return `${domain}:${userId}`;
  }
}
```
