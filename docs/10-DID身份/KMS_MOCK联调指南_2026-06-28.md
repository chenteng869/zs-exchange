# KMS Mock 联调指南

更新时间：2026-06-28

## 目标

在本地完成 DID 锚定链路与 KMS 签名适配联调，不依赖真实 KMS 服务。

## 新增能力

- 本地 KMS mock 服务：scripts/mock-kms-server.cjs
- 一键联调脚本：scripts/kms-mock-e2e.ts
- DID simulate 模式新增 signer 校验（可验证 keyRef -> 私钥解析是否正确）

## 快速开始

1. 启动 KMS mock

npm.cmd run kms:mock

2. 执行 e2e 联调

npm.cmd run kms:mock:e2e

成功输出示例：

{
  "success": true,
  "did": "did:sol:...",
  "keyRef": "did-sol:...",
  "transactionHash": "0x...",
  "blockNumber": 12345
}

## 可选环境变量

- KMS_MOCK_BASE_URL（默认 http://127.0.0.1:8787）
- KMS_MOCK_TOKEN（默认 dev-kms-token）
- KMS_MOCK_HOST（默认 127.0.0.1）
- KMS_MOCK_PORT（默认 8787）

## 生产切换提示

- 将 DID_SOLANA_SIGNER_PROVIDER 设为 kms
- 将 DID_SOLANA_KMS_ENDPOINT 切换到真实 KMS API
- 将 DID_SOLANA_KMS_AUTH_TOKEN 切换为真实凭据
- 关闭 simulate，开启真实上链验收
