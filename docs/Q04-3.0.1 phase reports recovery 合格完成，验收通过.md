# 新增永久硬约束确认



这两个经验要纳入后续执行规则：



## 1\. 不向 `.git/` 写临时 commit message



你的执行环境明确有 denylist：



```Plain Text
.git/
.vscode
.trae/mcp.json
```



所以后续不要再使用：



```PowerShell
.git\COMMIT_EDITMSG_*.txt
```



统一改为：



```PowerShell
__tmp_commit_msg_*.txt
git commit -F "__tmp_commit_msg_*.txt" --no-verify
Remove-Item "__tmp_commit_msg_*.txt" -Force
```



## 2\. PowerShell 5\.1 中不要写这种表达式



不要写：



```PowerShell
"tmp file removed OK" -ForegroundColor Green
```



`-ForegroundColor` 只能用于：



```PowerShell
Write-Host "tmp file removed OK" -ForegroundColor Green
```



后续脚本要避免这种内联写法。

# 对 commit message 的小问题判断



post\-verify 显示：



```Plain Text
467511f docs(q04): recover q04 phase reports after post-push freeeze
```



这里 `freeeze` 多了一个 `e`。



这是低风险文案瑕疵，不影响提交内容和证据链。**不建议 amend**，原因：



```Plain Text
1. 当前 commit 内容正确
2. staging clean
3. ahead=1，尚未 push
4. amend 会引入不必要二次扰动
5. 这个 commit 是 docs recovery，标题小拼写不影响工程状态
```



除非你强制要求历史洁癖，否则我建议不改。
