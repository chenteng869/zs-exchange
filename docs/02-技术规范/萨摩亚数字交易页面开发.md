# 萨摩亚数字交易页面开发

下面这份可以直接给 Trae CN Solo，用来要求它把「萨摩亚数字交易所」120 个页面中的某一个页面按统一标准开发，并且保证以后每个页面的功能、组件、字段、交互、视觉都对齐一致。



```Plain Text
任务说明：
现在要开发「萨摩亚数字交易所」后台/官网/APP 项目中的一个页面。
这个页面是整个项目约 120 个页面中的其中一页，因此不能孤立开发，必须和全站页面保持统一的业务口径、视觉风格、组件规范、字段规范、交互规范和数据结构。

项目名称：
萨摩亚数字交易所
Samoa Digital Exchange

业务定位：
这是一个面向全球用户、机构客户、数字资产项目方、合规团队、运营人员和交易所管理人员的数字资产交易所平台。
所有页面、字段、Mock 数据、状态、按钮、提示语、图表、表格、交互都必须围绕「数字资产交易所」业务设计。

请不要生成无关行业内容。
禁止出现：
- 电商商品
- 购物车
- 快递物流
- 集运
- SKU库存
- 店铺
- 直播带货
- 无关教育课程
- 普通商城订单
- 与交易所无关的内容

本次目标：
开发当前指定页面，并将其作为 120 个页面统一开发标准的一部分。
页面必须做到：
1. 业务真实；
2. 结构完整；
3. 风格统一；
4. 组件可复用；
5. 数据可演示；
6. 功能可点击；
7. 状态完整；
8. 后续可接真实接口。

当前页面信息：
页面名称：
【填写页面名称，例如：关键词管理 / 交易关键词 / SEO关键词 / 资产关键词 / 公告关键词】

页面路径：
【填写路由，例如：/admin/system/keywords 或 /keywords】

所属模块：
【填写所属模块，例如：系统设置 / 内容管理 / 官网运营 / 交易所配置 / 多语言管理】

页面用途：
【填写用途，例如：用于管理萨摩亚数字交易所官网、公告、帮助中心、资产页面、新闻资讯、交易对页面的关键词配置，提升内容检索、SEO、站内搜索和运营分发效率。】

一、全局统一要求

请先检查项目当前已有：
1. 路由结构；
2. 页面目录；
3. 组件目录；
4. 样式变量；
5. 表格组件；
6. 表单组件；
7. 弹窗/抽屉组件；
8. API/service 写法；
9. Mock 数据写法；
10. 权限写法；
11. 页面标题规范；
12. 状态标签规范。

不要重新发明一套风格。
请沿用项目当前已有组件和设计模式。
如果已有 PageHeader、DataTable、FilterBar、StatusTag、DetailDrawer、EditDialog、Pagination、EmptyState、LoadingState 等组件，请优先复用。
如果没有，则按当前项目风格实现，不要引入新的大型 UI 框架。

二、120 页统一页面标准

每个页面都必须包含以下基础结构：

1. 页面标题区
- 页面名称
- 页面说明
- 面包屑，如项目已有
- 右侧主操作按钮

2. 数据概览区
至少 3-4 个统计卡片，例如：
- 关键词总数
- 已启用关键词
- 待审核关键词
- 今日新增
- 关联页面数
- 搜索命中次数
- 异常关键词数

3. 筛选区
至少包含：
- 关键词搜索
- 类型筛选
- 状态筛选
- 语言筛选
- 时间范围
- 查询按钮
- 重置按钮

4. 数据表格区
必须有：
- 表格字段
- 状态标签
- 分页
- 操作列
- loading 状态
- empty 状态
- error 状态
- normal 状态

5. 详情区
点击「查看」后显示详情弹窗或抽屉。
详情中展示完整字段、关联页面、操作日志。

6. 新增/编辑区
点击「新增」或「编辑」后打开 Dialog 或 Drawer。
表单必须有校验。
保存后要有成功提示。
失败后要有错误提示。

7. 高风险操作
删除、停用、批量删除、批量停用等操作必须二次确认。

8. 操作日志
页面必须展示或预留操作日志，用于审计。

9. Mock 数据
如果真实接口未接入，必须使用结构化 Mock 数据。
不要让页面空白。
不要写“建设中”。

10. API 预留
即使使用 Mock，也要预留 service/api 方法，方便后续替换真实接口。

操作按钮：

- 查看命中页面

详情信息：
- 关键词ID

- 关键词类型

新增/编辑表单：

- SEO标题

- 关键词不能为空
- 关键词长度 2-80 个字符
- 类型不能为空
- 语言不能为空
- SEO标题不超过 120 个字符
- SEO描述不超过 300 个字符

五、组件统一要求

1. PageHeader
用于页面标题、说明、操作按钮。

2. StatCardGrid
用于统计卡片。

3. FilterBar
用于筛选条件。

4. DataTable
用于列表展示。

5. StatusTag
用于统一状态颜色。

6. DetailDrawer
用于查看详情。

7. EditDialog
用于新增/编辑。

9. OperationLog
用于操作日志。

10. EmptyState
用于空数据。

11. ErrorState
用于接口错误。

12. LoadingSkeleton
用于加载状态。

如果项目已有类似组件，请不要重复造轮子。
如果项目没有，可以在当前页面内按现有风格实现，但要保证后续可复用。

六、状态颜色统一

请按以下规则设置状态：

- 已启用：success / 绿色
- 已停用：info / 灰色
- 待审核：warning / 橙色
- 已驳回：danger / 红色
- 已归档：info / 灰色
- 风险：danger / 红色
- 正常：success / 绿色
- 处理中：primary / 蓝色

- deleteKeyword(id)
- updateKeywordStatus(id, status)
- batchUpdateKeywordStatus(ids, status)
- exportKeywords(params)
- syncKeywordI18n(ids)

建议权限：
- keyword:list
- keyword:detail
- keyword:create
- keyword:update
- keyword:delete
- keyword:enable
- keyword:disable
- keyword:export
- keyword:sync

如果当前权限系统未接入，可以先不强制拦截，但要保留字段或注释说明。

十、页面一致性要求

该页面必须和其他 120 页保持一致：

1. 页面标题区一致；
2. 筛选区布局一致；
3. 表格行高一致；
4. 状态标签颜色一致；
5. 按钮风格一致；
6. 弹窗/抽屉风格一致；
7. 空状态一致；
8. 错误状态一致；
9. loading 状态一致；
10. 分页位置一致；
11. 日期格式一致；
12. 数字格式一致；
13. 操作日志一致；
14. 高风险操作确认一致；
15. 文案风格一致。

十一、禁止事项

1. 禁止生成占位页。
2. 禁止出现“建设中”“敬请期待”“稍后实现”。
3. 禁止页面只有标题没有功能。
4. 禁止按钮点击无反应。
5. 禁止表格没有数据结构。
6. 禁止没有 loading / empty / error 状态。
7. 禁止 Mock 数据与萨摩亚交易所无关。
8. 禁止使用电商、物流、SKU、购物车等无关字段。
9. 禁止新建一套完全不同的 UI 风格。
10. 禁止大规模重构无关页面。
11. 禁止修改路由结构，除非当前页面路由缺失。
12. 禁止删除已有功能。
13. 禁止破坏现有菜单和权限。
14. 禁止引入新的大型依赖。
15. 禁止只做静态 UI。

十二、验收标准

完成后必须满足：

1. 页面能通过菜单或路由正常访问。
2. 页面标题、说明、统计卡、筛选区、表格、分页完整。
3. 查询和重置可用。
4. 表格有真实业务 Mock 数据。
5. 查看详情可用。
6. 新增/编辑可用。
7. 启用/停用可用。
8. 删除有二次确认。
9. loading、empty、error、normal 四态完整。
10. 状态标签颜色正确。
11. 页面风格与项目其他页面一致。
12. 无控制台红色错误。
13. 构建不报错。
14. 不影响其他页面。
15. 输出清晰变更说明。

十三、完成后输出

请完成后输出：

1. 修改/新增文件清单
2. 当前页面路由
3. 当前页面所属菜单
4. 使用的组件
5. 使用的 Mock/API 方法
6. 页面字段说明
7. 已实现功能清单
8. 未接真实后端接口的说明
9. 后续接真实接口需要的接口清单
10. 验收步骤

十四、执行方式

请先读取当前项目结构和同类页面，确认项目已有风格后再开发。
请不要凭空造新风格。
请不要只做当前页面外观。
请把当前页面作为 120 个页面统一规范的一部分来实现。
```
