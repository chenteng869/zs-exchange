# ZH / ZS 交易所钱包、DID、全球支付通道、Web3 钱包、浏览器、API 聚合服务商的专业级建设方案





> 全球机构客户、支付通道、Web3 钱包和交易所合规登记，不能只做“能充提币的钱包”。  
> 
> 必须按 **交易所级托管钱包 \+ Travel Rule \+ DID/KYC/KYB \+ 链上风控 \+ 法币支付 \+ 稳定币支付 \+ Web3 钱包 \+ 数据审计** 的标准建设。
> 
> 



这套体系要满足四类监管与商业要求：



1. **资产安全**：私钥、MPC、多签、冷热钱包、托管隔离  

2. **合规风控**：KYC/KYB、AML、Travel Rule、制裁名单、链上监控  

3. **支付能力**：法币入金、出金、银行卡、SWIFT、SEPA、本地支付、稳定币结算  

4. **Web3 能力**：钱包连接、DApp 浏览器、DID、链上身份、跨链、Swap、NFT、DeFi

# 一、迪拜合规登记要先分清监管口径



迪拜不是一个简单的“注册地”，它有不同监管口径。



## 1\. VARA：Dubai Virtual Assets Regulatory Authority



如果你在迪拜面向虚拟资产交易、经纪、托管、转账、支付、资产管理等业务，重点看 **VARA**。



可能涉及的牌照活动包括：



|业务|VARA 相关活动|
|---|---|
|交易所撮合|Exchange Services|
|托管钱包|Custody Services|
|经纪 / OTC|Broker\-Dealer Services|
|转账支付|Transfer and Settlement Services|
|资产管理 / Earn|VA Management and Investment Services|
|借贷|VA Lending and Borrowing Services|
|支付结算|Payment and Remittance 相关安排，需结合当地规则|



## 2\. DIFC / DFSA



如果你在 DIFC 金融自由区做证券、基金、资产管理、投资咨询，可能涉及 **DFSA**。



如果你们未来做 STO/RWA、数字证券、基金化产品，一定要提前区分：



- 普通虚拟资产交易

- 托管

- 支付

- 数字证券

- 投资产品

- 资产管理

    

## 3\. 迪拜合规建设的基本要求

- KYC / KYB

# 二、交易所钱包系统：最高标准架构



你们要做的不是普通 Web3 钱包，而是 **交易所级钱包体系**。



## 1\. 钱包体系应分成 5 层



|层级|功能|标准|
|---|---|---|
|用户钱包层|用户充值地址、资产账户、提现地址|多链、多地址、账户隔离|
|热钱包层|小额高频出入金|限额、自动风控、实时监控|
|温钱包层|中间调拨、运营流动性|多签/MPC、人工审批|
|冷钱包层|主资产存储|离线、多签、硬件隔离|
|托管钱包层|机构客户、企业客户、多签账户|MPC、权限审批、审计报表|



## 2\. 资产隔离要求



必须做到：



- 公司自有资产与客户资产隔离

- 用户交易资产与 Earn 理财资产隔离

- 热钱包与冷钱包隔离

- 托管资产与运营资产隔离

- 企业客户资产独立账本

- 每个币种、每条链、每类账户独立台账

# 八、DID 系统建议架构



## 1\. ZH DID 应包含 6 类凭证



|DID 凭证|用途|
|---|---|
|KYC Credential|个人实名验证|
|KYB Credential|企业实名验证|
|Wallet Binding Credential|钱包地址绑定|
|Risk Credential|风险等级|
|Trading Eligibility Credential|交易权限|
|Accredited / Professional Investor Credential|专业投资者资格，视地区合规使用|



## 2\. 用户身份等级



|等级|名称|权限|
|---|---|---|
|DID\-0|未验证用户|浏览、基础功能|
|DID\-1|基础 KYC 用户|小额交易、小额提现|
|DID\-2|高级 KYC 用户|标准交易、标准提现|
|DID\-3|高净值 / 专业用户|OTC、机构产品|
|DID\-4|企业 KYB 用户|企业账户、批量支付|
|DID\-5|机构用户|托管、API、Prime|



## 3\. DID 与钱包绑定



每个用户必须绑定：



- 登录账户

- KYC 身份

- 钱包地址

- 风险等级

- 交易权限

- 提现权限

- Travel Rule 信息
