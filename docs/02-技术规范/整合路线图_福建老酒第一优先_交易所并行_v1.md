# ZS 交易所 · 整合路线图（福建老酒第一优先 + 交易所并行）

> **第一优先级**: 福建老酒 369/699 全案作战系统  
> **第二优先级**: 交易所核心功能（现货 + 合约 + Web3 + DID + 法币入金）  
> **开发标准**: 地狱式开发，AI 深度辅助  
> **执行周期**: 2026年07月 - 2026年12月（与作战书同步）

---

## 一、战略整合：为什么福建老酒第一优先？

### 1.1 作战书的核心定位

福建老酒 369/699 不是单纯卖酒，而是：

> **ZS 与 AIOPC 创业家体系的第一支全球化旗舰战役产品**
> 
> 用高信任实体产品打通 ZS 的**市场入口、渠道入口、现金流入口和企业客户入口**

### 1.2 两个系统的关系

| 维度 | 福建老酒系统 | 交易所系统 | 关系 |
|------|------------|-----------|------|
| **定位** | 流量入口、现金流、渠道组织 | 核心交易、Web3生态、金融服务 | 前者引流，后者沉淀 |
| **用户** | 消费者、渠道商、企业客户 | 交易者、投资者、Web3用户 | 共享用户体系 |
| **支付** | USDT + 法币入金 | USDT + 法币入金 | 共享支付通道 |
| **钱包** | 余额支付、分润结算 | 充提币、交易 | 共享钱包系统 |
| **KYC** | 实名验证 | 实名验证 | 共享KYC系统 |
| **后台** | 商品/订单/渠道/分润 | 交易/钱包/合约/风控 | 统一管理后台 |

### 1.3 战略价值

```
福建老酒（第一优先）
  ↓
1. 快速验证商业模式（90天跑出样板）
2. 建立渠道组织（AIOPC创业家体系）
3. 产生现金流（100万瓶 = $36.9M - $69.9M GMV）
4. 积累企业客户和高净值用户
5. 验证支付/分润/会员系统
  ↓
交易所（第二优先，并行开发）
  ↓
1. 承接老酒用户转化为交易用户
2. 用老酒渠道推广交易所
3. 共享技术基础设施
4. 形成"实体 + 数字资产"双轮驱动
```

### 2.2 分润系统详解（最核心、最复杂）

**4/3/3 分润制度**：

```
销售收入 100%
  ├─ 40% 产品成本（产品+包装+仓储+物流+损耗+基础运营）
  ├─ 30% AIOPC 创业家大宗提成
  └─ 30% 剩余分润池
        ├─ $50  中萨合资公司
        ├─ $21  海外加密资产公司
        ├─ $12  AIOPC创业家商学院事业部
        ├─ $12  太初国链技术团队
        ├─ $8   各个事业部运营事业部
        ├─ $4   AIOPC创业家事务部
        └─ $3   AIOPC创业家推荐人
```

> 注：以上为 369 美元单瓶分润基准，699 按相同比例放大

**分润执行原则**：
- 成交即记录
- 订单即归属
- 分润即计算
- 结算即留痕
- 异常即审核

## 七、关键决策点

### 7.1 需要确认的技术选择

| 决策点 | 选项 | 建议 |
|--------|------|------|
| 老酒商城框架 | Next.js（复用现有） vs 单独项目 | **复用现有Next.js**，统一技术栈 |
| 分润结算周期 | T+0 / T+7 / 月结 | **T+7**，给退款和审核留缓冲 |
| 老酒支付方式 | USDT优先 / 法币优先 / 都支持 | **都支持**，USDT为主，法币入金为辅 |
| 交易所撮合方式 | 自研撮合 / 接入外部 | **自研简化版**，先用起来再优化 |
| Web3钱包方案 | 自研 / 集成WalletConnect / 混合 | **混合**，核心自研，WalletConnect兼容 |
| DID方案 | Polygon ID / Veramo / 自研 | **Polygon ID**，最成熟 |

### 7.2 安全底线

| 底线 | 说明 |
|------|------|
| 老酒支付先用测试网 | 确认流程无误再上主网 |
| 交易所先用测试网 | 现货/合约都先跑测试网 |
| 热钱包不放大量资金 | MVP阶段控制资金量 |
| 分润计算必须可审计 | 每笔分润都有完整日志 |
| 渠道数据必须可追溯 | 订单归属、推荐关系清晰 |
