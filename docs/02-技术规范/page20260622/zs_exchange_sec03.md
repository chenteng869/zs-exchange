## 3. 字体、排版与间距系统

中萨交易所（ZS Exchange）的排版系统遵循"分层字体栈"设计策略，即为不同内容类型（正文、代码、金融数据）配置独立且深度优化的字体栈，确保在多平台、多语言、多场景下均能保持阅读体验的一致性与专业性。间距系统基于 4px 基准网格构建，提供 14 级 Token 化间距值，覆盖从组件内部微调到页面级宏观布局的全部场景。

### 3.1 字体族配置

字体族（Font Family）的选型直接影响界面的可读性与品牌感知。中萨交易所针对不同内容类型配置了三个独立的字体栈，每个栈均包含完整的回退（fallback）序列，以应对字体文件加载失败或客户端环境不支持首选项字体的情形。

#### 3.1.1 正文字体栈

正文字体栈采用 Inter 作为首选项，该字体由 Rasmus Andersson 设计，针对屏幕显示优化，具有出色的 x-height（小写字母高度）和清晰的字形轮廓，在加密货币交易所界面的高密度信息场景中表现优异。当 Inter 不可用时，回退顺序依次为苹方（PingFang SC，覆盖 macOS 及 iOS 简体中文环境）、微软雅黑（Microsoft YaHei，覆盖 Windows 简体中文环境），最终落入各操作系统的系统默认无衬线字体族。该栈覆盖 H5 移动端、User PC 前台及管理后台的全部正文中文字符渲染需求。

#### 3.1.2 等宽字体栈

等宽字体（Monospace Font）栈用于代码展示、交易哈希（Transaction Hash）、钱包地址、JSON 数据等技术内容场景。首选项 JetBrains Mono 针对代码阅读优化，增大了字母间距并调整了运算符的字形，使 `0` 与 `O`、`1` 与 `l` 等易混淆字符具备可辨识性差异。回退链依次为 Fira Code（支持连字特性 ligature）、DIN、Roboto Mono，最终以 Consolas 及系统默认等宽字体兜底。该栈不用于普通正文，仅在 `code`、`pre` 等语义化标签及数据展示组件中调用。

#### 3.1.3 数字字体栈

数字字体栈专用于价格展示、涨跌幅百分比、资产金额、K 线数据等金融数值的渲染，对数字的等宽对齐和紧凑度有较高要求。首选项 DIN（Deutsches Institut für Normung）为经典工业标准字体，其数字字形具备几何感与辨识度，在金融终端界面中被广泛采用。回退顺序为 Roboto Mono 与 JetBrains Mono，两者均为现代等宽字体，可确保价格列在多行对齐时保持十进制小数点的纵向一致。数字字体栈通过 CSS `font-variant-numeric: tabular-nums` 配合，强制启用等宽数字字形，避免比例数字（proportional figures）导致的列抖动。

**表 3-1 字体族配置总览**

| 用途 | 字体栈 | 回退顺序 | 使用场景 |
|------|--------|---------|---------|
| **正文字体 (Sans)** | `'Inter', 'PingFang SC', 'Microsoft YaHei', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif` | Inter → 苹方 → 微软雅黑 → 系统默认 | 全局正文、标题、按钮、导航、卡片文字 |
| **等宽字体 (Mono)** | `'JetBrains Mono', 'Fira Code', 'DIN', 'Roboto Mono', Consolas, monospace` | JetBrains Mono → Fira Code → DIN → Roboto Mono → Consolas | 代码块、交易哈希、钱包地址、JSON 数据、技术日志 |
| **数字字体 (Number)** | `'DIN', 'Roboto Mono', 'JetBrains Mono', monospace` | DIN → Roboto Mono → JetBrains Mono → 系统等宽 | 价格数字、资产金额、涨跌幅、K 线坐标、KPI 指标 |

上述三个字体栈通过 CSS 变量或 Tailwind CSS 的 `font-sans`、`font-mono` 类名进行调用。在 TypeScript 设计令牌文件中，分别对应 `fontFamily.sans`、`fontFamily.mono` 与 `fontFamily.number` 三个 Token。数字字体栈作为中萨交易所品牌视觉的差异化要素之一，在资产总览卡、行情列表、订单簿等核心交易场景中统一使用，确保用户在高频数字读取时获得稳定、对齐、低认知负荷的排版体验。

### 3.2 字号体系

中萨交易所的字号体系分为三个层级：Web/H5 端专用显示字号（15 级，覆盖从超大标题到管理后台辅助文字的全场景）、Design Tokens 标准字号（8 级，对应 Tailwind CSS 的 `text-xs` 到 `text-4xl` 映射），以及 H5 移动端特殊字号（10 类，针对移动端小屏幕密度做精细化调整）。三层体系各有侧重，开发者在构建组件时应根据目标平台与内容语义选择合适的层级。

#### 3.2.2 Design Tokens 标准字号

Design Tokens 标准字号定义为 8 级，从 `xs`（`12px` / `0.75rem`）到 `4xl`（`36px` / `2.25rem`），对应 Tailwind CSS 的内置字体大小类名。该体系为全平台通用标准，在 Design Tokens 文件（`tokens.ts`）中以 TypeScript 常量形式维护，并在 Tailwind 配置中完成注册。使用 rem 单位确保了字号在根元素字体大小调整时的可伸缩性，满足 Web Content Accessibility Guidelines（WCAG）对可缩放文本的要求。

#### 3.2.3 H5 移动端特殊字号

H5 移动端受限于屏幕物理尺寸（容器最大宽度为 `480px`），需要在有限空间内呈现高密度金融信息。因此，移动端在通用字号体系之上定义了 10 类特殊字号规范，覆盖资产总览、行情列表、卡片组件、导航栏等核心场景。

- **总资产数字**（`24px` / `700` / `#F8FAFC`）：资产总览卡的核心余额展示，采用最大字号与最高字重的组合，确保用户进入资产页时第一时间捕获总资产信息。配合数字字体栈与 `font-variant-numeric: tabular-nums` 实现千分位对齐。
- **价格数字**（`14px` / `600` / `#F8FAFC`）：行情列表中各交易对的最新价格，Semibold 字重使其在列表中形成视觉锚点，便于纵向扫视时快速定位。
- **涨跌幅**（`11px` / `600`）：绿色（`#34D399`）表示上涨，粉色（`#F472B6`）表示下跌，紧凑的 `11px` 尺寸确保在价格旁并列显示时不占据过多横向空间。
- **卡片标题**（`14px` / `600` / `#F8FAFC`）：各功能模块卡片的标题文字，如"实时行情""热点资讯"等。
- **列表项文字**（`13px` / `400/500` / `#F8FAFC`）：行情列表中交易对名称、成交量等次要信息。
- **辅助说明**（`11px` / `400` / `#7B89B8`）：灰色辅助文字，用于解释性文案、提示信息。
- **时间戳**（`10px` / `400` / `#7B89B8`）：订单时间、公告时间等低优先级时间信息，使用最小字号降低视觉干扰。
- **Tab 导航文字**（`11px` / `400/600`）：非激活状态使用 `400` 字重与 `#7B89B8` 色值，激活状态切换为 `600` 字重与 `#FCD535` 金色，通过字重与颜色的双重变化提供清晰的导航状态反馈。
- **徽章/标签**（`10-12px` / `600`）：语义化标签（如"热门""新币""VIP"）根据重要性在 `10px` 与 `12px` 之间选择，字重固定为 `600` 以保证小尺寸下的可读性。
- **Logo 副标题**（`10px` / `400` / `#7B89B8`）：顶栏 Logo 下方的"中萨数科·移动版"等副标题文字，以最小字号呈现品牌附属信息。

### 3.3 字重与行高

字重（Font Weight）与行高（Line Height）是排版系统中调控信息层级与阅读节奏的核心参数。中萨交易所定义了四级字重与四类行高，通过组合使用实现从营销标题到密集数据表格的全场景覆盖。

#### 3.3.1 四级字重

字重系统仅使用四个离散值，避免过多中间层级导致的视觉混乱。`400`（Normal）为浏览器默认正文粗细，`500`（Medium）在按钮和标签中提供轻微的视觉增强，`600`（Semibold）用于次级标题和导航激活态，`700`（Bold）仅保留给需要最强视觉牵引力的元素。

**表 3-3 字重与行高规格映射**

| 类别 | 名称 | 值 | 使用场景 | 典型元素示例 |
|------|------|-----|---------|------------|
| **字重** | Normal | `400` | 正文、描述、辅助信息 | 段落文字、列表项、时间戳、占位符 |
| **字重** | Medium | `500` | 按钮、标签、次级导航 | 标准按钮文字、Tab 非激活态、表单标签 |
| **字重** | Semibold | `600` | 小标题、导航激活态、价格数字 | 卡片标题、行情价格、涨跌幅、Tab 激活态 |
| **字重** | Bold | `700` | 主标题、核心数据、品牌标识 | Hero 标题、资产总额、Logo 文字、CTA 按钮 |
| **行高** | 全局基础 | `1.6` | 页面级默认行高 | `body` 元素默认设置 |
| **行高** | 标题紧凑 | `1.2` | 大标题、区块标题 | Hero 标题、区段标题、页面标题 |
| **行高** | 正文舒适 | `1.5-1.6` | 段落文字、多行正文 | 文章正文、说明文案、帮助文档 |
| **行高** | 数字表格 | `1.0` | 价格对齐、数据表格 | 行情列表、订单簿、K 线数据、KPI 指标 |

四级字重的选择逻辑遵循"越少越清晰"的原则。在金融类应用中，价格数字与资产金额使用 `600` 或 `700` 的高字重，不仅是为了视觉层级，更是为了在复杂背景下快速抓取用户注意力。需要特别指出的是，`700` 字重的使用受到严格控制，仅在单一视口中出现 1-3 处，以防止 Bold 泛滥导致的视觉噪声。

#### 3.3.2 四类行高

行高（Line Height）的设定直接关联阅读的舒适度与信息的扫描效率。中萨交易所根据内容类型将行高划分为四个梯度。全局基础行高 `1.6` 设置在 `body` 元素上，为无特定行高要求的子元素提供默认继承值。标题紧凑行高 `1.2` 用于 Hero 超大标题与区段标题，紧凑的行高使标题呈现出凝聚力强的视觉块，避免因过大行距导致的标题松散感。正文舒适行高 `1.5-1.6` 针对多行段落文字设计，留有足够的纵向呼吸空间，降低长文本阅读的视觉疲劳。数字表格行高 `1.0` 是该系统的特殊设定，专用于行情列表、订单簿、K 线数据等需要对齐小数点的场景——当行高为 `1.0` 时，行内数字的基线完全对齐，配合等宽数字字体栈可实现十进制数字列的像素级纵向对齐，对于交易员快速扫视价格变动至关重要。

### 3.4 间距系统

间距系统（Spacing System）基于 4px 基准网格（Base Grid）构建，所有间距值均为 4 的整数倍。该系统提供 14 级 Token 化间距，从 `0`（`0px`）到 `24`（`96px`），每一级 Token 以数字命名（如 `space-3`、`space-4`），对应 Tailwind CSS 的 `p-3`、`m-4` 等工具类。4px 网格的选取兼顾了设计的一致性与开发的便利性——在 Tailwind 的默认配置中，`space-1` 即为 `4px`，中萨交易所的间距系统在此基础上进行了扩展。

#### 3.4.1 4px 基准网格 Token

14 级间距 Token 覆盖了从组件内部微调到页面级宏观布局的全部场景。Token `1`（`4px`）用于图标与相邻文字之间的紧凑间隙，是最小可用非零间距。Token `3`（`12px`）和 `4`（`16px`）分别用于小卡片内边距和默认卡片内边距，是组件级别使用频率最高的两个值。Token `6`（`24px`）与 `8`（`32px`）用于区块（section）级别的内边距与外边距。Token `16`（`64px`）及以上进入页面级间距范畴，用于首屏（Hero）区域和大型模块之间的呼吸留白。Token `24`（`96px`）为系统定义的最大标准间距，仅在超长页面的模块分隔或特殊营销页面中使用。

#### 3.4.2 H5 移动端常用间距

H5 移动端在 4px 基准网格之上，针对移动端特有的交互元素与安全约束定义了额外的间距规范。顶栏（Header）采用 `12px 16px` 的内边距配置，其中纵向 `12px` 确保顶栏内容在常规屏幕上的视觉居中，横向 `16px` 与页面内容区的左右边距保持一致，形成纵向对齐的视觉节奏。底部 Tab 栏采用 `8px 0` 的垂直内边距，配合 `env(safe-area-inset-bottom)` 适配 iPhone X 及以上机型的底部安全区域，避免导航元素被 home indicator 遮挡。按钮内边距根据按钮类型分为两种模式：全宽型主操作按钮（如"充币""交易"）采用 `10px 0` 的纵向内边距配合宽度 `100%`，横向按钮则采用 `10px 16-28px` 的不对称内边距，左侧较短（`16px`）、右侧较长（`28px`），以在按钮内部形成轻微的视觉重心偏移，引导用户从左向右阅读按钮文字。
