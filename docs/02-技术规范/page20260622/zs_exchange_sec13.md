# 13. 分阶段实施计划

中萨交易所量化交易模块全量页面清单共120页，按功能优先级、技术依赖关系和生态价值划分为三个实施阶段。Phase 1聚焦MVP（Minimum Viable Product，最小可行产品）核心链路跑通，Phase 2完善差异化核心能力并构建商业闭环，Phase 3构建四大联动生态壁垒与高级功能。三阶段页面总量分别为26页、42页和52页，总工期预估14-20周，通过流水线并行可压缩至10-14周。

---

## 13.1 Phase 1 — MVP核心链路（26页，4-6周）

### 13.1.1 目标：用户登录→提交分析→查看结果→基础跟单

Phase 1的核心目标是构建一条端到端的最小可行产品链路，覆盖用户身份认证、量化分析任务提交、Multi-Agent结果查看和基础跟单执行四个关键节点。该阶段的功能裁剪原则以"链路完整性"为首要标准：用户能够登录系统、输入股票代码、触发AI分析任务、查看各智能体生成的信号矩阵，并执行基础跟单操作。技术层面的核心验证点包括：JWT（JSON Web Token）认证体系在管理后台、H5移动端和User PC三端的一致性表现；WebSocket实时数据推送在行情更新和跟单状态同步中的稳定性；以及图表库在金融暗色主题（底色 `#0a1628`）下的渲染性能。Phase 1明确排除策略市场、回测中心、四大联动模块等高复杂度功能，将资源集中于核心量化引擎与前端界面的集成验证。

### 13.1.3 H5移动10页清单

H5移动端在Phase 1交付10个页面，核心交付物为AI策略大脑主模块（5页）与发现页入口框架（1页），辅以首页增强、导航改造和资产页AI嵌入。

AI策略大脑（H01-H05）是H5端的核心高复杂度模块，包含5个页面：入口页（H01）提供顶部导航+Tab切换栏+底部操作区的整体框架；信号推荐列表（H02）展示评分环、5维指标、大师观点摘要和一键跟单按钮，支持市场/方向/评级三维筛选；跟单管理中心（H03）呈现跟单概览卡（进行中/累计收益/胜率）和进行中的跟单列表；大师观点流（H04）以12位大师头像横向滚动+观点时间线的形式展示最新投资观点；我的订阅列表（H05）管理已订阅策略的续费和退订操作。发现页主入口（H06）在Phase 1仅搭建搜索框与分类Tab的框架结构，5个子Tab的内容填充推迟至Phase 2。首页增强（H12）在原有首页基础上新增「AI策略大脑·今日推荐」横幅卡片区域并支持轮播展示。底部导航从5Tab扩展至7Tab（H13），新增「AI策略」和「发现」两个入口。资产页（H17）为每项资产新增AI标签（看涨/观望/评分），顶部嵌入AI资产建议卡片。

### 13.1.4 公共系统5页清单

公共系统在Phase 1交付5个基础模块，为后续阶段提供底层技术支撑。

这5个公共模块的选择遵循基础设施优先原则。JWT认证中间件（S01）和WebSocket连接管理（S06）构成所有后续功能的数据通道与身份认证基础，前者实现三端统一的Token校验与权限注入，后者支撑实时行情、分析信号和跟单状态的推送服务。主题配置系统（S08）和图表组件库（S09）定义了金融终端的视觉规范与图表渲染标准，确保后续各阶段开发的页面在视觉表现上保持一致性。全局Loading骨架屏（S03）统一数据加载期间的用户体验。API限流提示（S02）、全局空状态页（S04）、错误边界（S05）等容错类组件因不影响核心链路跑通，推迟至Phase 3补充。

### 13.1.5 人力配置：2前端+2后端

Phase 1的人力配置为2名前端工程师与2名后端工程师。前端团队负责管理后台React组件开发（11页）、H5移动端适配开发（10页）以及公共系统基础组件封装（5页）；后端团队负责Multi-Agent引擎的RESTful API接口开发、JWT认证服务、WebSocket推送服务、数据库模型设计与数据对接。该配置基于各端页面复杂度评估确定：Phase 1包含高复杂度页面5个（W01、W06、W07、W10、H01-H03）、中复杂度页面8个、低复杂度页面13个，以4人并行开发模式可在4-6周内完成交付。工期弹性区间主要受Multi-Agent后端接口的调试复杂度影响，若LLM Provider集成顺利，可在4周内完成；若遇到Agent推理链路调优需求，则可能延长至6周。

## 13.2 Phase 2 — 核心能力完善（42页，4-6周）

### 13.2.1 目标：策略市场+回测+跟单闭环+移动端核心体验

Phase 2的核心目标是完善量化平台的差异化竞争能力，构建从策略发布、回测验证、订阅付费到自动跟单执行的完整商业闭环。该阶段承接Phase 1已跑通的分析链路，在管理后台补充风险调整信号（W08）、组合决策输出（W09）等关键分析子视图，上线回测中心（W15-W17）供用户在历史数据上验证策略表现，启用策略市场（W22-W23）实现策略的产品化与订阅，并通过跟单中心（W24-W25）完成分析结果到实际交易的自动执行链路。移动端层面，Phase 2填充发现页5个子Tab（H07-H11），新增代币AI分析（H15-H16）、信号/策略/跟单详情页（H22-H25）等18个页面，完成H5核心差异化体验的构建。User PC端同步完成9个增强页面（按工作量折算为8页），将量化AI能力以最小侵入方式嵌入现有交易界面。管理后台在本阶段建立RBAC（Role-Based Access Control，基于角色的访问控制）体系（W55-W56），为后续多角色协作奠定权限管理基础。

## 13.3 Phase 3 — 生态壁垒构建（52页，6-8周）

### 13.3.1 目标：代币评分+企业诊断+分销中心+IPO评估+AI对话

Phase 3是构建生态壁垒的关键阶段，目标是完成四大联动模块——代币智能评分系统、企业AI投研助手、策略分销中心和IPO可行性评估系统——的全部22个后台页面，上线H5高级功能（AI对话助手、企业诊断报告等8页），补充User PC新增独立页面（6页）和剩余公共系统组件（5页），并完成管理后台的收尾工作（4页未在Phase 1-2完成的通用支撑页）。四大联动模块的深度集成使中萨交易所从单一量化工具升级为覆盖代币发行评估、企业投研诊断、策略分销网络和上市可行性评估的全生态金融服务平台。AI对话助手（H33-H35）基于AIOPC超级个体提供自然语言投研交互能力，是差异化竞争的核心功能之一。Phase 3的工作量（52页）显著高于前两阶段，工期相应延长至6-8周，需要适度扩充人力以保障交付节奏。

### 13.3.2 四大联动22页：代币评分6/企业投研6/策略分销5/IPO评估6

四大联动模块是Phase 3后台开发的核心内容，合计22个页面，构成中萨交易所生态壁垒的技术基础。

**代币智能评分系统（6页）**：包含代币项目总览列表（W26）、AI综合评分卡详情（W27）、经济模型审计面板（W28）、链上健康度分析（W29）、上币定价建议（W30）和风险预警列表（W31）。该模块覆盖从代币项目接入到100分制多维度评分（经济模型25分/增长潜力25分/风控安全25分/AIOPC 15分/社区活跃10分）的完整评估链路。经济模型审计面板（W28）通过DCF（Discounted Cash Flow，贴现现金流）估值Agent审查通缩机制、释放节奏和质押率；链上健康度分析（W29）集成Nansen链上数据追踪持仓分布与巨鲸地址异动；上币定价建议（W30）基于估值模型给出建议发行价和初始流通量区间。

**企业AI投研助手（6页）**：包含入驻企业看板（W32）、企业AI诊断报告（W33）、行业对标矩阵（W34）、供应链关系图谱（W35）、AIOPC工具推荐（W36）和数字化成熟度评估（W37）。企业AI诊断报告（W33）是核心高复杂度页面，整合12位投资大师观点、6大分析师数据和AIOPC行动建议，形成完整的诊断报告模板。供应链关系图谱（W35）以力导向图可视化展示企业与陈氏集团旗下公司的供应链关系网络。

**策略分销中心（5页）**：包含策略产品化面板（W38）、分销网络看板（W39）、电商转化漏斗（W40）、流量来源分析（W41）和收益分成报表（W42）。该模块支撑将量化策略打包为标准化订阅产品（定价$29/月或利润分成模式），建立L1-L4四级代理分销网络，并提供策略师/分销商/平台三方分账的结算能力。电商转化漏斗（W40）可视化展示WOPC购物用户到策略付费订阅的完整转化链路。

**IPO可行性评估系统（6页）**：包含IPO候选企业Pipeline（W43）、上市可行性评分详情（W44）、DCF估值报告（W45）、同行可比公司分析（W46）、IPO定价模拟器（W47）和路演材料生成器（W48）。该模块提供多维度上市评分体系（财务健康度30分/公司治理20分/成长性25分/市场时机15分/合规准备度10分），定价模拟器（W47）支持蒙特卡洛模拟上市后12个月股价走势，路演材料生成器（W48）可一键输出PPT风格投资亮点报告供企业使用。

### 13.3.3 H5 P2功能8页：AI对话3/企业诊断2/其他3

H5移动端在Phase 3交付8个P2（Phase 2优先级）高级功能页面。AI对话助手模块占据3页：对话主页（H33）是基于AIOPC超级个体的高复杂度对话界面，包含对话区域、快捷问题建议输入框和结构化分析结果展示；对话内嵌分析卡片（H34）为AI回复中嵌入的评分/亮点/风险/建议结构化组件；对话历史列表（H35）支持历史对话的搜索和收藏。企业诊断报告模块包含2页：企业诊断报告移动端（H36）将后台完整诊断报告适配为移动端长页面滚动体验；报告分享下载页（H37）支持生成分享链接和PDF下载。其他P2功能包含3页：大师关注管理（H38）提供已关注大师的推送频率设置和取消关注；推广佣金页面（H39）展示佣金明细、邀请链接和推广二维码；跟单收益明细（H40）提供每笔跟单收益、平台抽成、净收益和结算状态的明细查询。

### 13.3.4 User PC新增6页/公共系统5页/后台收尾4页

User PC端在Phase 3新增6个独立量化页面，充分利用桌面端宽屏优势：PC端AI策略中心（U10）是H05 AI策略大脑的PC版适配，采用更高信息密度的多列布局；PC端跟单管理（U11）以表格+图表形式展示跟单数据，更适合桌面端的数据分析场景；PC端策略市场浏览（U12）以卡片网格布局提供比H5更多信息密度的策略展示；PC端大师观点墙（U13）支持同时展示多位大师的观点对比；PC端我的报告库（U14）归档用户购买和生成的AI分析报告，支持预览和下载；PC端收益分析（U15）是个人投资收益分析仪表板，覆盖跟单收益、策略收益和资产趋势的多维度图表分析。

公共系统在Phase 3补充剩余的5个模块：API限流提示组件（S02）在429限流时展示友好提示和剩余等待倒计时；全局空状态页（S04）提供无数据时的插图和引导文案；全局错误边界（S05）通过React Error Boundary捕获渲染错误并展示降级UI；国际化语言包（S07）提供中英文双语文案配置；文件上传/导出组件（S10）统一头像/证件上传和Excel/PDF数据导出功能。

管理后台收尾工作包含4个补充页面：智能体推理详情展开页（W19）补全智能体面板功能，支持查看某位大师对某只股票的完整推理过程；模型配置页（W21）补全系统设置模块，支持默认模型选择和Provider切换；全局告警通知（W57）和安全监控面板（W58）为运维团队提供告警处理状态流转和服务器资源监控能力。此外，404页面（W51）、无权限页（W53）、数据导出中心（W59）和全局消息通知中心（W60）在本阶段同步完成，标志着管理后台全部55个页面的开发工作正式收尾。

### 13.4.2 阶段间依赖关系分析

三阶段之间存在明确的串行依赖关系，关键路径（Critical Path）上的模块无法跳阶段开发。Phase 2对Phase 1的依赖主要体现在三个层面：分析控制台（W06）和信号矩阵表（W07）必须在Phase 1完成，才能支撑Phase 2的风险调整信号（W08）和组合决策输出（W09）在此基础上叠加风控层和决策层；WebSocket连接管理（S06）和JWT认证（S01）是Phase 2跟单中心（W24-W25）和策略市场（W22-W23）的实时通信与身份认证基础；全局布局框架（W50）定义了后台240px侧边栏的导航结构，Phase 2新增的所有后台页面均需在此框架内开发。

Phase 3对Phase 2的依赖同样体现在三个层面：RBAC体系（W55-W56）必须在Phase 2完成，因为四大联动模块涉及策略师、分销商、风控官、企业客户等多角色协作，需要角色权限矩阵支撑细粒度访问控制；策略市场（W22-W23）必须在Phase 2上线，策略分销中心（W38-W42）才能在此基础上构建分销网络；策略列表（W22）和跟单配置（W24）是AI对话助手（H33-H35）回答用户策略咨询的数据来源，对话助手需要调用已上架的策略数据和历史跟单绩效数据来生成结构化回复。

### 13.4.3 流水线方式与工期压缩

虽然三阶段在关键路径上存在串行依赖，但可通过流水线（Pipeline）并行方式压缩总工期。在Phase 1开发后期（第4-5周），当分析核心链路的接口契约已稳定定义后，Phase 2中相对独立的模块——回测中心（W15-W17）的前端配置页面、RBAC体系（W55-W56）的权限数据模型——可提前并行启动开发。这些模块不直接依赖Phase 1的分析结果数据，仅需遵循统一的接口规范即可独立推进。同理，在Phase 2中期（第3-4周），四大联动模块的前端组件设计、国际化语言包（S07）的文案提取、系统监控面板（W58）的数据采集逻辑等独立工作可与Phase 2剩余任务并行展开，提前进入Phase 3的准备工作。

通过流水线并行，总工期可从串行的14-20周压缩至10-14周（约2.5-3.5个月），压缩幅度约25%-30%。关键路径上的串行节点包括：分析链路（W06-W07）→ 策略市场（W22-W23）→ 策略分销中心（W38-W42），以及JWT/WebSocket（S01/S06）→ 跟单中心（W24-W25）→ AI对话助手（H33-H35）的数据通道建设，这些核心链路上的模块因存在数据流依赖而无法并行。但外围独立模块——包括审计日志（W56）、系统监控（W58）、404/无权限页（W51/W53）、数据导出（W59）、空状态页（S04）、错误边界（S05）等——可在各阶段穿插提前实施，不占用关键路径时间。建议在实际执行中采用"关键路径保序、非关键路径并行"的调度策略，以最大化开发资源的利用率。
