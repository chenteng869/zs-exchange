## 5. 玻璃拟态卡片规范

Glassmorphism 2.0 是 Aurora Premium v7 主题的核心视觉特征之一。该规范在初代玻璃拟态（Glassmorphism）的基础上进行了金融级场景的深度优化：通过 `backdrop-filter` 的精细化调校、多层阴影的复合叠加以及顶部高光线的色彩编码，构建出一套具有层次感的通透卡片体系。相较于传统毛玻璃效果仅追求"模糊"的单一维度，v7 版本的玻璃卡片同时承载了信息层级划分、功能模块识别和交互状态反馈三重职责。本章将从核心 CSS 配方出发，逐步展开卡片变体体系、Hover 交互效果以及 H5 移动端的实际样式实现。

### 5.1 核心配方

#### 5.1.1 极光玻璃卡片标准 CSS 配置

标准极光玻璃卡片（`.glass-aurora`）是 v7 主题中最基础的卡片单元，广泛应用于 H5 移动端首页、User PC 前台以及官网的常规内容承载场景。其 CSS 配置包含五个关键维度：背景渐变、背景滤镜（Backdrop Filter）、边框、三层复合阴影以及 inset 顶部高光线。每一项参数的设定均服务于"深邃通透"的整体视觉目标。

背景渐变采用从上至下的线性渐变（`180deg`），起始色为 `rgba(26, 36, 86, 0.55)`，终止色为 `rgba(21, 34, 74, 0.70)`。这一配色以皇家深蓝 `#0F1B3D` 为基底向上提亮，透明度从 55% 过渡到 70%，形成底部略深、顶部略浅的微妙纵深感。`backdrop-filter: blur(24px) saturate(180%)` 是整个玻璃效果的核心引擎：`blur(24px)` 提供足够的背景模糊度，确保后方内容（如极光渐变网格背景或光斑动画）不会干扰卡片内文字的阅读；`saturate(180%)` 则对透过卡片的背景色彩进行饱和度增强，使后方极光光晕在穿透玻璃时更加鲜艳夺目。`-webkit-backdrop-filter` 前缀确保了 Safari 浏览器的兼容性。

边框采用 `1px solid rgba(148, 163, 184, 0.18)`，这是一条极低不透明度的金属灰边框，其视觉存在感微弱但足以勾勒出卡片的物理边界，赋予玻璃材质"边缘折射"的质感。三层 `box-shadow` 的设计是 v7 相较于普通玻璃拟态的关键升级：第一层 `0 8px 32px rgba(15, 27, 61, 0.4)` 提供主投射阴影，将卡片从背景中抬升；第二层 `0 0 0 1px rgba(148, 163, 184, 0.05)` 是一道极细的轮廓线，增强边缘锐度；第三层 `inset 0 1px 0 rgba(255, 255, 255, 0.05)` 是嵌入卡片顶部的白色微光线，模拟玻璃表面对上方光源的镜面反射。`border-radius: 16px` 与 H5 卡片标准圆角 Token 保持一致。

```css
/* 极光玻璃卡片 — 标准配置 */
.glass-aurora {
  background: linear-gradient(180deg,
    rgba(26, 36, 86, 0.55) 0%,
    rgba(21, 34, 74, 0.70) 100%
  );
  backdrop-filter: blur(24px) saturate(180%);
  -webkit-backdrop-filter: blur(24px) saturate(180%);
  border: 1px solid rgba(148, 163, 184, 0.18);
  box-shadow:
    0 8px 32px rgba(15, 27, 61, 0.4),
    0 0 0 1px rgba(148, 163, 184, 0.05),
    inset 0 1px 0 rgba(255, 255, 255, 0.05);
  border-radius: 16px;
}
```

上述配置中的三层 `box-shadow` 各司其职，共同构建出玻璃材质的立体幻觉。主阴影（`0 8px 32px`）的模糊半径较大，负责营造卡片"悬浮于深邃背景之上"的空间关系；轮廓线（`0 0 0 1px`）无模糊偏移，仅在卡片外缘增加一道几乎不可见的冷灰描边，避免高透明度背景下边界消融；顶部 inset 高光线则是玻璃质感的精神所在——`inset 0 1px 0 rgba(255, 255, 255, 0.05)` 在卡片内部顶端渲染一条 1 像素高的白色半透明线，模拟光线从上方照射到玻璃表面时的菲涅耳反射（Fresnel Reflection）。开发者在使用该配置时应完整保留三层阴影，删减任何一层均会导致视觉质感的显著下降。

#### 5.1.2 轻量玻璃卡片配置

轻量玻璃卡片（`.glass-aurora-light`）适用于信息密度较高、卡片数量较多的场景，如资讯列表、次级功能入口和数据概览区块。相较于标准配置，轻量卡片降低了背景的不透明度和边框对比度，使卡片更加"透气"，减少多卡片堆叠时的视觉拥挤感。

轻量配置的背景渐变改用白色系透明渐变：`rgba(255, 255, 255, 0.06)` 至 `rgba(255, 255, 255, 0.02)`。这一配色刻意回避了深蓝底色，使卡片在深色背景上呈现出更轻薄的"浮冰"质感。`backdrop-filter` 的 blur 值从 `24px` 降至 `20px`，略微降低模糊强度的同时保留了足够的背景通透感；`saturate(180%)` 维持不变。边框透明度从 `0.18` 下调至 `0.12`，进一步弱化边界存在感，让相邻卡片之间的过渡更加柔和。

```css
/* 轻量玻璃 — 用于次要卡片 */
.glass-aurora-light {
  background: linear-gradient(180deg,
    rgba(255, 255, 255, 0.06) 0%,
    rgba(255, 255, 255, 0.02) 100%
  );
  backdrop-filter: blur(20px) saturate(180%);
  -webkit-backdrop-filter: blur(20px) saturate(180%);
  border: 1px solid rgba(148, 163, 184, 0.12);
  border-radius: 16px;
}
```

轻量配置省略了标准配置中的三层 `box-shadow`，仅保留基础边框。这一简化基于实际场景的视觉需求：当页面中存在 5 张以上轻量卡片时，每张卡片均携带完整阴影将导致暗色背景上出现多个重叠的阴影区域，产生"脏污"感。去掉外阴影后，卡片依靠 `backdrop-filter` 产生的背景模糊差异来界定范围，整体布局更加清爽。在实际项目中，轻量卡片通常与标准卡片混合使用——标准卡片承载核心信息（如资产总览、主要功能入口），轻量卡片承载辅助信息（如资讯条目、次要数据），通过玻璃密度的差异自然形成信息层级。

### 5.2 卡片变体体系

v7 主题定义了 7 种卡片变体，每种变体通过顶部高光线的颜色、背景透明度和边框样式来传达不同的功能语义。这一"色彩编码"机制使用户无需阅读文字即可通过视觉特征快速识别卡片所属的功能模块。以下表格汇总了全部 7 种变体的 CSS 类名、视觉特征及推荐使用场景。

| 变体类名 | 顶部高光线颜色 | 背景特征 | 边框特征 | 使用场景 |
|:---|:---|:---|:---|:---|
| `.card-aurora` | 金色 `#F0B90B` | 标准玻璃渐变 `rgba(26,36,86,0.55)` | `rgba(148,163,184,0.18)` 金属灰 | 通用卡片，最常用，适用于无特殊语义的内容承载 |
| `.card-aurora-cyan` | 青色 `#38BDF8` | 标准玻璃渐变 | `rgba(56,189,248,0.20)` 青色边框 | 科技/行情模块，如实时价格、K线数据区、技术指标卡 |
| `.card-aurora-violet` | 紫色 `#A78BFA` | 标准玻璃渐变，略带紫调 | `rgba(167,139,250,0.18)` 紫色边框 | AI/Web3 模块，如策略大脑、DeFi 入口、NFT 画廊 |
| `.card-aurora-rainbow` | 彩虹渐变 `金→青→紫` | 标准玻璃渐变，透明度略高 | `rgba(240,185,11,0.25)` 金色强边框 | VIP/核心功能，如会员中心、高阶交易权限、专属服务 |
| `.glass-royal` | 金色 `#F0B90B` | 高不透明 `85%–95%`，接近实色 | `rgba(240,185,11,0.30)` 金色粗边框 | 后台管理场景，需要更高阅读稳定性的面板和表单区 |
| `.card-deep` | 无高光线 | 深色实底 `#1A2456`，无 backdrop-filter | `1px solid #2E3A6B` 实色边框 | 数据密集型表格、复杂表单的底层容器，确保数据可读性 |
| `.card-light` | 无高光线 | 白色实底 `#FFFFFF` | `1px solid #E5E7EB` 浅灰边框 | Admin 后台（Light Admin 主题），浅色主题下的标准卡片 |

7 种变体按照视觉重量从高到低排列，覆盖了从"深色透明玻璃"到"浅色实底卡片"的完整光谱。前 4 种变体（`.card-aurora` 系列）共享相同的玻璃拟态基础配置，差异仅体现在顶部高光线的颜色和边框的色相偏转。`.card-aurora-rainbow` 作为 VIP 专用变体，其高光线采用 `linear-gradient(90deg, transparent → 金 → 青 →紫 → transparent)` 的彩虹渐变，是 4 种 Aurora 变体中视觉冲击力最强的一种，仅应用于需要用户特别关注的核心功能入口。`.glass-royal` 和 `.card-deep` 两种变体定位于"非玻璃"的实色领域，分别服务于后台管理和数据表格两类对阅读清晰度要求极高的场景。`.card-light` 则是 Light Admin 浅色主题下的唯一标准卡片，其白色背景与 Admin 主题的 `#F5F7FA` 浅灰底色协调统一。

顶部高光线的实现通常采用 `border-image` 属性或伪元素（`::before`）叠加渐变背景。以 `.card-aurora-cyan` 为例，其顶部青色高光线的推荐实现为：在卡片容器上添加一个 `::before` 伪元素，高度设为 `2px`，背景为 `linear-gradient(90deg, transparent, #38BDF8, transparent)`，并叠加微弱的青色发光阴影 `0 0 8px rgba(56, 189, 248, 0.40)`。这一处理使高光线不仅是静态装饰，更成为卡片顶部的一个微型光源，增强了玻璃材质的体积感。

### 5.3 卡片 Hover 交互效果

玻璃拟态卡片的 Hover 状态是交互设计中不可或缺的一环。v7 主题为极光玻璃卡片定义了一套完整的 Hover 变换规则，包含边框颜色迁移、背景亮度提升、阴影加深、微上浮位移以及统一的过渡动画参数。Hover 效果的设计目标是：在不破坏玻璃通透感的前提下，向用户传达"该元素可交互"的明确信号。

```css
.glass-aurora-hover:hover {
  border-color: rgba(240, 185, 11, 0.40);
  background: linear-gradient(180deg,
    rgba(30, 42, 95, 0.65) 0%,
    rgba(26, 36, 86, 0.80) 100%);
  box-shadow:
    0 12px 40px rgba(15, 27, 61, 0.5),
    0 0 32px rgba(240, 185, 11, 0.15),
    inset 0 1px 0 rgba(255, 255, 255, 0.08);
  transform: translateY(-2px);
  transition: all 0.4s cubic-bezier(0.4, 0, 0.2, 1);
}
```

上述 Hover 配置包含五项同步变换。第一，边框颜色从默认的冷灰 `rgba(148, 163, 184, 0.18)` 切换为暖金色 `rgba(240, 185, 11, 0.40)`，不透明度从 0.18 提升至 0.40，边框的视觉存在感显著增强，同时金色语义与"可交互/重要"形成关联。第二，背景渐变的两个色标均向更不透明的方向偏移：起始色从 `rgba(26, 36, 86, 0.55)` 变为 `rgba(30, 42, 95, 0.65)`，终止色从 `rgba(21, 34, 74, 0.70)` 变为 `rgba(26, 36, 86, 0.80)`，背景整体变亮约 10%–15%，使卡片在 Hover 时"点亮"。第三，`box-shadow` 的三层同步升级：主阴影的 Y 偏移从 `8px` 增至 `12px`、模糊半径从 `32px` 增至 `40px`，投射阴影更加扩散；新增一层金色辉光 `0 0 32px rgba(240, 185, 11, 0.15)`，在卡片外围形成金色光晕；顶部 inset 高光线的不透明度从 `0.05` 提升至 `0.08`，反射光更强。第四，`transform: translateY(-2px)` 使卡片整体上移 2 像素，产生被"托起"的物理感知。第五，`transition: all 0.4s cubic-bezier(0.4, 0, 0.2, 1)` 统一控制全部属性的过渡动画，400ms 的时长确保了变换过程的可感知性，而 `cubic-bezier(0.4, 0, 0.2, 1)` 缓动函数则提供了先快后慢的自然减速效果。

在实际开发中，Hover 效果的实现需要注意两个技术细节。其一，`backdrop-filter` 属性在部分浏览器中触发 GPU 层合成，与 `transform` 同时使用时应确保 `translateY(-2px)` 的硬件加速不会导致模糊效果闪烁；推荐为卡片容器添加 `will-change: transform` 以提前告知浏览器进行层优化。其二，400ms 的过渡时长适用于标准卡片，对于轻量卡片（`.glass-aurora-light`）建议缩短至 300ms，因为轻量卡片的视觉变化幅度较小，较长的过渡反而显得拖沓。此外，Hover 效果仅在支持 `hover` 媒体查询的设备上生效——在触控设备上，由于没有光标悬停状态，应将 Hover 样式映射到 `:active` 伪类，或完全禁用上浮和阴影变换以避免与滚动操作冲突。

### 5.4 H5 移动端实际卡片样式

H5 移动端由于屏幕宽度受限（最大 480px），卡片设计需要在有限的空间内同时保证视觉美感和信息密度。v7 主题在 H5 端对玻璃拟态卡片进行了移动端适配，主要调整包括：圆角统一使用 `16px`（对应 Token `xl`）、内边距收紧至 `12–16px`、backdrop-filter 的 blur 值略微降低以提升渲染性能。以下表格汇总了 H5 端三种典型卡片的样式规格。

| 卡片类型 | 背景样式 | `backdrop-filter` | 边框 | 圆角 | 内边距 | 阴影 |
|:---|:---|:---|:---|:---|:---|:---|
| 资产总览卡 | `linear-gradient(135deg, rgba(240,185,11,0.15), rgba(56,189,248,0.15))` 金青渐变 | `blur(20px) saturate(180%)` | `1px solid rgba(240,185,11,0.25)` 金色 | `16px` | `16px` | `0 0 24px rgba(240,185,11,0.10), inset 0 1px 0 rgba(255,255,255,0.05)` |
| 标准内容卡 | `linear-gradient(180deg, rgba(26,36,86,0.55), rgba(21,34,74,0.70))` 标准玻璃 | `blur(20px) saturate(180%)` | `1px solid rgba(148,163,184,0.18)` 标准灰 | `16px` | `12–14px` | 无额外阴影 |
| 牌照信任卡 | `linear-gradient(180deg, rgba(240,185,11,0.10), rgba(21,34,74,0.70))` 金色半透渐变 | `blur(20px) saturate(180%)` | `1px solid rgba(240,185,11,0.25)` 金色 | `16px` | `14–16px` | `inset 0 1px 0` 金色高光线 |

三种卡片在 H5 首页（H5Home）中承担不同的信息展示职责。资产总览卡作为用户进入 App 后视觉优先级最高的元素，其金青双色渐变背景与品牌主色调（皇家金 `#F0B90B` + 钻石青 `#38BDF8`）直接关联，金色边框和金色阴影进一步强化了"财富/资产"的语义。标准内容卡是 H5 端使用频率最高的卡片类型，行情模块、业务矩阵入口和热点资讯均使用该配置；省略额外阴影的设计使多个标准卡片在纵向堆叠时不会产生阴影重叠的杂乱感。牌照信任卡是特殊场景卡片，用于展示合规牌照和信任背书信息，其金色半透渐变背景与金色标题文字 `#FCD535` 共同构建出"权威/可信"的视觉印象。

#### 5.4.1 资产总览卡

资产总览卡（Asset Overview Card）是 H5 首页顶部的核心组件，直接展示用户的总资产估值和当日盈亏。其背景采用 `135deg` 对角线渐变，从 `rgba(240, 185, 11, 0.15)`（金色 15% 不透明）过渡到 `rgba(56, 189, 248, 0.15)`（青色 15% 不透明），对角线方向使两种品牌色在卡片内部形成自然交融。金色边框 `rgba(240, 185, 11, 0.25)` 的不透明度高于标准玻璃边框，使卡片边界在深色背景上更加醒目。阴影配置中的 `0 0 24px rgba(240, 185, 11, 0.10)` 是一道无偏移的金色环境光，从卡片四周均匀散射，营造出"发光的金盒子"效果。卡片内部通常包含三个信息层级：顶部为"总资产"标签（`12px`，`#7B89B8`），中部为资产数字（`24px`，`#F8FAFC`，字重 `700`），底部为当日盈亏百分比（`11px`，绿色 `#34D399` 或粉色 `#F472B6` 表示涨跌）。

#### 5.4.2 标准内容卡

标准内容卡（Standard Content Card）是 H5 端的功能型卡片，适用于行情列表、业务入口网格和资讯流等场景。其 CSS 配置与 PC 端的 `.glass-aurora` 基本一致，但 `backdrop-filter` 的 blur 值从 `24px` 调整为 `20px`，这一改动兼顾了移动设备的 GPU 性能——iOS Safari 和 Android Chrome 在处理 `backdrop-filter` 时均有不同程度的渲染开销，降低 4px 的模糊半径可减少约 15%–20% 的 GPU 合成时间，同时视觉差异几乎不可察觉。内边距采用 `12–14px` 的弹性范围：行情类卡片（信息密度高）使用 `12px`，功能入口类卡片（需要更大的触控区域）使用 `14px`。标准内容卡不设置额外阴影，完全依赖 `backdrop-filter` 产生的背景模糊差异来界定卡片边界，这一"无阴影玻璃"的处理方式使页面整体更加干净通透。

#### 5.4.3 牌照信任卡

牌照信任卡（License Trust Card）是 H5 首页底部的特殊展示卡片，用于呈现交易所持有的合规牌照、监管机构认证和安全审计信息。该卡片的设计目标是传递"权威可信"的品牌信号，因此在视觉处理上大量运用金色元素。背景渐变为 `180deg` 线性渐变，从 `rgba(240, 185, 11, 0.10)` 过渡到 `rgba(21, 34, 74, 0.70)`，顶部金色成分营造出"被金色光芒照耀"的隐喻。边框使用与资产总览卡相同的金色配置 `rgba(240, 185, 11, 0.25)`。顶部高光线采用金色渐变而非白色，即 `inset 0 1px 0` 配合金色渐变背景，使光线本身带有金色温度。卡片标题固定使用金色 `#FCD535`，与白色背景形成高对比度。在实际布局中，牌照信任卡内部通常采用横向滚动的牌照 Logo 列表或网格排列的认证图标，每个子元素外部包裹一层轻量玻璃胶囊作为背景容器。
