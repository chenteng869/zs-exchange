## 1. 项目概述与视觉定位

### 1.1 文档说明与技术栈

#### 1.1.1 文档版本信息与调研参考来源

本文档为**中萨交易所（ZS Exchange）综合设计与开发参考手册**的第1章，全手册共14章，涵盖从设计哲学到组件级实现规范的完整技术体系。文档对应的 UI 设计规格版本为 **Aurora Premium v7**（极光尊享主题），该版本在前代纯黑底色 `#0B1124` 的基础上进行了全面视觉升级，引入皇家深蓝基底与极光五色呼吸体系，标志着 ZS Exchange 视觉语言从"深色基础版"向"金融级沉浸式体验"的跨越。

本手册的调研参考覆盖了当前金融科技与加密货币交易领域的四款标杆产品：**Stripe 2026**（企业级金融界面设计）、**Coinbase Advanced**（专业交易终端交互模式）、**OKX Web3**（去中心化金融模块的视觉表达）、**Kraken Pro**（专业交易界面信息层级）、**Bybit V5**（衍生品交易终端的用户流程）。上述产品在其各自擅长的领域内代表了行业最高实践标准，本手册在吸收其设计方法论的基础上，结合 ZS Exchange 自身的品牌定位与业务特征，形成了独特的极光尊享视觉体系。

文档的目标读者为前端开发工程师、UI 设计师与产品经理。前端工程师可通过本文档获取精确的色值、尺寸与 CSS 实现参数；UI 设计师可据此理解设计决策的底层逻辑并保持设计输出的一致性；产品经理则可借助本文档评估功能实现的技术可行性与视觉呈现效果。

#### 1.1.2 核心技术栈

ZS Exchange 前端技术体系建立在现代化的全栈框架之上，以类型安全和开发效率为首要考量。核心技术栈的构成如下表所示。

| 技术层 | 技术选型 | 版本 | 职责说明 |
|--------|---------|------|---------|
| 前端框架 | Next.js | 14 | React 全栈框架，支持 App Router、服务端组件（Server Components）与流式渲染 |
| UI 库 | React | 18 | 组件化 UI 开发，并发特性（Concurrent Features）与自动批处理提升交互响应 |
| 样式方案 | Tailwind CSS | 3.x | 实用工具优先（Utility-First）的 CSS 框架，支持自定义 Design Tokens 与暗色模式 |
| 类型系统 | TypeScript | 5.x | 静态类型检查，覆盖组件 Props、Design Tokens 接口与 API 契约 |

技术栈的选型遵循"金融级前端"的稳定性原则。Next.js 14 的 App Router 架构为交易所多页面场景（首页、行情页、交易页、资产页）提供了清晰的路由组织与灵活的布局嵌套能力。React 18 的并发渲染特性在行情数据高频更新的场景下能够显著提升界面响应性能，避免大数据量推送导致的界面卡顿。Tailwind CSS 的 Utility-First 方法论与 Design Tokens 深度集成，使得极光主题的色彩、间距、阴影等视觉参数能够通过配置层统一管控，确保跨端一致性。TypeScript 的严格类型模式则贯穿于 `src/styles/tokens.ts`（543 行 Design Tokens 定义）与 `src/styles/design-tokens.ts`（483 行双主题令牌定义）之中，为设计系统提供了编译期保障。

#### 1.1.3 适用端说明

ZS Exchange 的视觉系统覆盖四类终端场景，每类场景在界面密度、交互模式与主题适配方面存在明确差异。

**H5 移动端** 面向移动设备浏览器与 WebView 容器，采用 `max-width: 480px` 的受限布局，模拟原生 App 体验。H5 端使用极光尊享主题的全部视觉特征——皇家深蓝背景、玻璃拟态卡片、极光光带与呼吸动画——并通过 `env(safe-area-inset-*)` 适配刘海屏安全区域。其页面结构遵循"固定顶栏 + 可滚动内容区 + 固定底部 Tab 导航"的三段式布局。

**User PC 前台** 面向桌面端浏览器用户，提供完整的交易功能与信息展示。PC 前台同样采用极光尊享主题，但在信息密度与布局复杂度上显著提升，支持多列网格、侧边面板与浮层弹窗等大面积组件。响应式断点系统覆盖 480px 至 1280px 以上的完整设备区间。

**官网** 作为品牌展示与流量入口，采用极光尊享主题中最具视觉冲击力的元素——Hero 级大标题渐变（`linear-gradient(135deg, #FCD535 → #38BDF8 → #A78BFA)`）、极光光流体背景与金色高光线装饰——以传达"尊享、科技、金融级"的品牌心智。

**Light Admin 管理后台** 面向内部运营、风控与合规人员，采用独立的浅色主题系统（`#F5F7FA` 浅灰底色 + 深色侧栏），在视觉风格上与前台系统形成明确区隔，强调信息密度与操作效率，弱化装饰性元素。

### 1.2 设计理念与核心关键词

#### 1.2.1 极光尊享主题定位

ZS Exchange 的核心视觉主题被命名为 **Aurora Premium（极光尊享）**，其定位由四个关键词定义：**深邃**、**通透**、**科技感**、**金融级**。

"深邃"体现为背景色调从纯黑 `#0B1124` 向皇家深蓝 `#0F1B3D` 的升级，配合多层极光渐变网格与星空点缀，营造出如极地夜空般深远的空间纵深感。"通透"由玻璃拟态 2.0（Glassmorphism 2.0）技术实现，通过 `backdrop-filter: blur(24px) saturate(180%)` 与 55%–70% 的背景透明度，使界面元素呈现出如磨砂水晶般的半透明质感，底层的光晕与色彩能够穿透卡片表面，形成丰富的层次叠加效果。"科技感"来源于极光五色呼吸体系——金（`#F0B90B`）、青（`#38BDF8`）、紫（`#A78BFA`）、翠（`#34D399`）、粉（`#F472B6`）——五种光色以不同的节奏缓慢呼吸移动，配合钻石青与 Web3 紫的品牌色系，传达出前沿技术与数字金融的尖端气质。"金融级"则通过金属感边框（`border-image` 渐变高光线）、严谨的信息层级与高对比度文字系统（`#F8FAFC` 主文字），确保在高度装饰化的视觉环境中，关键数据的可读性与操作的可信赖感不受影响。

#### 1.2.2 五大视觉特征概述

极光尊享主题的视觉识别系统由五项核心特征构成，这些特征共同定义了 ZS Exchange 的界面外观。

| 特征维度 | 具体规格 | 视觉作用 |
|---------|---------|---------|
| 皇家深蓝背景 | `#0F1B3D` + 多层极光渐变网格 + 星空点缀 | 替代纯黑底色，提供更丰富的空间深度与色彩层次，降低深色界面的压抑感 |
| 玻璃拟态 2.0 卡片 | `backdrop-filter: blur(24px) saturate(180%)`，透明度 55%–70%，圆角 16px | 卡片表面呈现半透明磨砂质感，底层光晕穿透可见，增强界面的通透感与层次叠加效果 |
| 极光五色呼吸体系 | 金 `#F0B90B` / 青 `#38BDF8` / 紫 `#A78BFA` / 翠 `#34D399` / 粉 `#F472B6`，各色的光晕以不同周期（12–22s）呼吸循环 | 背景光斑以缓慢节奏缩放、位移与透明度变化，营造出具有生命力的环境光效 |
| 金属感边框 | 卡片顶部 `border-image` 渐变高光线（金→青→紫），主边框 `rgba(148,163,184,0.18)` | 模拟金属反光边缘，提升界面的精致度与金融级品质感 |
| 光流体动画 | `aurora-flow`（12–14s 周期）、`aurora-drift`（18s 周期）、`aurora-rise`（22s 周期） | 三种光斑运动模式——缩放位移循环、慢速漂移、上下浮动——叠加形成复杂而和谐的环境动态 |

上述五项特征并非独立存在，而是构成了一个有机的视觉系统。皇家深蓝背景提供了深沉而丰富的画布，玻璃拟态卡片在此之上建立起半透明的信息层级，极光五色呼吸体系赋予背景以生命力，金属感边框为每张卡片勾勒出精致的轮廓，光流体动画则通过持续而克制的运动避免了静态界面的沉闷。在实际实现中，`src/app/globals.css`（905 行）定义了全部动画关键帧与工具类，`src/components/ui/Card.tsx`（83 行）封装了四种卡片变体与 Hover 交互，确保五大特征在代码层面的一致复用。

### 1.3 三大主题系统

ZS Exchange 采用三主题并行策略，针对不同业务场景提供差异化的视觉环境。每个主题在背景底色、色彩饱和度与装饰强度上进行了精确调校，以匹配用户的操作语境与心理预期。

| 主题名称 | 英文标识 | 适用场景 | 背景色调 | 视觉特征 | 用户类型 |
|---------|---------|---------|---------|---------|---------|
| 极光尊享 | Aurora Premium | 官网 / H5 移动端 / User PC 前台 | `#0F1B3D` 皇家深蓝 + 极光渐变 | 玻璃拟态 2.0、极光五色呼吸、光流体动画、金属感边框 | 外部交易用户 |
| 深色专业终端 | Dark Trading | 交易页 / K 线终端 | 比极光尊享更深的交易专用底色 | 去除装饰性光效，提高信息密度与数据可读性 | 专业交易员 |
| 浅色后台 | Light Admin | 管理后台 / 风控系统 | `#F5F7FA` 浅灰底色 + 深色侧栏 | 弱化装饰，强调信息层级与操作效率，标准表单与表格组件 | 运营 / 风控 / 合规人员 |

三主题的设计决策基于对三类用户行为模式的分析。外部交易用户在开户、浏览、充提币等流程中，需要感知到平台的专业性与品牌调性，极光尊享主题通过丰富的视觉层次与动态光效建立了这一信任感。专业交易员在 K 线分析与下单执行过程中，对信息密度与响应速度的要求远高于视觉装饰，Dark Trading 主题通过抑制非必要的光效元素，将视觉注意力完全集中于价格数据与图表指标。后台管理人员面对的是高频的表格操作、审核流程与风控决策，Light Admin 主题的浅色底色提供了接近桌面办公软件的熟悉环境，深色侧栏则保留了导航的清晰层级。

#### 1.3.1 Aurora Premium（极光尊享）

极光尊享是 ZS Exchange 的主品牌主题，覆盖所有面向外部用户的触点。该主题以 `#0F1B3D` 皇家深蓝为基底，叠加三层视觉要素：底层的极光渐变网格背景（由 `aurora-bg` 类与 6 种颜色的 `aurora-orb-*` 光斑元素构成）、中层的玻璃拟态卡片系统（`.glass-aurora` 标准配置与 `.glass-aurora-light` 轻量配置）、顶层的金属感高光线装饰（`.gold-top-line` 与 `.border-aurora-*` 边框系列）。

在色彩运用上，极光尊享主题拥有完整的品牌色系与功能色系。品牌色系包含品牌蓝（`#1677FF`）、钻石青（`#38BDF8`）、Web3 紫（`#7C3AED`）、皇家金（`#F0B90B`）及其衍生色阶；功能色系覆盖交易涨跌色（上涨 `#16C784` / 下跌 `#EA3943`）、语义状态色（警告 `#F59E0B`、信息 `#1677FF`、高风险 `#F97316`、冻结 `#7C3AED`）以及用户状态色、订单状态色与风险等级色。极光五色（金、青、紫、翠、粉）以 `rgba` 半透明形式作为环境光晕使用，不承载信息传达功能。

#### 1.3.2 Dark Trading（深色专业终端）

Dark Trading 主题专为交易场景设计，适用于 K 线图表页、深度图、订单簿与合约交易终端。该主题在极光尊享的基础上进一步压低背景明度，采用更深的交易专用底色（具体色值在后续章节中给出），以消除长时间盯盘带来的视觉疲劳。

与极光尊享主题相比，Dark Trading 在视觉处理上遵循"减法设计"原则：关闭光流体背景动画（`animation: none`），移除卡片顶部的高光线装饰，将玻璃拟态卡片的透明度从 55%–70% 下调至更保守的范围，边框对比度相应提高以确保数据区域的清晰分隔。图表网格线采用 `rgba(148,163,184,0.08)` 的极低对比度，坐标轴文字使用 `#7B89B8`，确保 K 线主体在复杂背景下保持高辨识度。该主题的目标是在保持深色环境一致性的同时，将用户的注意力完全引导至价格走势与交易数据。

#### 1.3.3 Light Admin（浅色后台）

Light Admin 主题面向管理后台与风控系统，是 ZS Exchange 三主题中唯一的浅色方案。该主题采用 `#F5F7FA` 浅灰作为全局底色，左侧导航栏使用深色（`#1E293B`）以保持导航层级的清晰，内容区域则保持浅色以提高长时间阅读与表单操作的舒适度。

在组件层面，Light Admin 使用 `.card-light` 浅色卡片替代玻璃拟态卡片，背景为纯白 `#FFFFFF` 搭配浅灰边框 `#E2E8F0`，文字色系统也切换为适合浅色背景的深灰系列（主文字 `#1E293B`、次文字 `#64748B`、辅助文字 `#94A3B8`）。按钮系统保留了 primary（`#1677FF`）、danger（`#DC2626`）、success（`#16A34A`）等语义色，但移除了极光主题中的渐变按钮与辉光效果，改用扁平化设计以提高操作界面的理性感。图表色板也调整为更适合浅色背景的高饱和度方案，确保后台数据可视化的可读性。

三主题之间的切换通过 `tailwind.config.ts` 中的暗色模式配置实现，Design Tokens 的双主题定义（`src/styles/design-tokens.ts`）为每个 Token 提供了暗色与浅色两种取值，主题切换在运行时通过 CSS 变量重计算完成，无需重新加载样式文件。这一架构确保了同一套组件代码能够在三种主题环境下保持一致的交互行为与可维护性。
