# 6. 按钮系统

按钮系统是用户与交易所界面发生交互的核心触点，其设计需要在视觉层次、操作反馈和品牌识别之间取得精确平衡。中萨交易所（ZS Exchange）的按钮体系覆盖 Web 前台、Admin 管理后台以及 H5 移动端三大场景，共定义 8 种语义变体、3 级尺寸梯度、5 种交互状态，并在 Aurora Premium v7 主题中引入 3 款极光特色按钮。全系统遵循统一的 4px 基准网格，所有尺寸参数均与间距 Token 对齐，确保跨端开发的一致性。

## 6.1 按钮变体

### 6.1.1 8 种变体 Web 端与 Admin 端样式对照

按钮变体（Button Variant）按语义功能划分为 8 种类型，覆盖从主操作到弱化的完整操作强度谱系。每种变体在 Web/H5 端（深色主题）与 Admin 端（浅色主题）分别配置了独立的色值，以适应不同背景的对比度需求。

**Primary（主按钮）** 承担界面中最核心的操作入口，如表单提交、确认下单等。Web 端采用品牌蓝 `#1677FF` 作为背景色，文字为纯白色 `#FFFFFF`；Admin 端同样使用品牌蓝，保持跨端认知一致性。该变体在视觉上具有最高的权重，通常以 solid 填充形式出现，吸引用户首要注意力。

**Secondary（次按钮）** 用于与主按钮配对的反向操作，如取消、返回上一步。Web 端采用透明背景加 `#334155` 边框的线框风格，在深色界面中形成柔和的次级视觉；Admin 端则使用透明背景加 `#D1D5DB` 边框，适配浅色主题下的可读性。

**CTA（Call-To-Action，行动号召按钮）** 是品牌差异化最强的变体，专门用于高价值转化场景，如新用户注册、VIP 订阅、活动参与等。该变体采用蓝紫渐变 `linear-gradient(135deg, #1677FF, #7C3AED)` 作为背景，并叠加青色辉光阴影 `0 4px 24px rgba(56,189,248,0.35)`，在深色背景上形成强烈的视觉跃出感。Web 端与 Admin 端共享同一套 CTA 样式。

**Success（成功按钮）** 用于正向确认类操作，如启用某项功能、通过审核。Web 端使用翠绿 `#16C784`，与交易上涨色保持同系；Admin 端则采用略深的 `#16A34A`，在浅色背景上提供更稳定的对比度。

**Danger（危险按钮）** 用于不可逆的破坏性操作，如删除账户、撤销订单。Web 端使用红宝石色 `#EA3943`，Admin 端使用 `#DC2626`，两者均属于高饱和红色系，在各自主题下均能快速传达风险警示。

**Warning（警告按钮）** 用于需要用户谨慎对待的操作，如解除风控、修改关键参数。背景色为 `#F59E0B`，文字色为深色 `#0F1830`，采用深色文字在黄色底上的搭配方式，确保可读性优于白色文字。

**Ghost（幽灵按钮）** 是最轻量的变体，透明背景、无可见边框，文字色在 Web 端为 `#94A3B8`、Admin 端为 `#6B7280`。适用于工具栏中的图标按钮、批量操作栏中的低频命令等场景，避免界面因过多实心按钮而显得拥挤。

**Link（链接按钮）** 以纯文字形态呈现，外观类似超链接。Web 端文字色为品牌蓝 `#1677FF`，Hover 状态下显示下划线；Admin 端保持一致。适用于"查看详情"、"了解更多"等引导类操作，视觉权重最低。

| 变体名称 | Web 端背景/边框样式 | Web 端文字色 | Admin 端背景/边框样式 | Admin 端文字色 | 典型场景 |
|:---|:---|:---|:---|:---|:---|
| **primary** | `bg: #1677FF` | `#FFFFFF` | 同 Web 端 | `#FFFFFF` | 主要操作、表单提交、确认 |
| **secondary** | 透明 + `border: #334155` | `#F8FAFC` | 透明 + `border: #D1D5DB` | `#1F2937` | 取消、返回、次要操作 |
| **cta** | 渐变 `#1677FF` → `#7C3AED` + 发光 | `#FFFFFF` | 同 Web 端 | `#FFFFFF` | 注册、订阅、活动参与 |
| **success** | `bg: #16C784` | `#FFFFFF` | `bg: #16A34A` | `#FFFFFF` | 启用、通过确认、正向操作 |
| **danger** | `bg: #EA3943` | `#FFFFFF` | `bg: #DC2626` | `#FFFFFF` | 删除、移除、拒绝 |
| **warning** | `bg: #F59E0B` | `#0F1830` | 同 Web 端 | `#0F1830` | 谨慎操作、风控解除 |
| **ghost** | 透明无框 | `#94A3B8` | 透明无框 | `#6B7280` | 图标按钮、弱化操作 |
| **link** | 透明无框 | `#1677FF` | 透明无框 | `#1677FF` | 文字链接型操作、查看详情 |

上表完整列出了 8 种变体在双主题下的色值差异。需要特别注意的是，CTA、warning 和 link 三种变体在 Web 端与 Admin 端完全复用同一套色值，这是因为 CTA 的渐变本身在深浅两种背景下均具有足够的对比度，warning 的黄色底配深色文字的组合同样不受背景色影响，而 link 作为文字链形态依赖的是上下文环境的文字对比。success 和 danger 则因 Admin 浅色主题需要更高的对比稳定性而采用了比 Web 端略深的色值。开发者在实现时应以 design-tokens.ts 中定义的主题变量为准，避免硬编码色值。

## 6.2 按钮尺寸

### 6.2.1 三级尺寸完整规格

按钮尺寸系统采用三级梯度设计，分别对应小（sm）、中（md）、大（lg）三种规格，覆盖从紧凑工具栏到 Hero 区域号召性按钮的全场景需求。所有尺寸参数基于 4px 基准网格推导，高度、内边距、字号、图标间距之间保持固定的比例关系，确保视觉节奏的连贯性。

**sm（小尺寸）** 高度为 `32px`（对应 Tailwind 的 `h-8`），水平内边距 `12px`（`px-3`），字号 `14px`（`text-sm`），图标与文字间距 `6px`，圆角 `8px`（`rounded-lg`）。该规格适用于数据表格内的行内操作按钮、筛选栏中的标签按钮、卡片头部的工具按钮等空间受限的场景。

**md（中尺寸）** 高度为 `40px`（`h-10`），水平内边距 `20px`（`px-5`），字号 `14px`（`text-sm`），图标与文字间距 `8px`，圆角 `8px`。md 是系统的默认按钮尺寸，适用于表单底部操作区、弹窗确认栏、页面级操作按钮等绝大多数常规场景。其高度与标准输入框（`40px`-`44px`）对齐，保证表单元素在同一行中视觉等高。

**lg（大尺寸）** 高度为 `48px`（`h-12`），水平内边距 `28px`（`px-7`），字号 `16px`（`text-base`），图标与文字间距 `10px`，圆角 `8px`。lg 规格用于页面级核心转化入口，如注册页提交按钮、资产充值页主操作按钮等需要强化点击冲动的场景。其较大的水平内边距使按钮在留白充分的 Hero 区域中不至于显得单薄。

| 尺寸标识 | 高度 | 水平内边距 | 字号 | 图标间距 | 圆角 | Tailwind 类名参照 | 适用场景 |
|:---|:---|:---|:---|:---|:---|:---|:---|
| **sm** | `32px` | `12px` | `14px` | `6px` | `8px` | `h-8 px-3 text-sm rounded-lg` | 表格行内操作、筛选标签、紧凑工具栏 |
| **md** | `40px` | `20px` | `14px` | `8px` | `8px` | `h-10 px-5 text-sm rounded-lg` | 表单操作、弹窗确认、页面常规按钮（默认） |
| **lg** | `48px` | `28px` | `16px` | `10px` | `8px` | `h-12 px-7 text-base rounded-lg` | Hero CTA、核心转化入口、大额操作确认 |

三种尺寸的高度增量遵循 `+8px` 的等差规律（32px → 40px → 48px），水平内边距的增量为 `+8px` 至 `+8px`（12px → 20px → 28px），图标间距增量为 `+2px`（6px → 8px → 10px）。这种一致的增量模式使得按钮家族在并排放置时呈现和谐的阶梯感。所有尺寸统一使用 `8px` 圆角，与系统圆角规范中的 `md` Token 保持一致，避免过多圆角规格带来的碎片化。需要指出的是，H5 移动端因触控交互的需要，按钮的最小触控区域应不小于 `44px × 44px`，因此 sm 规格在移动端应谨慎使用，或辅以额外的触控热区扩展。

## 6.3 H5 移动端按钮

H5 移动端按钮在保持与 Web 端语义一致的基础上，针对移动场景的视觉习惯和触控特性进行了专项优化。移动端按钮采用自适应高度策略（以垂直内边距驱动），圆角统一为 `12px`，字号 `13px`，主操作字重 `700`、次操作字重 `600`，在较小屏幕上依然保持良好的可读性和触控体验。

### 6.3.1 金色主按钮

金色主按钮用于充值、买入等资产流入类操作，是移动端最高频的交互元素之一。其背景采用金渐变 `linear-gradient(135deg, #F0B90B, #FCD535)`，从皇家金过渡到金亮，在深色极光背景上形成温暖的视觉焦点。文字色为 `#0F1B3D`（皇家深蓝），深色文字在金色底上提供了优异的对比度，避免了白色文字在亮色底上常见的光晕模糊问题。按钮叠加金色阴影 `0 4px 16px rgba(240,185,11,0.30)`，在滚动页面时金色光晕能够穿透内容层，持续引导用户注意力。

### 6.3.2 蓝色操作按钮

蓝色操作按钮用于交易、兑换等核心功能入口，与金色主按钮构成移动端的双主色操作体系。背景采用青渐变 `linear-gradient(135deg, #38BDF8, #6366F1)`，从钻石青过渡到暮蓝，传达科技感和信任感。文字色同样使用 `#0F1B3D` 深色文字，确保跨按钮家族的阅读体验统一。阴影为青色辉光 `0 4px 16px rgba(56,189,248,0.30)`，与金色主按钮的阴影形成冷暖对比，帮助用户在视觉层面快速区分操作类型。

### 6.3.3 次操作按钮

次操作按钮用于提币、转出等资产流出类操作，或作为金色/蓝色主按钮的配对选项。该变体采用低调处理策略：背景为半透明 `rgba(148,163,184,0.10)`，边框为 `1px solid rgba(148,163,184,0.20)` 的玻璃质感细线，文字色为 `#F8FAFC`。不叠加任何阴影效果，使其在视觉层级上自然退后，避免与主操作按钮争夺注意力。这种处理方式在移动端有限的屏幕空间内尤为重要，能够有效地建立操作优先级，减少用户的误触概率。

| 按钮类型 | 背景样式 | 文字色 | 阴影/发光 | 字重 | 典型用途 |
|:---|:---|:---|:---|:---|:---|
| **金色主按钮** | `linear-gradient(135deg, #F0B90B, #FCD535)` | `#0F1B3D` | `0 4px 16px rgba(240,185,11,0.30)` | `700` | 充值、买入、资产流入 |
| **蓝色操作按钮** | `linear-gradient(135deg, #38BDF8, #6366F1)` | `#0F1B3D` | `0 4px 16px rgba(56,189,248,0.30)` | `700` | 交易、兑换、核心功能 |
| **次操作按钮** | `rgba(148,163,184,0.10)` + 玻璃边框 | `#F8FAFC` | 无 | `600` | 提币、转出、次要操作 |

上表汇总了 H5 移动端三种按钮的完整样式参数。金色与蓝色两种主按钮在设计上有意识地形成对称关系：两者均使用 `135deg` 对角渐变方向，均采用深色文字以保证可读性，阴影的扩散半径和透明度也保持一致（`0 4px 16px`，透明度 `0.30`）。这种对称性使得用户在熟悉其中一种按钮后，能够快速理解另一种的交互含义。次操作按钮的玻璃质感风格则与系统整体的 Glassmorphism 2.0 设计语言保持一致，其半透明背景在实际渲染中会轻微透出下方的内容，增强了移动端的层级纵深感。

## 6.4 按钮交互状态

### 6.4.1 五状态处理

按钮交互状态（Interaction State）定义了用户从发现到触发操作的全过程中的视觉反馈路径。系统为所有按钮变体统一设计了 5 种状态：Normal（常态）、Hover（悬浮）、Active/Pressed（按下）、Disabled（禁用）和 Loading（加载中），状态切换之间的过渡动画时长统一为 `200ms-300ms`，缓动函数为 `cubic-bezier(0, 0, 0.2, 1)`（ease-out）。

**Normal 状态** 即按钮的默认显示样式，按各变体定义的背景色、边框、文字色和阴影参数渲染。该状态下按钮处于静态，无额外的动画或特效。

**Hover 状态** 在鼠标指针悬浮于按钮上方时触发。视觉处理包含三个同步变化：按钮整体上移 `translateY(-1px)`，营造轻盈的浮起感；阴影扩散增强，solid 按钮的阴影从基础级提升一级（如从 sm 到 md），CTA 和极光特色按钮的辉光透明度提升 `0.10`-`0.15`；primary 变体的背景色从 `#1677FF` 过渡到悬停蓝 `#4096FF`，secondary 变体的边框色提亮。`200ms` 的过渡时长确保反馈即时但不过于急促。

**Active/Pressed 状态** 在用户按下按钮时触发，模拟物理按键的按压反馈。Primary 变体的背景色切换为激活蓝 `#0958D9`，Success 变体切换为更深一级的绿色，Danger 变体同理加深。同时按钮取消 Hover 状态的上浮效果，恢复 `translateY(0)` 并略微内缩 `scale(0.98)`，整体呈现"被按下"的凹陷感。该状态的持续时间极短，通常在鼠标释放或手指抬起后立即结束。

**Disabled 状态** 用于按钮所依赖的条件未满足时的不可交互状态，如表单校验未通过、余额不足、权限受限等。处理方式为全局透明度降至 `0.5`（`opacity: 0.5`），鼠标指针变为 `not-allowed`，同时移除所有 hover 和 active 的交互响应。Disabled 状态下按钮保持原有的颜色和形状信息，仅通过透明度传达不可用状态，这比传统的灰度化方案保留了更多的视觉上下文。

**Loading 状态** 用于异步操作进行中的等待场景，如提交订单、发起提现请求。按钮内部的文字内容被一个旋转的 spinner 动画替代，spinner 使用按钮变体对应的主题色（primary 用白色，CTA 用青色等），直径为按钮高度的 `40%`，以 `1s` 为周期匀速旋转。按钮在 Loading 期间保持原有背景和边框样式，但禁用点击事件，防止重复提交。若需显示进度信息，可在 spinner 旁保留简短的百分比文字。

五状态处理的核心原则是在不破坏按钮基本识别性的前提下，提供清晰、即时的操作反馈。Hover 的上浮和阴影增强引导用户确认可点击性，Active 的色变和缩微反馈确认操作已被接收，Disabled 的半透明态明确告知当前不可操作的原因不在按钮本身，Loading 的 spinner 则有效管理了异步等待期间的用户预期。

## 6.5 极光特色按钮（v7 新增）

极光特色按钮（Aurora Premium Buttons）是 v7 主题中引入的高端视觉组件系列，专为品牌强相关的关键场景设计。该系列按钮融合了极光渐变色系、动态辉光效果和交互式光效动画，在视觉表现力上显著超越标准按钮变体，适用于首页 Hero 区、VIP 入口、限量活动等高价值展示位。

### 6.5.1 btn-aurora-cta

`.btn-aurora-cta` 是极光 CTA 按钮，采用蓝紫渐变背景 `linear-gradient(135deg, #1677FF, #7C3AED)`，叠加青色辉光 `0 0 24px rgba(56,189,248,0.35)` 和 `0 0 48px rgba(56,189,248,0.15)` 的双重发光层，在深色背景上形成类似极光的光晕效果。Hover 状态下，按钮上浮 `translateY(-2px)` 并触发**闪光扫过效果（shine sweep）**：一道倾斜的白色高光带以 `400ms` 的时长从按钮左上角扫向右下角，高光带的透明度为 `rgba(255,255,255,0.25)`，宽度约为按钮宽度的 `30%`，模拟金属表面被光线扫过的质感。该效果通过 CSS 伪元素配合 `background-position` 动画实现，无需 JavaScript 参与。

### 6.5.2 btn-aurora-gold

`.btn-aurora-gold` 是极光金色按钮，面向 VIP 专属功能和财富类模块。背景采用金渐变 `linear-gradient(135deg, #FCD535, #F0B90B)`，叠加强金辉光 `0 4px 24px rgba(240,185,11,0.45)` 和 `0 0 56px rgba(240,185,11,0.20)`，金色光晕的扩散半径和透明度均高于标准金色按钮，在视觉上具有更强的"发光体"特征。Hover 时同样触发闪光扫过效果，但高光带色调略偏暖（`rgba(255,248,220,0.30)`），与金色底形成更和谐的反射质感。按钮文字色为 `#0F1B3D`，字重 `700`，在金色背景上呈现沉稳而醒目的深色标签效果。

### 6.5.3 btn-royal-gold

`.btn-royal-gold` 是皇家金按钮，设计上兼顾 Admin 管理后台和 Web 前台的双主题兼容性，是唯一一款在深浅两种背景下均保持完整视觉表现力的极光按钮。背景采用皇家金渐变 `linear-gradient(135deg, #FEF3C7, #FCD535, #F0B90B, #E8A317)`，四色梯度从极浅的奶油金过渡到深金，比 `btn-aurora-gold` 的梯度更丰富，金属质感更强。辉光层使用标准金色发光 `0 0 32px rgba(240,185,11,0.35)`。与另外两款极光按钮不同，`.btn-royal-gold` 在 Hover 状态下不触发闪光扫过动画，而是采用更克制的上浮 `translateY(-1px)` 加辉光增强方案，以适配 Admin 后台相对保守的视觉语境。在 Admin 浅色背景下，该按钮的金色渐变依然保持足够的饱和度，辉光效果会相对减弱但仍可辨识，确保了跨主题的一致性体验。

三款极光特色按钮在实现上共享同一套基础结构：均使用相对定位的容器包裹渐变背景和伪元素动画层，辉光效果通过 `box-shadow` 的多层叠加实现，闪光扫过动画通过 `@keyframes` 定义并在 Hover 时触发。开发者在调用时仅需添加对应的类名即可，无需手动配置任何渐变或动画参数。极光按钮不建议在单一页面中密集使用，推荐每个视口区域内至多出现一款，以维持其视觉稀缺性和品牌高端感。
