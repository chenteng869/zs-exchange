# 萨摩亚交易所开发施工标准

明白，你的重点不是“复制币安源码”，而是：

> **Trae Solo / AI 编程出来的 UI 很差，页面不像一线交易所，不够高级、不够专业、不够金融科技感。**

这个问题很正常。AI 写代码可以很快，但如果没有完整的 **UI设计系统 + 页面规范 + 组件规范 + 参考标准 + 自动验收规则**，它生成出来的界面通常会：

```Plain
1. 像模板站；
2. 间距混乱；
3. 字体层级差；
4. 按钮大小不统一；
5. 颜色不高级；
6. 卡片很粗糙；
7. 交易页面没有金融终端感；
8. 移动端体验像普通后台；
9. 没有币安/OKX/Coinbase那种成熟产品感。
```

所以你现在要解决的不是“AI会不会写代码”，而是：

> **先建立一套交易所级 UI Design System，然后让 Trae Solo / Codex 按规范生成代码。**

***

# 二、为什么 AI 直接做 UI 效果差？

因为 AI 默认只会“把功能做出来”，不会自动理解：

```Plain
金融交易所的视觉密度
行情页的信息层级
交易按钮的心理暗示
红绿色交易语义
K线区域布局
买卖盘口比例
移动端底部导航习惯
资产页的安全感
首页的国际交易所质感
```

它生成的 UI 通常是：

```Plain
普通 SaaS 风格
普通表单风格
普通卡片风格
普通移动 App 风格
```

但交易所需要的是：

```Plain
金融终端风
高信息密度
低干扰
强状态识别
深色专业感
数据优先
操作路径极短
```

***

# 三、正确做法：不要让 AI 设计，让 AI 执行

你的流程应该改成：

```Plain
1. 先定 UI 风格
2. 先定 Design Token
3. 先定组件规范
4. 先定页面结构
5. 先定参考图
6. 再让 Trae Solo / Codex 写代码
7. 用截图自动验收
8. 不合格就让 AI 按差异修复
```

一句话：

> **AI 不负责审美，AI 负责按规范施工。**
