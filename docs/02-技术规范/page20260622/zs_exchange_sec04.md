# 4. 圆角、阴影与光效系统

中萨交易所 UI 采用了一套完整的圆角、阴影与光效体系，旨在通过一致的几何语言建立视觉层级，同时以极光主题（Aurora Premium）特有的发光效果营造科技感与金融尊贵气质。本章系统性地阐述从基础圆角 Token 到多层复合阴影、从功能性提升到极光辉光效果的全部规格参数，为前端实现提供可直接落地的 CSS 配置参考。

## 4.1 圆角规范

### 4.1.1 8级圆角Token

中萨交易所的圆角系统采用 8 级离散刻度，覆盖了从无圆角到完全圆角的完整区间。该体系与 Tailwind CSS 的 rounded 命名规范保持映射关系，同时在数值上针对金融交易场景进行了微调——例如将 H5 卡片的标准圆角设定为 `16px`（`xl`），以在移动端小屏幕上营造更柔和、更具现代感的视觉体验。每一级 Token 均对应明确的组件使用场景，避免圆角值的随意选用。

| Token | 主体系值 | Tailwind映射 | 备选值(Admin) | 典型用途 |
|-------|---------|-------------|--------------|---------|
| `none` | `0px` | `rounded-none` | `0px` | 数据表格、代码块、输入框、图表容器（前后台通用） |
| `sm` | `4px` | `rounded-sm` | `2px` | 前台标签/徽章；后台紧凑徽章、内联标签 |
| `md` | `8px` | `rounded-md` | `6px` | 前台标准按钮/输入框；后台表单、次要按钮 |
| `lg` | `12px` | `rounded-lg` | `12px` | 中型卡片、对话框、浮动面板（前后台通用） |
| `xl` | `16px` | `rounded-xl` | `16px` | **H5 卡片标准圆角**、大卡片、模态框（前后台通用） |
| `2xl` | `24px` | `rounded-2xl` | — | 大型面板、模态框容器、全宽区块 |
| `3xl` | `32px` | `rounded-3xl` | — | Hero 级卡片、全屏弹窗、欢迎浮层 |
| `full` | `9999px` | `rounded-full` | `9999px` | 头像、药丸按钮（Pill）、圆形徽章（前后台通用） |

上述 8 级圆角体系的设计逻辑遵循"场景驱动、逐级递增"的原则。`none` 至 `md` 区间主要服务于功能性、高密度信息展示区域——表格和输入框采用直角或微圆角，以保证数据排列的整齐性和阅读效率；`lg` 至 `xl` 区间是中萨交易所使用频率最高的圆角段，覆盖了绝大多数卡片和容器组件，`xl(16px)` 作为 H5 移动端卡片的标准圆角值，能够在 375px–480px 的屏幕宽度内提供最佳的视觉平衡；`2xl` 与 `3xl` 则用于需要强烈视觉焦点的大型容器，配合极光渐变背景构建层次感；`full(9999px)` 通过 CSS 的极大半径值实现完全椭圆形，专用于头像和 Pill 按钮等需要"无限圆角"的组件。

备选圆角值主要服务于 Admin 管理后台场景，其数值整体比主体系减小 2px，在保持设计语言一致的同时，提升了后台界面中表格密集排列时的空间利用效率。备选定义中的 `lg(12px)` 和 `xl(16px)` 与主体系保持完全一致，这意味着在跨越前台（Aurora Premium）与后台（Light Admin）两个主题时，中型及以上容器的圆角视觉语言是连续统一的，不会产生认知断裂。开发者在实现双主题系统时，只需对 `sm` 和 `md` 两级进行条件切换，`lg` 及以上级别可直接共享同一套 Token 值。这种"部分共享、局部差异化"的策略在保证视觉一致性的前提下，兼顾了不同场景对信息密度的差异化需求——前台面向终端用户，更强调视觉亲和力；后台面向运营管理人员，较小的圆角有助于提升信息排列的紧凑度。

## 4.2 基础阴影体系

### 4.2.1 四级基础阴影

基础阴影系统（Box Shadow）采用四级分层设计，从微妙的垂直提升到强烈的覆盖层深度，覆盖了界面中所有需要表达 Z 轴层级的场景。四级阴影均以纯黑色（`rgba(0,0,0)`）为基准色，通过调整透明度和偏移量来实现不同的深度感知。在深色极光背景（`#0F1B3D`）之上，这些阴影的透明度被精确控制，既能提供足够的层级暗示，又不会造成沉闷、脏污的视觉感受。

| 名称 | CSS `box-shadow` 值 | Z轴层级 | 典型使用场景 |
|------|-------------------|---------|-------------|
| `sm`（微妙提升） | `0 1px 2px rgba(0, 0, 0, 0.05)` | 1–2px | 小型按钮默认态、Tooltip 浮动提示、内联下拉菜单 |
| `md`（卡片悬浮） | `0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -2px rgba(0, 0, 0, 0.1)` | 4–6px | 卡片 Hover 状态、可点击面板悬浮反馈 |
| `lg`（弹窗深度） | `0 10px 15px -3px rgba(0, 0, 0, 0.1), 0 4px 6px -4px rgba(0, 0, 0, 0.1)` | 10–15px | 模态对话框（Modal）、下拉选择面板、日期选择器弹层 |
| `xl`（覆盖层） | `0 20px 25px -5px rgba(0, 0, 0, 0.1), 0 8px 10px -6px rgba(0, 0, 0, 0.1)` | 20–25px | 全屏遮罩层、侧边导航抽屉（Drawer）、全屏弹窗 |

四级阴影的设计遵循了 CSS 多重阴影（multi-layer shadow）的最佳实践：每一级阴影均由两个阴影层叠加而成——第一层提供主要的方向性深度（较大的垂直偏移和模糊半径），第二层提供边缘收束（较小的偏移和负扩展值），从而在视觉上形成更自然、更接近真实光照条件的柔和过渡。以 `md` 级为例，`0 4px 6px -1px` 构建主要下沉阴影，`0 2px 4px -2px` 则 tighter 地收束底部边缘，避免阴影过度扩散导致界面显得松散。

在实际开发中，基础阴影与 `transition` 属性配合使用，能够实现流畅的悬浮深度变化。例如卡片组件从默认态到 Hover 态时，`box-shadow` 从无阴影平滑过渡到 `md` 级别，配合 `transition: box-shadow 0.25s cubic-bezier(0.4, 0, 0.2, 1)`，可在 250ms 内完成深度感知的变化，向用户传达明确的可交互反馈。对于弹窗类组件，建议在进入动画中将阴影从 `sm` 级别逐渐提升到 `lg` 或 `xl` 级别，配合 `transform: scale(0.95) → scale(1)` 的缩放过渡，可营造出从远处飞入并落定的空间感。值得注意的是，在深色极光背景下，阴影的感知强度相较浅色背景会自然减弱，因此四级阴影的透明度设定（0.05–0.10）较传统浅色 UI（通常 0.15–0.30）明显更低，开发者在自定义组件时不应直接套用浅色系统的阴影参数，而应以本章给出的 rgba 值为准。

## 4.3 极光发光效果（v7核心）

### 4.3.1 七种辉光效果完整规格

极光发光效果（Aurora Glow）是中萨交易所 v7 版本"极光尊享"主题最具辨识度的视觉特征之一。与常规阴影以黑色为基准不同，极光辉光采用品牌色系中的高饱和色彩作为光源色，通过多层 `box-shadow` 的叠加营造出霓虹般的呼吸感光晕。该体系包含 7 种独立的辉光效果，覆盖品牌蓝、Web3 紫、皇家金以及极光多色组合，每种效果均定义了精确的 CSS 值与应用场景。

| 名称 | CSS `box-shadow` 值 | 光源色 | 典型使用场景 |
|------|-------------------|--------|-------------|
| `glow-blue`（蓝辉光） | `0 0 24px rgba(22, 119, 255, 0.30)` | `#1677FF` 品牌蓝 | 品牌元素高亮、主链接悬停、蓝色按钮焦点态 |
| `glow-purple`（紫辉光） | `0 0 24px rgba(124, 58, 237, 0.30)` | `#7C3AED` Web3紫 | Web3 模块装饰、AI 功能区标识、紫色主题卡片 |
| `glow-gold`（金辉光） | `0 0 32px rgba(240, 185, 11, 0.35), 0 0 48px rgba(240, 185, 11, 0.15)` | `#F0B90B` 皇家金 | VIP 标识、财富模块、金色按钮默认辉光 |
| `glow-gold-strong`（强金辉） | `0 4px 24px rgba(240, 185, 11, 0.45), 0 0 56px rgba(240, 185, 11, 0.20)` | `#F0B90B` 皇家金 | VIP 核心 CTA、重要财富入口、需要极强视觉牵引的行动点 |
| `glow-aurora`（极光合辉） | `0 8px 32px rgba(56, 189, 248, 0.20), 0 0 64px rgba(167, 139, 250, 0.15)` | `#38BDF8` 青 + `#A78BFA` 紫 | 极光玻璃卡片装饰、首页核心功能区、品牌展示位 |
| `glow-aurora-cyan`（极光青辉） | `0 0 24px rgba(56, 189, 248, 0.35), 0 0 48px rgba(56, 189, 248, 0.15)` | `#38BDF8` 钻石青 | 行情模块聚焦、科技感按钮、青色主题入口 |
| `glow-aurora-violet`（极光紫辉） | `0 0 24px rgba(167, 139, 250, 0.35), 0 0 48px rgba(167, 139, 250, 0.15)` | `#A78BFA` 极光紫 | AI 策略模块、Web3 功能区、紫色主题装饰 |

7 种辉光效果在结构上可分为三类。第一类为单色单层辉光（`glow-blue`、`glow-purple`），采用单一 `box-shadow` 层，24px 的模糊半径配合 0.30 的 alpha 值，适用于需要点缀性发光的交互反馈场景，如链接悬停或焦点指示。第二类为单色双层辉光（`glow-gold`、`glow-gold-strong`、`glow-aurora-cyan`、`glow-aurora-violet`），通过两层不同半径（通常 24px/32px + 48px/56px）和不同透明度（外层较内层弱）的阴影叠加，构建由内到外逐渐衰减的光晕层次；其中 `glow-gold-strong` 相较 `glow-gold` 提升了 alpha 值（0.35→0.45、0.15→0.20）并增加了 4px 的垂直偏移，形成更具方向性的"发光体"质感，专用于页面中需要最高视觉优先级的 CTA 位置。第三类为多色复合辉光（`glow-aurora`），将青色与紫色两种光源融合于同一阴影声明中，64px 的超大模糊半径营造出极光交织的梦幻效果，是中萨交易所品牌视觉的核心表达。

从实现角度看，所有极光辉光效果的 `box-shadow` 均不含水平/垂直偏移的弥散层，而是采用零偏移（`0 0`）的纯外发光形式，这种处理确保了光晕从元素边缘向四周均匀扩散，更接近自然界中气体发光的物理特征。开发者在应用这些效果时，建议配合 `transition: box-shadow 0.3s ease-out` 实现辉光的淡入淡出，避免发光状态的突兀切换。对于需要呼吸动效的场景，可结合 CSS `@keyframes` 在辉光的模糊半径和透明度之间循环过渡，模拟极光自然呼吸的视觉节奏。具体的动画参数可参考 `gold-shine` 关键帧动画（3s 周期内 `box-shadow` 的模糊半径在 24px 与 48px 之间呼吸变化），该动画已在全局 CSS 中预定义，开发者通过添加 `anim-gold-shine` 类名即可直接调用。此外，应始终关注发光效果对界面可读性的潜在影响——当辉光应用于文字元素或紧邻文字区域时，需确保发光层不会与文字色产生对比度冲突，必要时可通过 `text-shadow` 为文字添加同色系微弱描边以维持可读性。

## 4.4 卡片与按钮阴影

### 4.4.1 三级卡片阴影与两种按钮发光

卡片组件作为中萨交易所界面中承载信息的核心容器，其阴影系统需要同时满足"深度表达"与"主题氛围营造"两个目标。为此，卡片阴影采用三级递进结构：标准卡片提供微妙的立体基底，悬浮卡片增强交互反馈，极光玻璃卡片则通过三层复合阴影实现 Glassmorphism 2.0 的通透视觉效果。按钮作为引导用户行动的核心交互元素，其辉光效果以钻石青和皇家金为光源色，分别传达科技感与尊贵感。

| 名称 | 类型 | CSS `box-shadow` 值 | 层级/光源 | 使用场景 |
|------|------|-------------------|----------|---------|
| `card` | 卡片 | `0 1px 3px rgba(0, 0, 0, 0.06), 0 1px 2px rgba(0, 0, 0, 0.04)` | 单层微提升 | 默认状态的信息卡片、数据面板、列表容器 |
| `card-hover` | 卡片 | `0 4px 12px rgba(0, 0, 0, 0.08)` | 中度悬浮深度 | 卡片 Hover 交互态、可点击卡片的反馈态 |
| `aurora-card` | 卡片 | `0 8px 32px rgba(15, 27, 61, 0.4), 0 0 0 1px rgba(148, 163, 184, 0.10), 0 0 48px rgba(56, 189, 248, 0.08)` | 三层复合：深度+边框+辉光 | 极光玻璃卡片、首页核心功能卡、VIP 展示卡 |
| `cta-glow` | 按钮 | `0 4px 24px rgba(56, 189, 248, 0.35), 0 0 48px rgba(56, 189, 248, 0.15)` | `#38BDF8` 钻石青 | 蓝紫 CTA 行动按钮（注册、订阅、购买等） |
| `gold-glow` | 按钮 | `0 4px 24px rgba(240, 185, 11, 0.45), 0 0 48px rgba(240, 185, 11, 0.20)` | `#F0B90B` 皇家金 | 金色主按钮（充币、VIP 入口、财富操作等） |

三级卡片阴影的设计呈现出明显的复杂度递进。标准卡片阴影（`card`）采用双层微阴影策略，1px–3px 的垂直偏移配合 0.04–0.06 的极低透明度，在深色背景上营造出几乎不可见但足以区分层级的"空气感"深度，适用于静态展示的信息密度较高的场景。悬浮卡片阴影（`card-hover`）将模糊半径提升至 `12px`，透明度加倍至 `0.08`，在 Hover 触发时提供明确的抬升感知，配合 `transform: translateY(-2px)` 的微位移动画，可构建完整的悬浮交互体验。

极光玻璃卡片阴影（`aurora-card`）是最复杂的三层复合结构，每一层承担不同的视觉职能：第一层 `0 8px 32px rgba(15, 27, 61, 0.4)` 以主背景色（`#0F1B3D` 的 rgba 形式）构建环境深度阴影，其较高的透明度（0.4）确保玻璃卡片在极光渐变背景上拥有清晰的轮廓边界；第二层 `0 0 0 1px rgba(148, 163, 184, 0.10)` 利用零模糊、1px 扩展的技巧绘制一道极细腻的玻璃边框，模拟物理玻璃的边缘折射效果；第三层 `0 0 48px rgba(56, 189, 248, 0.08)` 注入极弱的青色辉光，使卡片与整体的极光主题光环境产生呼应。这三层阴影必须与 `backdrop-filter: blur(24px) saturate(180%)` 和渐变背景（`linear-gradient(180deg, rgba(26,36,86,0.55), rgba(21,34,74,0.70))`）配合使用，才能完整呈现 Glassmorphism 2.0 的视觉效果。

两种按钮辉光在结构上保持一致：均为双层阴影，第一层提供 4px 垂直偏移和 24px 模糊半径的"落地感"光晕，第二层提供零偏移、48px 大半径的"环境扩散"光晕。这一结构设计使按钮看起来如同放置在深色表面上、底部有方向性光源照射的发光体，而非凭空漂浮的光点。CTA 辉光的 alpha 值为 0.35/0.15，整体发光强度相对克制，以确保在蓝紫渐变按钮（`linear-gradient(135deg, #1677FF → #7C3AED)`）上不会产生过曝；金辉光则将 alpha 提升至 0.45/0.20，配合金色渐变按钮（`linear-gradient(135deg, #FCD535 → #F0B90B)`），形成更明亮、更温暖的视觉焦点。

在开发实现中，卡片阴影与按钮辉光均可与 CSS 过渡属性配合，实现交互状态下的平滑切换。卡片从默认态到 Hover 态时阴影从 `card` 级别平滑过渡到 `card-hover` 级别；按钮辉光通常在 Hover 状态下激活，通过 `transition: box-shadow 0.2s ease-out` 实现淡入效果。此外，按钮辉光可与极光辉光效果叠加使用——在需要极强视觉牵引的场景（如首页首屏的核心行动点），可通过同时应用按钮辉光和极光辉光实现"双层发光"效果，但需注意 alpha 值的总和不应超过 0.60，以避免光晕过度扩散影响相邻内容的可读性。建议通过 CSS 变量（Custom Properties）封装各辉光参数，以便在不同主题模式下动态调整发光强度。对于卡片与按钮的协同设计，应确保同一视口内的高光元素不超过两个——当页面已存在极光玻璃卡片（带 `aurora-card` 辉光）时，其内部的 CTA 按钮应降级使用标准渐变背景，避免多重辉光在局部区域形成视觉噪声。
