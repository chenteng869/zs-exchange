# 7. 布局、响应式、动效与H5规范

## 7.1 断点系统

### 7.1.1 7级断点完整规格

中萨交易所采用 Tailwind CSS 默认断点体系，并针对 H5 移动端场景增设自定义容器约束，形成覆盖从大屏手机到桌面显示器的 7 级响应式断点矩阵。断点系统以 `min-width` 为触发条件，确保样式从小屏向大屏逐级叠加，避免重复定义。H5 移动端采用独立的 `max-width: 480px` 容器策略，在 PC 端浏览器中以居中模拟器形态呈现。

| 断点名 | 触发条件 | 目标设备 | 布局策略 |
|--------|---------|---------|---------|
| **xs** | `≥480px` | 大屏手机 / 横屏手机 | 单列布局，字号微调 |
| **默认** | `≥640px`（Tailwind `sm`） | 平板竖屏 | 容器开始展宽，间距增加 |
| **sm** | `≥640px` | 平板横屏 / 小平板 | 双列网格可用 |
| **md** | `≥768px` | 标准平板 | 多列布局，侧边栏可展开 |
| **lg** | `≥1024px` | 笔记本 / 小桌面 | 完整导航，三列布局 |
| **xl** | `≥1280px` | 桌面显示器 | 最大内容宽度，全功能展示 |
| **H5模拟器** | `max-width: 480px` | H5 移动端容器 | PC 上居中模拟手机外壳 |

断点设计遵循"移动优先"（Mobile First）原则。基础样式默认服务于 375px–480px 区间的手机视口，随着屏幕宽度增大逐步解锁更复杂的布局能力。在 Tailwind 配置中，标准断点 `sm`（640px）、`md`（768px）、`lg`（1024px）、`xl`（1280px）保持框架默认值不变，开发时通过 `md:`、`lg:` 等前缀编写响应式类名。

H5 模拟器断点是本系统的特殊设计。当用户在 PC 端访问 H5 页面时，内容区域被严格限制在 `480px` 最大宽度内，通过 `margin: 0 auto` 实现水平居中，外围辅以阴影和圆角模拟手机外壳效果。该容器内采用独立的 H5 样式集，确保移动端交互在桌面浏览器中的视觉一致性。实际运行在真机环境时，`max-width` 约束自然失效，页面填满屏幕宽度。

## 7.2 H5 移动端布局框架

### 7.2.1 三层结构：fixed 顶栏 + main 内容滚动区 + fixed 底部 Tab 导航

H5 移动端采用经典的三层固定布局架构，顶栏与底部导航栏始终悬浮于视口边缘，中间内容区独立滚动。这种结构确保导航入口随时可达，同时最大化屏幕纵向空间的利用率。

顶栏（Header）以 `position: fixed` 附着于视口顶部，`z-index` 设为 100，高度约 60px，背景采用 `rgba(15, 27, 61, 0.92)` 叠加 `backdrop-filter: blur(20px)` 实现毛玻璃效果，既保证内容滚动时的视觉通透性，又提供足够的背景遮挡力使文字可读。顶栏内部承载 Logo 区域、应用名称与通知铃铛图标。

主内容区（Main）为页面唯一可滚动区域，通过 `padding-top: 60px` 避开顶栏占用空间，`padding-bottom: 72px` 为底部导航预留安全距离。内容区内再设 `padding: 12px` 作为标准页面边距，所有卡片、列表、表单元素在此区域内纵向排列。

底部 Tab 导航同样以 `position: fixed` 附着于视口底部，`z-index` 为 100，高度约 60px（含安全区域适配），采用 5 列 Grid 布局承载首页、行情、交易、资产、我的五大入口。导航栏背景使用 `rgba(15, 27, 61, 0.95)` 配合 `blur(24px)` 毛玻璃处理，顶部以 `1px solid rgba(148, 163, 184, 0.15)` 的细线边框与内容区形成视觉分隔。

### 7.2.2 容器最大宽度 480px 居中展示

H5 容器的核心约束参数为 `max-width: 480px` 与 `margin: 0 auto`。在 PC 端浏览器中，这一约束使内容区域呈现为竖直放置的手机形态，外围通过 `box-shadow` 投射出柔和的环境阴影，模拟实体设备的悬浮感。容器内部的所有尺寸、间距、字号均针对移动端触控操作优化，按钮高度不低于 40px，可点击区域满足 44×44px 的人机工程学最小触控面积要求。

容器背景色设为皇家深蓝 `#0F1B3D`，顶部叠加多层极光渐变网格（由 `aurora-bg` 类名控制），在固定顶栏与滚动内容区之间形成连贯的深邃空间感。文字颜色默认为 `#F8FAFC`（高对比雪白），确保在深色背景上的可读性。字体栈优先使用系统字体：`Inter` → `PingFang SC` → `Microsoft YaHei` → `-apple-system` → `BlinkMacSystemFont`，在 iOS 与 Android 平台上均能获得原生的排版质感。

### 7.2.3 安全区域适配：env(safe-area-inset-top/bottom)

为适配 iPhone X 及后续机型、各品牌刘海屏 / 挖孔屏 / 灵动岛设备的异形屏幕特征，H5 布局通过 CSS `env()` 函数引入安全区域变量。顶栏的上内边距设为 `max(12px, env(safe-area-inset-top))`，在普通屏幕上保持 12px 的标准间距，在带有刘海的屏幕上自动扩展至系统安全区域高度，避免 Logo 或文字被刘海遮挡。

底部导航栏的下内边距采用 `max(env(safe-area-inset-bottom), 8px)` 的声明，确保导航入口不会落入 Home 指示条（Home Indicator）区域。主内容区的底部内边距在安全区域基础上叠加导航栏高度，完整表达式为 `calc(72px + env(safe-area-inset-bottom))`。以上安全区域适配逻辑通过 `H5Layout.tsx` 壳组件统一封装，各业务页面无需重复处理。

## 7.3 底部导航 Tab 规格

### 7.3.1 5列 Grid 布局：首页 / 行情 / 交易 / 资产 / 我的

底部导航栏采用 CSS Grid 的 `repeat(5, 1fr)` 布局，五个 Tab 项等分容器宽度。每个 Tab 项内部采用 Flex 纵向排列，图标在上、文字在下，整体垂直居中。导航栏的 `padding` 上下分别为 8px 和 `env(safe-area-inset-bottom)` 的较大值，确保触控区域充裕且不与系统手势冲突。

| 属性 | 规格值 | 说明 |
|------|--------|------|
| **布局模式** | `grid-template-columns: repeat(5, 1fr)` | 5 列等分 |
| **导航栏内边距** | `8px 0`（上/下），底部叠加安全区域 | 底部触控区充足 |
| **图标尺寸** | `20px`，`line-height: 1` | SVG 线性图标，视觉居中 |
| **文字尺寸** | `11px` | 小字号标签，紧凑可读 |
| **激活文字色** | `#FCD535`（金亮） | 高辨识度，与品牌金色一致 |
| **非激活文字色** | `#7B89B8`（雾灰） | 低调后退，不分散注意力 |
| **激活字重** | `600`（Semibold） | 视觉强调 |
| **非激活字重** | `400`（Normal） | 标准字重 |
| **激活指示条** | `2px` 高，渐变（金→青），圆角 `2px` | 顶部微型指示器 |
| **指示条阴影** | `0 0 8px rgba(56, 189, 248, 0.6)` | 青色光晕点缀 |

Tab 切换交互包含两个并行动效：激活指示条以 `250ms ease-out` 的时长从原位置横向滑动至新激活项，同时颜色渐变过渡；文字颜色在 `150ms` 内从雾灰切换为金亮，字重同步由 400 过渡至 600。非激活项的反向过渡同样遵循 `150ms` 的快速响应规格，确保视觉反馈即时清晰。

### 7.3.2 激活指示条与视觉反馈

激活指示条高度仅 2px，采用 `linear-gradient(90deg, #FCD535, #38BDF8)` 的金→青渐变填充，两端设 `2px` 圆角使其呈现胶囊状微条形态。指示条下方投射 `0 0 8px rgba(56, 189, 248, 0.6)` 的青色光晕，在深色导航栏背景上形成微妙的霓虹发光效果。该指示条通过绝对定位与 `transform: translateX()` 实现位置切换，配合 `transition: transform 250ms ease-out` 获得顺滑的滑位动画。

每个 Tab 项的图标采用 SVG 线性风格，描边宽度统一为 `1.5px`，激活状态下图标描边色跟随文字同步变为 `#FCD535`，非激活状态下为 `#7B89B8`。图标与文字间距保持 `4px`，整体触控区域覆盖整个 Grid 单元格，满足 WCAG 2.1 对触控目标最小尺寸的要求。

## 7.5 动效体系

### 7.5.1 5级动画时长

中萨交易所的动效体系将交互响应划分为 5 个时长等级，从快速的微交互到缓慢的环境动画，覆盖所有 UI 状态转换场景。

| 时长等级 | 值 | 应用场景 | 响应目标 |
|----------|-----|---------|---------|
| **快速（fast）** | `150ms` | hover 状态切换、focus 反馈、文字颜色过渡 | 即时微交互，用户无感知延迟 |
| **正常（normal）** | `250ms` | 展开/收起、淡入淡出、Tab 切换滑位 | 清晰可辨的过渡，保持流畅感 |
| **慢速（slow）** | `400ms` | 页面切换、卡片悬浮效果、复杂位移动画 | 优雅从容的大型元素运动 |
| **极慢（slower）** | `600ms` | 大型转场、模态框出现、页面级导航 | 戏剧性强调，引导注意力 |
| **极光呼吸（aurora）** | `12–14s` | 背景光斑缩放/位移/透明度循环 | 环境氛围，无感知重复 |

快速级别的 `150ms` 用于所有需要即时反馈的微交互场景，如按钮 hover 时的背景色变化、输入框 focus 时的边框颜色切换、Tab 文字颜色的渐变过渡。此时长等级下用户几乎无法察觉延迟，但视觉上仍能感知到平滑过渡而非突兀跳变。正常级别的 `250ms` 服务于中等复杂度的动画，如手风琴展开收起、Tooltip 淡入淡出、底部导航指示条的横向滑位，该时长在"足够快以不造成等待感"与"足够慢以看清运动轨迹"之间取得平衡。

慢速级别的 `400ms` 应用于页面级过渡与卡片悬浮效果。当用户 hover 于玻璃拟态卡片时，卡片以 `400ms ease-out` 完成上浮 `2px`、边框变为金色、阴影加深的三重过渡。极慢级别的 `600ms` 保留给弹窗出现、路由切换等需要用户注意力重新定向的场景。极光呼吸级别专属于环境氛围动画，背景光斑以 `12–14s` 为周期进行缓慢呼吸，该时长远超人类注意力的刷新频率，用户在意识层面不会察觉循环的存在，仅感受到背景"活着"的有机质感。

### 7.5.2 5种缓动函数完整值

缓动函数（Easing Function）控制动画在时间轴上的速度分布，中萨交易所的动效体系定义了 5 种经过调优的贝塞尔曲线，分别服务于不同类型的物理隐喻。

| 缓动名称 | Cubic Bezier 值 | 速度特征 | 适用场景 |
|----------|----------------|---------|---------|
| **default** | `cubic-bezier(0.4, 0, 0.2, 1)` | 缓入缓出，前后平滑 | 通用过渡、颜色变化、透明度 |
| **ease-in** | `cubic-bezier(0.4, 0, 1, 1)` | 缓慢启动，快速结束 | 元素退出、收起、消失 |
| **ease-out** | `cubic-bezier(0, 0, 0.2, 1)` | 快速启动，缓慢着陆 | 元素进入、展开、出现 |
| **bounce** | `cubic-bezier(0.68, -0.55, 0.265, 1.55)` | 超出回弹，弹性收尾 | 需要强调反馈的交互 |
| **aurora** | `cubic-bezier(0.45, 0, 0.55, 1)` | 正弦往返，对称平滑 | 极光呼吸、光斑漂移专用 |

default 缓动曲线采用近似 `ease-in-out` 的对称形态，适用于绝大多数不涉及方向性隐喻的过渡场景。ease-in 曲线在动画起始段保持低速，随后急剧加速，模拟物体被"吸入"或"收起"的物理感受，适用于弹窗关闭、下拉菜单收起、页面退出等方向向内的交互。ease-out 曲线恰好相反，起始段全速前进，末端柔和减速着陆，模拟物体从外部进入视口时的惯性表现，适用于页面进入、弹窗出现、下拉菜单展开等方向向外的交互。

bounce 缓动曲线通过控制点超出 `[0, 1]` 的坐标范围产生弹性回弹效果，动画结束时会轻微超出目标值再回调到位，赋予界面物理弹性感。该缓动适用于需要强调操作反馈的场景，如按钮点击确认、开关切换、重要通知出现。aurora 缓动曲线是专为环境氛围动画设计的近似正弦曲线，其对称形态使往返运动在视觉上等速平滑，避免光斑漂移中出现人为加速或减速的"机械感"。

### 7.5.3 4种极光核心动画参数

极光核心动画是 Aurora Premium v7 主题的标志性视觉特征，通过 4 个独立的 CSS 动画关键帧控制背景光斑的有机运动，各动画以不同的周期和路径叠加运行，形成不可预测的随机感。

**aurora-flow（光斑呼吸）** 周期为 `12–14s`，光斑执行三重属性的同步循环：缩放因子从 `0.7` 膨胀至 `1.15` 再收缩至 `0.95`，水平位移从原点偏移 `+30px` 再回退至 `-30px`，垂直位移从 `0` 偏移 `-20px` 再回调 `+20px`，透明度同步在 `0.3` 与 `0.6` 之间呼吸。三重属性的相位差通过关键帧的 `0% / 33% / 66% / 100%` 节点精确错开，使运动轨迹呈现类流体般的自然变化。

**aurora-drift（慢漂移）** 周期 `18s`，控制光斑进行更大范围的缓动漫游。缩放从 `0.6` 扩展至 `1.2` 最终收敛至 `1.0`，位移路径从原点漂移至 `(-50px, +40px)` 后返回起点。该动画的速度曲线采用 aurora 专用缓动，往返运动在视觉上等速平滑，与 aurora-flow 叠加后产生层次丰富的深度错觉。

**aurora-rise（上升浮动）** 周期 `22s`，是四种动画中最缓慢的环境运动。光斑在垂直方向上进行 `0 → ±20px` 的上下浮动，伴随微小的缩放变化（`0.98` 至 `1.02`）。该动画模拟大气中光粒子受热气流影响而缓慢上升的自然现象，为静态背景注入微妙的生机。

**gold-shine（金色脉冲）** 周期 `3s`，是唯一非环境性的功能性动画，专用于通知红点、VIP 标识等需要吸引注意力的元素。动画通过交替调整 `box-shadow` 的扩散半径在 `24px` 与 `48px` 之间呼吸，配合透明度从 `0.6` 至 `1.0` 的脉冲，形成类似心跳的节奏感。该动画在 `prefers-reduced-motion` 环境下被强制关闭。

## 7.6 过渡动画

### 7.6.1 六种过渡效果

系统级过渡动画覆盖六大交互场景，每种场景配有精确的动效参数组合，确保交互反馈的一致性与可预测性。

**卡片 Hover 过渡** 触发于用户手指/鼠标悬停于玻璃拟态卡片时。动画组合为：垂直上浮 `translateY(-2px)`、边框颜色过渡至 `rgba(240, 185, 11, 0.40)`（金色）、背景透明度提升 `5%–10%`、阴影从标准卡片阴影扩展为悬浮阴影。总时长 `300–400ms`，缓动函数 `ease-out`，所有属性通过 `transition: all 400ms ease-out` 统一声明。

**按钮 Hover 过渡** 采用更克制的微动效：垂直上浮 `translateY(-1px)`、阴影扩散增强。时长 `200–300ms`，缓动 `ease-out`。对于极光特色按钮（`.btn-aurora-cta`、`.btn-aurora-gold`），hover 时额外触发一道闪光扫过效果（shimmer），以 `45deg` 角度从左上至右下穿越按钮表面，时长 `600ms`。

**页面进入（fadeInUp）** 是路由切换时的标准入场动效。页面内容初始状态为 `opacity: 0` 配合 `translateY(10px)` 的轻微下沉，在 `400ms ease-out` 过渡后达到 `opacity: 1` 与 `translateY(0)` 的最终状态。该动效作用于主内容区的顶层容器，确保每次页面跳转都伴随内容向上浮入的视觉引导。

**弹窗出现（scaleIn）** 用于模态框、底部弹出层（Bottom Sheet）、Toast 通知等覆盖层元素。动画从 `scale(0.95)` 与 `opacity: 0` 的缩小透明状态开始，在 `300ms ease-out` 过渡中扩展至 `scale(1)` 与 `opacity: 1`。弹窗背景遮罩以 `200ms` 的时长从透明过渡至 `rgba(0, 0, 0, 0.5)`，略早于弹窗主体完成，形成"空间先暗、内容后亮"的层次节奏。

**Tab 切换** 已在 7.3.1 节详述，动画组合为指示条横向滑位（`transform` + `250ms ease-out`）与文字颜色渐变（`150ms`）。

**数字滚动** 专用于资产总览等展示大额数字的场景。当数字从初始值更新至目标值时，数字以 `500–800ms` 的时长从 `0` 滚动至最终数值，滚动过程中使用等宽字体 `DIN` 确保数字宽度不发生抖动。该动画在移动端资产页面为用户带来"财富增长"的直观视觉满足感。

## 7.7 无障碍支持

### 7.7.1 prefers-reduced-motion 媒体查询配置

中萨交易所严格遵循 WCAG 2.1 关于动效可访问性的指导原则，通过 CSS `prefers-reduced-motion` 媒体查询尊重用户对减少动效的系统性偏好。当用户在操作系统或浏览器中开启"减弱动态效果"选项时，所有极光环境动画、金色脉冲动画、卡片悬浮动效均被关闭，界面切换退化为瞬时状态变化或极简淡入淡出。

```css
@media (prefers-reduced-motion: reduce) {
  .anim-aurora-flow,
  .anim-aurora-drift,
  .anim-aurora-rise,
  .anim-gold-shine {
    animation: none !important;
  }

  .glass-aurora-hover:hover {
    transform: none !important;
    transition: opacity 150ms ease !important;
  }

  .btn-aurora-cta:hover::before,
  .btn-aurora-gold:hover::before {
    display: none !important;
  }

  * {
    transition-duration: 0.01ms !important;
    animation-duration: 0.01ms !important;
  }
}
```

上述配置分为三个层级。第一层级针对四个极光核心动画类名（`.anim-aurora-flow`、`.anim-aurora-drift`、`.anim-aurora-rise`、`.anim-gold-shine`）直接禁用 `animation` 属性，背景光斑停止运动，界面回归静态。第二层级保留玻璃卡片的 hover 状态变化，但移除 `transform` 位移动画，仅保留 `150ms` 的透明度过渡作为最低限度的状态反馈。第三层级通过通配符 `*` 将所有过渡与动画时长压缩至 `0.01ms`，近乎瞬时完成，确保未被显式命名的动画不会绕过媒体查询的约束。该媒体查询声明放置于全局 CSS 文件末尾，以最高优先级覆盖所有组件级动画定义。
