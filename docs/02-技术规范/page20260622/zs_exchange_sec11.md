# 11. User PC端开发清单

## 11.1 User PC端概述

### 11.1.1 在现有代码库基础上增强/15页总量/9增强+6新增

User PC端面向交易所终端用户，依托现有的 `src/app/user/*` 代码库进行增量开发。与从零构建的管理后台和H5移动端不同，User PC端的核心工作模式为**增强型开发**：在已具备完整交易功能的既有页面之上，注入量化交易模块所需的新组件、新数据区块和新交互逻辑。

全量页面共 **15 个**，按开发性质分为两类：

- **核心页面增强（9 页）**：编号 U01–U09，对应现有 9 个用户端核心页面（Dashboard、三种交易界面、资产页、充值页、订单管理、个人中心、IDO 申购）。开发方式为"页面内插入"，即在既有布局的指定位置新增量化相关区块或扩展字段，不涉及路由变更。复杂度以"中"和"低"为主，单个页面的工作量相当于在成熟组件树中挂载新子树。

- **新增独立页面（6 页）**：编号 U10–U15，为量化模块专设的独立路由页面，采用 PC 端宽屏布局重新组织 H5 已验证的功能（AI 策略中心、跟单管理、策略市场、大师观点、报告库、收益分析）。开发方式为"全新页面搭建"，需定义新的路由、布局模板和数据流。复杂度以"高"和"中"为主，图表密集页面（U10、U15）的工作量显著高于普通列表页。

从实施阶段看，9 个增强页（U01–U09）全部归入 Phase 2（核心能力完善阶段），与 H5 移动端的核心功能同步交付；6 个新增独立页（U10–U15）全部归入 Phase 3（生态壁垒构建阶段），作为桌面端的差异化体验在最后阶段集中开发。增强页先行、新增页随后的排期策略，可确保在既有用户界面中先完成量化能力的渗透，再推出 PC 专属的完整功能入口。

## 11.2 核心页面增强（9 页）

核心页面增强遵循统一的注入式开发范式：每个目标页面均已在 `src/app/user/` 下具备完整的业务逻辑和样式体系，量化模块的增强工作聚焦于"在指定 DOM 位置插入新组件"，而非重构既有代码。以下为 9 个增强页的详细规格。

### 11.2.1 U01 Dashboard增强：量化策略绩效卡片

U01 在现有 Dashboard 首页（`/user/dashboard`）的顶部区域插入「量化策略绩效」卡片区块。该区块为横向排列的三卡片布局，单卡片宽度约占主内容区的 1/3（在 1440px 视口下约 380px）。

- **AI 信号提醒卡片**：展示最新 3 条未读 AI 交易信号，每条信号包含币种代码、方向标识（做多/做空/观望）、信号强度进度条（0–100%）和生成时间戳。卡片右上角设"查看全部"链接，跳转至 U10 AI 策略中心。
- **跟单收益卡片**：以数字滚动动画展示累计跟单收益（USDT 计价），下方附当日收益波动折线（Sparkline，7 日数据点）。收益为正时数字颜色使用 `#10b981`（绿色系），为负时使用 `#ef4444`（红色系）。
- **策略排名卡片**：展示用户当前订阅策略的 7 日收益排名 Top 3，每项显示策略名称、收益率百分比和排名变化箭头（↑/↓/–）。

该区块采用服务端渲染（SSR）方式加载初始数据，随后通过 WebSocket（S06 服务）推送增量更新。复杂度评定为"中"，主要工作量在于三卡片的数据聚合接口设计与 Sparkline 微图表的实现。

### 11.2.2 U02–U04 三交易增强：AI信号提示条

U02（现货交易 `/user/trading/spot`）、U03（合约交易 `/user/trading/futures`）、U04（杠杆交易 `/user/trading/margin`）采用完全一致的增强模式：在交易界面 K 线图上方的工具栏区域插入一条 AI 信号提示条（Signal Banner）。

提示条高度固定为 40px，背景色使用半透明主题色 `rgba(59, 130, 246, 0.08)`，左侧边框 4px 颜色根据信号方向变化：看涨 `#10b981`、看跌 `#ef4444`、中性 `#6b7280`。提示条内部从左至右依次展示：当前交易对代码（如 BTC/USDT）、AI 综合评分（0–100 分数字 + 环形进度图）、12 位大师观点摘要（"8 看涨 / 3 看跌 / 1 中性"格式）以及"查看分析"按钮（点击展开侧边抽屉面板，展示完整 AI 分析数据）。

三个页面的提示条共享同一套组件实现（`<AiSignalBanner />`），通过 `tradingType` prop 区分现货/合约/杠杆语境下的文案差异。合约和杠杆版本额外展示"建议杠杆倍数"字段（仅当 AI 信号强度 ≥ 70% 时显示）。复杂度评定为"低"，属于纯展示型组件，无复杂交互。

### 11.2.3 U05 资产页增强：跟单持仓分组与AI评分标签

U05 在现有资产页（`/user/assets`）的持仓列表中引入两项结构性变更。

第一项为**跟单持仓分组**：在原有资产列表（按币种字母序排列）的顶部新增一个独立的"跟单持仓"分组，以折叠面板（Collapse Panel）形式展示。该分组内列出用户当前通过量化策略自动持有的全部仓位，每项展示币种代码、持仓数量、成本均价、当前盈亏和关联策略名称。分组头部显示跟单持仓总估值（USDT）。

第二项为**AI 评分标签**：在所有持仓项（含手动持仓和跟单持仓）的币种代码右侧追加 AI 评分标签组件。标签为圆角矩形（border-radius: 4px），尺寸约 32px × 18px，背景色按评分区间分级：90–100 分为 `#059669`（深绿）、70–89 分为 `#10b981`（绿）、50–69 分为 `#f59e0b`（黄）、0–49 分为 `#ef4444`（红）。标签内显示评分数值，鼠标悬停时通过 Tooltip 展示评分维度的细分（增长潜力 / 风控安全 / 经济模型 / 社区活跃度 / AIOPC 赋能）。

复杂度评定为"中"，工作量主要在于列表数据结构的改造（引入分组逻辑）和评分标签的批量渲染性能优化。

### 11.2.4 U06 充值页增强：策略订阅费支付入口

U06 在现有充值页（`/user/wallet/deposit`）的支付用途选择区域新增"策略订阅费"选项。现有充值流程的支付用途通常包含"交易资金""合约保证金"等分类，本次在用途下拉菜单（或单选按钮组）中追加"策略订阅费"项。

当用户选择该用途时，下方动态展开关联策略选择器（下拉列表展示用户已订阅/待付费的策略名称和对应金额），并将充值金额输入框的默认值自动填充为所选策略的订阅费用。充值成功后，资金直接进入策略订阅托管账户（而非普通交易账户），并触发后端订阅状态更新。复杂度评定为"低"，属于表单字段扩展，无新增页面路由。

### 11.2.5 U07 订单管理增强：来源列与按策略筛选

U07 在现有订单管理页（`/user/trading/orders`）的订单表格中新增两个功能点。

- **来源列（Origin Column）**：在表格头部新增"来源"列，位于"订单类型"列之后。该列以标签（Tag）形式标记每条订单的触发来源：`手动`（蓝色 `#3b82f6` 标签）、`量化跟单`（紫色 `#8b5cf6` 标签）、`API`（灰色 `#6b7280` 标签）。量化跟单类订单的标签支持点击，点击后弹出 Tooltip 展示关联的策略名称和信号 ID。
- **按策略筛选**：在表格上方的筛选栏新增"策略"筛选条件（下拉选择器），选项列表动态拉取用户当前订阅的全部策略名称。选择某一策略后，表格仅展示由该策略触发的跟单订单。筛选条件与现有的时间范围、币种、订单状态筛选器支持联合使用。

复杂度评定为"中"，主要工作量在于表格列配置的扩展和筛选逻辑的组合查询实现。

### 11.2.6 U08 个人中心增强：VIP等级与量化权限

U08 在现有个人中心页（`/user/profile`）新增两个信息展示卡片。

- **VIP 等级与量化权限卡片**：位于用户信息卡片下方，展示当前 VIP 等级（Level 1–5）、等级对应的量化功能权限清单（以勾选列表形式）。Level 3 以下用户的部分高阶功能（如 API 跟单、自定义策略参数）以锁定图标标识，点击锁定项弹出 VIP 升级引导浮层。
- **策略订阅状态卡片**：展示当前生效的策略订阅列表，每项显示策略名称、订阅周期（月/年）、到期时间倒计时、续费按钮。已过期策略以灰色背景高亮显示，并提供"重新订阅"快捷入口。

复杂度评定为"中"，主要工作量在于权限矩阵的 UI 映射和订阅状态的时间计算逻辑。

### 11.2.7 U09 IDO申购增强：AI评分标签与代币评分入口

U09 在现有 IDO 申购页（`/user/ido`）的项目卡片上追加 AI 评分相关元素。

- **AI 评分标签**：在每个 IDO 项目卡片的右上角叠加 AI 评分角标（Badge），尺寸 48px × 20px，圆角 `border-radius: 0 8px 0 8px`，背景色与评分区间对应（同 U05 的评分色彩体系）。评分 ≥ 80 分的项目角标附带"推荐"文案。
- **代币评分快速查看入口**：项目卡片的操作区新增"AI 评估报告"按钮，点击后以侧边抽屉（Drawer）形式加载该代币的完整 AI 评分报告（复用 H5 端的评分报告组件，进行 PC 宽屏适配）。抽屉宽度占视口 60%（最小 720px），内部展示五维雷达图、6 大分析师结论和 12 位大师观点摘要。

复杂度评定为"低"，评分角标为纯展示组件，报告抽屉为跨端组件复用。以下为 9 个核心增强页面的汇总表。

## 11.3 新增独立页面（6 页）

新增独立页面构成 PC 端量化功能的完整入口矩阵。6 个页面全部挂载于 `/user/quant/*` 路由空间下，采用统一的 PC 宽屏布局模板：左侧固定导航栏（200px，量化模块专属菜单）+ 右侧主内容区（自适应宽度，最大 1440px）。各页面的布局模板和主题配置继承自 S08（主题配置系统）和 S09（图表组件库封装），确保与管理后台 Web 端在视觉体系上保持一致。

### 11.3.1 U10 PC端AI策略中心

U10（`/user/quant/center`）为 PC 端量化功能的聚合入口，定位为 H05（H5 AI 策略大脑）的桌面宽屏版本。页面采用三栏布局：左侧 280px 为策略分类导航（全部策略 / 信号推荐 / 跟单管理 / 大师观点 / 我的订阅），中间为主内容区（信号卡片列表或策略表格），右侧 320px 为侧边信息面板（市场行情快讯 + 持仓概览迷你卡）。

主内容区的信号卡片采用高信息密度设计：单卡片高度 160px，包含评分环形图（直径 56px）、币种代码、AI 方向标签、5 维迷你柱状图、12 位大师观点投票计数以及"一键跟单"按钮。卡片网格在 1440px 视口下每行展示 2 张卡片，1920px 视口下每行 3 张。页面顶部设全局筛选栏，支持按市场（现货/合约）、方向（看涨/看跌/中性）、评分区间（≥90 / 70–89 / 50–69 / <50）和大师偏好进行复合筛选。复杂度评定为"高"，主要工作量在于三栏响应式布局、信号卡片的密集信息排布以及复合筛选的联调。

### 11.3.2 U11 PC端跟单管理

U11（`/user/quant/copy-trades`）为桌面端跟单数据分析中心，充分利用 PC 屏幕的横向空间展示多维度数据。页面顶部为 4 张 KPI 卡片（进行中跟单数 / 累计收益 / 今日收益 / 胜率百分比），中部为主数据表格（跟单记录表），底部为收益趋势折线图。

跟单记录表格列数约 12 列，包含：跟单 ID、关联策略、交易对、方向、开仓价、当前价、盈亏金额、盈亏比例、跟单时间、持仓时长、状态（进行中 / 已平仓 / 止损触发）、操作（查看详情 / 平仓）。表格支持按全部列排序和复合筛选，每页默认 20 行。收益趋势图使用 ECharts 实现，展示近 30 日的累计跟单收益曲线，支持鼠标悬停查看单日明细。复杂度评定为"高"，主要工作量在于高密度数据表格的性能优化和收益曲线的实时更新。

### 11.3.3 U12 PC端策略市场浏览

U12（`/user/quant/marketplace`）为策略订阅服务的 PC 端展示页，采用卡片网格布局展示平台上架的全部量化策略产品。

策略卡片尺寸为 360px × 280px，包含以下元素：策略师头像（40px 圆形）、策略名称、策略类型标签（趋势跟踪 / 均值回归 / 套利 / 网格）、历史收益曲线缩略图（Sparkline，30 日数据）、关键指标行（年化收益率 / 最大回撤 / 夏普比率）、订阅价格（USDT/月）和"立即订阅"按钮。卡片网格在 1440px 视口下每行 3 张，1920px 下每行 4 张。页面左侧设筛选侧边栏（按策略类型 / 收益率区间 / 风险等级 / 价格区间筛选），顶部设排序控件（按收益 / 订阅人数 / 上架时间）。复杂度评定为"高"，主要工作量在于策略卡片的信息密度控制和筛选排序的联调。

### 11.3.4 U13 PC端大师观点墙

U13（`/user/quant/masters`）为 12 位投资大师 AI 智能体观点的 PC 端展示页。与 H5 版本的单大师纵向滚动流不同，PC 版采用宽屏多列布局，在 1440px 视口下同时展示 3 位大师的观点面板。

每位大师的面板宽度约 400px，包含：大师头像（56px 圆形）+ 名称 + 投资风格标签、最近 5 条观点的时间线（每条观点含标的代码、方向、时间和摘要）、观点筛选控件（按市场 / 方向 / 时间范围）以及"关注"切换按钮。面板之间支持拖拽重新排列顺序（通过 `react-beautiful-dnd` 或类似库实现），用户可将最关注的大师固定在首列。页面顶部设全局大师对比工具，选择多位大师后可对比其对同一标的的观点分歧度。复杂度评定为"中"，拖拽排序和观点对比为主要的交互开发点。

### 11.3.5 U14 PC端我的报告库

U14（`/user/quant/reports`）为用户已购买或生成的 AI 分析报告的归档管理中心。页面采用文件库式布局：顶部为搜索栏 + 类型筛选（代币评分报告 / 策略回测报告 / 企业诊断报告 / IPO 评估报告），中部为报告卡片网格，底部为分页控件。

单张报告卡片尺寸 280px × 200px，包含：报告类型图标、报告标题（截断至 2 行）、生成日期、页数/文件大小、操作按钮组（预览 / 下载 PDF / 分享）。点击"预览"按钮后在页面中央打开 PDF 预览模态框（使用 `react-pdf` 实现首页缩略预览，支持翻页）。报告数据按生成时间倒序排列，支持文件夹式分类（按报告类型自动分组）。复杂度评定为"中"，PDF 预览模态框和文件下载为主要的开发点。

### 11.3.6 U15 PC端收益分析

# 12. 公共/系统组件清单

## 12.1 全局认证与通信

全局认证与通信类组件是支撑全部 120 个页面正常运行的基础设施。它们不直接面向终端用户展示 UI，但在每一次页面加载、每一次数据请求和每一条实时推送中发挥核心作用。本节涵盖 S01（JWT 认证中间件）和 S06（WebSocket 连接管理）两个系统级模块。

### 12.1.1 S01 JWT认证中间件

S01 负责交易所全端（管理后台 / H5 移动端 / User PC 端）的用户身份认证流程。中间件以高阶组件（HOC）形式封装，挂载于 React 应用的根组件外层，拦截所有需要认证的页面访问。

核心功能包含三个层面：**Token 校验**——在每个 API 请求的 `Authorization` Header 中携带 JWT Access Token，后端返回 401 时触发登录态失效处理；**过期刷新**——实现双 Token 机制（Access Token 有效期 15 分钟 + Refresh Token 有效期 7 天），Access Token 过期前 2 分钟自动调用 `/auth/refresh` 接口换取新 Token，该过程对用户无感知；**权限注入**——Token 解析后将用户角色（`role`）和量化权限位（`quant_permissions` 字段）写入 React Context，供各页面的权限控制逻辑读取（如 U08 的 VIP 等级展示和量化功能锁）。

中间件需处理边界场景：多标签页登录态同步（通过 `BroadcastChannel API` 或 `localStorage` 事件实现）、Refresh Token 过期后的统一登出跳转、以及网络异常时的重试机制（最多 3 次，间隔 1 秒）。复杂度评定为"低"，属于成熟的认证模式实现。

### 12.1.2 S06 WebSocket连接管理

S06 为量化模块的实时数据推送提供底层连接管理，覆盖行情数据（价格变动 / K 线更新）、AI 信号推送（新信号生成 / 信号状态变更）和跟单状态通知（开仓 / 平仓 / 止损触发）。

连接管理模块封装为一个独立的 Service Class（`WebSocketService`），采用单例模式确保全应用仅维护一条 WebSocket 连接。核心功能包括：**自动重连**——连接断开后在 3 秒内发起首次重连，采用指数退避策略（重连间隔按 3s / 6s / 12s / 24s 递增，最大 60 秒）；**频道订阅**——支持按业务模块订阅不同数据频道（`market:*` / `signal:*` / `copytrade:*`），页面卸载时自动取消订阅；**心跳保活**——每 30 秒发送一次 `ping` 帧，未在 10 秒内收到 `pong` 响应则判定连接异常并触发重连。

WebSocket 连接在 S01 认证通过后建立，连接建立时的握手请求携带 JWT Token 进行鉴权。量化相关的消息体采用统一格式：`{ channel, event, payload, timestamp }`，前端接收后通过 EventEmitter 分发至各页面的监听组件。复杂度评定为"中"，主要工作量在于重连策略的鲁棒性和大规模消息分发的性能优化。

## 12.2 全局UI组件

全局 UI 组件为全部页面提供一致的异常状态、加载状态和边界错误处理体验。4 个组件（S02–S05）均设计为 React 函数组件，通过 Props 接收差异化配置，主题样式继承 S08 主题配置系统的 CSS 变量。

### 12.2.1 S02 API限流提示

S02 处理后端返回 429（Too Many Requests）状态码时的用户提示。当检测到 429 响应时，全局 Axios 拦截器触发 S02 组件展示。

组件以顶部横幅（Top Banner）形式呈现，高度 48px，背景色 `#fef3c7`（浅黄色），文字色 `#92400e`（深棕色）。横幅左侧为警告图标，中部文案为"操作过于频繁，请等待 {countdown} 秒后重试"，其中 countdown 为后端响应头 `Retry-After` 返回的剩余等待时间（以 1 秒间隔递减的倒计时）。横幅右侧提供"立即升级 VIP"入口（引导用户升级以提高请求配额）。倒计时归零后横幅自动消失，同时触发被 429 拦截的请求重试。复杂度评定为"低"。

### 12.2.2 S03 全局Loading骨架屏

S03 为数据加载中的占位组件，用于替代空白页面等待，减少用户感知延迟。组件基于 CSS 动画实现，支持两种形态：**骨架条（Skeleton Bar）**——灰色渐变扫描动画的矩形条，用于列表行、卡片标题等线性内容的占位；**骨架图（Skeleton Card）**——预置了常见布局模板（Dashboard 卡片 / 表格 / 表单）的复合骨架结构，按真实布局的区块划分多个骨架条。

骨架颜色使用主题系统中的中性色：`#1e293b` 为骨架底色，`#334155` 为扫描高亮色，动画周期 1.5 秒，线性无限循环。组件通过 `variant` prop 选择模板类型（`bar` / `card` / `table` / `form`），通过 `rows` prop 控制骨架条数量。复杂度评定为"低"，为纯 CSS 动画组件。

### 12.2.3 S04 全局空状态页

S04 为数据为空或无搜索结果时的友好提示组件，替代空白区域展示插图和引导文案。

组件包含三个部分：空状态插图（使用 Lottie 动画或 SVG 静态插图，尺寸 200px × 200px）、主文案（如"暂无数据""暂无跟单记录"等，通过 `title` prop 传入）和引导操作（如"去浏览策略市场"按钮，通过 `action` prop 传入插图下方的按钮配置）。插图和文案按使用场景分为 5 套预设模板：`empty-data`（通用无数据）、`empty-search`（搜索无结果）、`empty-trades`（无交易记录）、`empty-subscription`（无订阅策略）、`empty-notifications`（无通知消息）。组件支持通过 `type` prop 快速切换模板。复杂度评定为"低"。

### 12.2.4 S05 全局错误边界

S05 基于 React Error Boundary 机制实现，捕获 React 组件树中任意位置的渲染错误，防止单个组件的崩溃导致整个页面白屏。

错误边界以 HOC 形式封装（`<ErrorBoundary />`），包裹在路由级组件外层。当捕获到错误时，展示降级 UI：包含错误图标、友好提示文案"页面加载出现异常，请稍后重试"、"刷新页面"按钮和"返回首页"链接。开发环境下额外展示错误堆栈信息（折叠面板形式，默认收起）。错误信息通过日志上报接口（`/log/frontend`）异步提交至后端监控系统，包含错误消息、组件栈、用户浏览器信息和发生时间戳。复杂度评定为"低"，属于标准 Error Boundary 模式的工程化封装。

## 12.3 全局配置与工具

全局配置与工具类组件提供跨页面、跨模块的共享能力，包括国际化文案、主题变量、图表封装和文件处理。4 个模块（S07–S10）以配置文件、CSS 变量集合和封装组件的形式存在，被全部 120 个页面引用。

### 12.3.1 S07 国际化i18n

S07 负责交易所的中英文双语文案管理。配置采用 JSON 结构组织，按页面模块划分命名空间（`common` / `trading` / `quant` / `copytrade` / `profile` 等），每个命名空间下以 Key-Value 形式存储文案键值对。

语言切换通过顶部导航栏的语言选择器触发，切换后将语言标识（`zh-CN` / `en-US`）写入 `localStorage` 并刷新页面。量化模块相关的文案键约 500 条，涵盖 AI 信号状态（"看涨" / "Bearish"）、策略类型（"趋势跟踪" / "Trend Following"）、大师名称（"Warren Buffett" / "巴菲特"）等专业术语。所有文案键统一采用 `snake_case` 命名规范，如 `quant.signal_direction_bullish`、 `quant.strategy_type_trend_following`。复杂度评定为"低"，主要工作量为量化模块 500 条文案的翻译和录入。

### 12.3.2 S08 主题配置系统

S08 定义交易所全端的暗色金融终端主题变量，以 CSS Custom Properties（CSS 变量）形式存储于全局样式文件的 `:root` 选择器下。

主题色板围绕底色 `#0a1628`（深蓝黑色）构建，主要变量包括：`--bg-primary: #0a1628`（主背景）、`--bg-secondary: #111d32`（卡片/面板背景）、`--bg-tertiary: #1a2744`（浮层/弹窗背景）、`--border-default: #1e3a5f`（边框色）、`--text-primary: #e2e8f0`（主文案）、`--text-secondary: #94a3b8`（辅助文案）、`--accent-primary: #3b82f6`（主操作色）、`--accent-success: #10b981`（盈利/上涨）、`--accent-warning: #f59e0b`（警告/观望）、`--accent-danger: #ef4444`（亏损/下跌）。此外定义了字体族（`--font-family: 'Inter', -apple-system, sans-serif`）、圆角规范（`--radius-sm: 4px` / `--radius-md: 8px` / `--radius-lg: 12px`）和层级系统（`--z-toast: 9000` / `--z-modal: 8000` / `--z-dropdown: 7000` / `--z-sticky: 100`）。主题切换通过给 `<html>` 标签添加/移除 `data-theme` 属性实现，目前仅提供暗色单主题，变量系统预留了亮色主题的扩展空间。复杂度评定为"低"。

### 12.3.3 S09 图表组件库封装

S09 基于 ECharts 进行金融暗色主题的封装，为全站的图表需求提供统一的渲染组件和预设配置。

封装层包含两个核心输出：**`<FinancialChart />` 组件**——接收 `type`（line / bar / pie / radar / area）、`data`（数据集）、`options`（覆盖配置）三个主要 prop，自动注入暗色主题配色和交互配置；**主题配置对象**——预定义的 ECharts `theme` 对象，包含背景色（`#0a1628`）、坐标轴样式（轴线 `#1e3a5f`、刻度 `#475569`）、系列配色数组（`['#3b82f6', '#10b981', '#f59e0b', '#ef4444', '#8b5cf6', '#06b6d4']`）和提示框样式（半透明黑底 + 白字）。

封装组件默认启用以下交互：鼠标悬停十字准线、数据点 Tooltip 提示、图例点击筛选、图表区域缩放（`dataZoom`，仅折线图和柱状图）。对于量化模块高频出现的雷达图（代币五维评分、大师观点一致性），封装层额外提供 `radarIndicator` 预设配置。复杂度评定为"中"，主要工作量在于 ECharts 主题配置的细调和响应式尺寸适配。

### 12.3.4 S10 文件上传/导出组件

S10 提供统一的文件处理能力，覆盖头像上传、证件上传和数据导出（Excel / PDF）三种场景。

- **上传组件（`<FileUpload />`）**：支持拖拽上传和点击选择两种交互，文件类型校验（图片：jpg/png，最大 5MB；文档：pdf，最大 20MB），上传进度条显示，上传成功后返回文件 URL 预览。头像上传模式下组件显示为圆形裁剪区域（1:1 比例），证件上传模式下显示为矩形区域（4:3 比例）。

两个组件共享同一套文件类型图标系统和上传状态管理逻辑。复杂度评定为"中"，PDF 导出的前端实现（含中文字体嵌入）为较复杂的技术点。以下为 10 个公共/系统组件的完整汇总。
