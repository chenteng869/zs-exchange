# 需求分析报告

## 一、文档1分析（UI设计规格说明书）

### 1.1 文档覆盖范围

文档1《中萨交易所 (ZS Exchange) · APP UI 设计规格说明书 v7》覆盖了以下12个UI设计维度：

1. **设计理念与视觉定位**：定义了"极光尊享 (Aurora Premium)"核心主题，包含核心关键词、视觉特征描述、三大主题系统（Aurora Premium / Dark Trading / Light Admin）
2. **色系系统**：包含主背景色阶（11个色值）、边框色（5级）、文字色（5级）、品牌色系（12色）、交易涨跌色（跨Web/H5/Admin三端）、语义状态色（4类）、用户状态色（8种）、交易状态色（7种）、订单状态色（6种）、风险等级色（5级）、极光五色光晕（6色+1暮蓝）、渐变色板（11种）、图表色板（Web/H5与Admin各6色）
3. **字体与排版**：字体族（3类：正文Sans/等宽Mono/数字Number）、Web/H5字号系统（15级）、Design Tokens标准字号（8级）、字重（4级）、行高（4类）、H5移动端特殊字号（9类）
4. **间距系统**：4px基准网格（14个token值）、H5移动端常用间距（10类场景含安全区适配）
5. **圆角规范**：8级圆角token（none/sm/md/lg/xl/2xl/3xl/full）、Design Tokens圆角备选（5级）
6. **阴影与光效**：基础阴影（4级sm/md/lg/xl）、极光发光效果（7种glow变体）、卡片阴影（3级含aurora-card三层复合阴影）、按钮发光（2种CTA/金色）
7. **玻璃拟态卡片 Glassmorphism 2.0**：核心CSS配方（含backdrop-filter blur(24px) saturate(180%)）、7种卡片变体（标准/青/紫/彩虹/皇家/深色/浅色）、Hover效果、H5移动端实际卡片样式（3类：资产总览卡/标准内容卡/牌照信任卡）
8. **按钮系统**：8种按钮变体（primary/secondary/cta/success/danger/warning/ghost/link）、3级尺寸（sm/md/lg）、H5移动端按钮样式（4类：金色主按钮/蓝色操作按钮/次操作按钮/高度自适应）、5种交互状态（Normal/Hover/Active/Disabled/Loading）、3种极光特色按钮
9. **布局与响应式**：7级断点系统（xs/默认/sm/md/lg/xl+自定义容器480px）、H5移动端布局约束（max-width:480px居中+fixed顶栏底栏+安全区适配）、底部导航Tab规格（5列Grid）、顶栏Header规格
10. **动效与过渡**：5级动画时长（150ms-22s）、5种缓动函数（default/ease-in/ease-out/bounce/aurora）、4种极光核心动画（aurora-flow/aurora-drift/aurora-rise/gold-shine）、5类过渡动画（卡片hover/按钮hover/页面进入/弹窗出现/Tab切换/数字滚动）、无障碍减弱动效支持
11. **H5移动端专属规范**：页面结构模板（顶栏+内容区+底栏）、8个模块配色速查、业务矩阵入口配色（10个入口）
12. **组件速查表**：CSS类名索引（30+类名）、Tailwind自定义颜色使用方式、CSS变量直接引用

### 1.5 多端适配要求

文档明确覆盖三个终端：

**H5移动端**：
- 容器最大宽度480px，PC上居中展示模拟手机效果
- 固定顶栏（z-index:100，height约60px，backdrop-filter模糊）+ 固定底部Tab导航（5列Grid）
- 安全区适配：env(safe-area-inset-top/bottom)
- 独立字号系统（资产数字24px/价格14px/涨跌幅11px等）
- 独立间距系统（页面左右边距12px/卡片内边距12-16px等）
- 独立按钮样式（高度自适应/圆角12px/字号13px）
- 业务矩阵入口从6个扩展至10个

**User PC前台**：
- 使用Aurora Premium主题（深色 #0F1B3D + 极光渐变）
- 断点从640px(sm)到1280px(xl)
- 与H5共享色系/字体/阴影等基础token
- 具体的PC布局规范相对简略，更多依赖Tailwind响应式类

**管理后台**：
- 使用Light Admin浅色主题（浅灰 #F5F7FA + 深色侧栏）
- 独立色板：语义状态色/用户状态色/交易状态色等均有Admin端适配
- 独立图表色板（Admin 6色 vs Web 6色）
- 独立字号（后台页面标题24px/20px，表格14px/13px等）
- 独立按钮变体（primary/success/danger在Admin端有不同色值）
- 卡片变体含.card-light（白色底+浅灰边框）

**隐性多端适配需求**：
- 三端需要统一的设计token管理系统来维护一致性
- 主题切换需要在运行时动态生效（CSS变量方案）
- H5的480px约束在平板设备上需要特殊处理
- 管理后台的深色侧栏+浅色内容区属于混合主题，需要额外适配

### 1.6 特殊设计特征

**玻璃拟态 2.0 (Glassmorphism 2.0)**：
- 核心技术参数：backdrop-filter: blur(24px) saturate(180%)
- 双层背景渐变：从 rgba(26,36,86,0.55) 到 rgba(21,34,74,0.70)
- 三层复合阴影系统：外阴影 + 内边框 + inset顶部高光线
- 金属感边框：rgba(148, 163, 184, 0.18)
- 7种变体通过顶部高光线颜色区分（金/青/紫/彩虹/皇家等）
- Hover效果：边框变金色 + 背景变亮 + 阴影加深 + 微上浮2px
- 轻量版本（glass-aurora-light）：blur(20px) + 更透明的白色渐变

**极光主题 (Aurora Premium)**：
- 背景从纯黑(#0B1124)升级为皇家深蓝(#0F1B3D) + 多层极光渐变网格
- "光流体"动画（aurora-flow）：12-14s周期的光斑呼吸移动效果
- 4种极光核心动画：flow（光斑呼吸）/ drift（慢漂移）/ rise（上升浮动）/ gold-shine（金色脉冲）
- 顶部2px极光光带装饰：渐变(透明→金→青→紫→透明)

**五大光晕色（极光五色呼吸体系）**：
- 皇家金 rgba(240, 185, 11, 0.35) — 温暖、尊贵、财富
- 钻石青 rgba(56, 189, 248, 0.32) — 冷静、科技、信任
- 紫罗兰 rgba(167, 139, 250, 0.30) — 神秘、Web3、AI
- 翠绿 rgba(52, 211, 153, 0.25) — 生长、上涨、健康
- 玫瑰粉 rgba(244, 114, 182, 0.22) — 活力、时尚、社交
- 暮蓝 rgba(99, 102, 241, 0.28) — 深沉、底部衬托

**其他特殊设计特征**：
- 卡片顶部「高光线」装饰（border-image渐变），营造金属质感
- 极光标题渐变文字（linear-gradient 135deg 金→青→紫）
- 金色按钮/CTA按钮带闪光扫过hover效果
- 数字滚动动画（资产数字从0滚动到目标值，500-800ms）
- H5底部导航激活指示条（2px高，渐变金→青，带发光阴影）

**隐性设计约束**：
- backdrop-filter性能消耗在低端设备上可能成为瓶颈
- 极光动画需要GPU加速，在电池模式下可能需要降级
- 多层渐变和阴影的组合会增加CSS bundle体积
- 玻璃拟态效果在纯色背景上不可见，必须配合渐变背景

### 2.1 页面规模与分布

**总页面数量：120页**（含主页面、子页面/Tab页、弹窗页、详情页、增强页）

| 端 | 页面数 | 占比 | 子分类 |
|----|--------|------|--------|
| 管理后台 Web | 55页 | 45.8% | 核心量化模块24页 + 四大生态联动20页 + 后台通用支撑11页 |

- 增强页（如H12-H14首页增强、U01-U09 PC增强）需要在现有代码上修改，存在回归测试成本

### 2.3 复杂度分布

| 复杂度等级 | 数量 | 占比 | 等效工作量估算 |
|-----------|------|------|--------------|
| 高（图表+多Tab+实时数据+AI交互） | 28页 | 23% | 每页约3-5个普通页面工作量，合计约84-140个普通页面当量 |
| 中（表单+列表+筛选+简单交互） | 52页 | 43% | 每页约1.5-2个普通页面工作量，合计约78-104个普通页面当量 |
| 低（纯展示/配置/静态内容） | 40页 | 34% | 每页约0.5-1个普通页面工作量，合计约20-40个普通页面当量 |

**按端统计高复杂度页面分布**：
- 管理后台高复杂度页面集中在：仪表盘（信号雷达图+市场热力图）、智能分析（4个子视图）、回测结果图表、四大联动模块（22页中大量评分/图谱/模拟器页面）
- H5高复杂度页面集中在：AI策略大脑入口、信号推荐列表、AI对话助手、企业诊断报告移动端
- User PC高复杂度页面集中在：PC端AI策略中心、PC端收益分析

**隐性复杂度因素**：
- 实时数据推送（WebSocket）增加了前后端联调复杂度
- AI交互（12位投资大师、6大分析师）涉及大模型集成，存在不确定性
- 蒙特卡洛模拟等计算密集型功能需要后端性能优化
- 跨模块数据关联（如分析结果→策略市场→跟单执行→收益分成）增加了业务流程复杂度

### 2.4 分阶段实施计划

### 2.5 核心功能模块深度分析

**量化交易模块（核心差异化）**：
- 12位投资大师智能体（Agent）：每位有独立头像/名称/风格标签/最近信号
- 6大分析师维度：信号矩阵表展示各Agent对每只股票的看涨/看跌/中性信号
- 风险管理层：风险管理器输出调整后信号和最大仓位建议，风控官可覆盖全局限制
- 组合决策：最终买入/卖出建议、数量、置信度和理由，可生成CEX订单草稿
- 回测中心：支持策略参数配置、时间范围选择、初始资金设定，输出净值曲线/最大回撤/夏普比率

**代币评分模块**：
- 100分制+5子维度：经济模型25/增长潜力25/风控安全25/AIOPC15/社区活跃10
- 集成Nansen链上数据：持仓分布图、巨鲸地址追踪、异常转账检测
- DCF估值Agent审查通缩机制、释放节奏、质押率
- 上币定价建议：基于估值模型给出建议发行价和初始流通量区间

**企业投研模块**：
- 面向AIOPC产业园入驻企业
- 12位大师观点+6大分析师数据+AIOPC行动建议
- 行业对标矩阵：同行业企业横向对比（营收/利润/增长率/市值）
- 供应链关系图谱：力导向图可视化
- 数字化成熟度评估：Level 1-5评分+升级路径甘特图

**策略分销模块**：
- 三级分销网络：L1-L4代理层级
- 多渠道：直销/电商（WOPC购物用户）/AIOPC
- 电商转化漏斗：浏览→试用→付费→跟单执行
- 三方分账：策略师/分销商/平台

**IPO评估模块**：
- 五维度评分：财务健康度30/公司治理20/成长性25/市场时机15/合规准备度10
- 三阶段DCF估值模型+参数敏感性分析
- 蒙特卡洛模拟上市后12个月股价走势
- 路演材料一键生成（PPT风格投资亮点报告）

**隐性功能需求**：
- AI推理过程的可解释性：需要展示Agent推理过程（可脱敏后出售）
- 数据安全与合规：API密钥脱敏显示、审计日志、RBAC权限控制
- 运营审核流程：策略上架需运营审核、代币风险预警需风控官审核
- 国际化：中英文双语支持
- 推送通知体系：新信号/跟单成交/止损止盈/新币上线/IPO进展等多场景

### 2.6 与UI设计规范的关联点

## 三、整合需求分析

### 3.1 两份文档的关联关系

**文档1（UI设计规格说明书）是文档2（开发页面清单）的视觉实现基础**。具体关联如下：

1. **主题系统映射**：文档2的120个页面需要依据文档1的三大主题系统进行视觉实现——H5移动端和User PC前台使用Aurora Premium深色主题，管理后台使用Light Admin浅色主题，交易终端使用Dark Trading主题

2. **组件复用关系**：文档2中反复出现的卡片布局（信号卡片、策略卡片、大师卡片、代币评分卡等）需要复用文档1定义的Glassmorphism 2.0卡片系统；按钮交互需要复用文档1的8种变体+3种极光特色按钮

3. **数据可视化关联**：文档2中大量高复杂度页面（市场热力图、信号雷达图、资产饼图、净值曲线、转化漏斗、供应链力导向图等）需要依赖文档1中图表色板和图表组件库封装（S09）

4. **动画体验关联**：文档2中的数字滚动（资产概览）、页面切换、弹窗出现等交互需要依赖文档1的动效系统；极光背景动画是Aurora Premium主题的核心识别特征

5. **缺口领域**：文档1缺少的表单组件、数据表格、Toast通知、Modal弹窗等规范恰好是文档2中大量页面急需的，存在供需不匹配

### 3.2 整合后的目标读者

**一级读者（直接影响项目交付）**：
- **前端开发团队**：两份文档共同构成前端开发的完整输入——文档1提供视觉规范，文档2提供页面清单和功能描述
- **UI设计师**：文档1是设计基准，需要基于文档2的页面清单扩展缺失组件（表单/表格/通知等）的设计
- **技术负责人**：需要统筹两份文档的匹配度，识别缺口并安排补齐

**二级读者（间接影响项目质量）**：
- **产品经理**：通过两份文档理解功能范围和视觉呈现能力的匹配情况
- **后端工程师**：文档2提供API设计参考（路由结构、数据展示需求），文档1提供数据格式化的视觉约束（数字字体/状态色/涨跌色等）
- **测试工程师**：文档2提供测试覆盖范围（120页的功能点），文档1提供视觉验收标准

**三级读者（项目管理和决策）**：
- **项目管理者**：文档2的分阶段计划提供排期依据，文档1的复杂度影响工时估算
- **业务方/客户**：通过文档理解产品能力和视觉定位

  4. 增加图标系统规范

### 3.4 内容优先级

**P0 — 阻塞性缺口（必须立即补齐）**：
1. 表单组件规范：文档2中的回测配置、API密钥管理、跟单配置、登录页等大量页面依赖表单组件
2. 数据表格规范：文档2中的持仓表格、信号矩阵、审计日志、用户管理等核心页面依赖表格
3. Modal弹窗规范：文档2中至少有6个弹窗页（W11/H30-H32等）

**P1 — 重要缺口（影响开发效率）**：
4. Toast/Notification组件规范：推送通知体系（H26-H29）和全局消息通知中心（W60）需要
5. 图标系统规范：业务矩阵10个入口、底部导航7Tab、操作按钮等均依赖图标
6. Loading/空状态/错误状态详细规范：文档2中S03-S05为公共组件，影响所有页面体验

**P2 — 优化项（提升一致性）**：
7. PC端布局网格系统：文档1对PC布局的规范相对简略
8. 打印/PDF导出样式：文档2中W48路演材料生成器和H37报告下载需要
9. 文档间交叉引用表：建立页面→设计元素的映射关系

**开发优先级建议**：
- **第一阶段（与Phase 1并行）**：补齐P0缺口，优先完成表单+表格+Modal规范，确保MVP 26页能按规范开发
- **第二阶段（与Phase 2并行）**：补齐P1缺口，完善通知+图标+状态规范，支撑42页核心能力
- **第三阶段（与Phase 3并行）**：补齐P2优化项，提升全量120页的体验一致性

**隐性风险提醒**：
- 文档1中的极光动画和玻璃拟态效果在低端设备上可能存在性能问题，建议在开发清单中增加性能测试项

- 三端主题切换（尤其是管理后台的混合主题）需要在开发早期建立技术基础，否则后期返工成本高
