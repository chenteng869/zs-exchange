## 1. 项目摘要

福建老酒 RWA-DID 全球生态计划，旨在将“一亿瓶福建老酒”转化为可验证、可追踪、可消费、可流通、可金融化但合规边界清晰的现实资产网络。项目以 DID 身份为入口，以福建老酒实物资产为锚，以 WOPC 人工智能集团的 AI 商业系统为流量和智能分发引擎，以前沿科技研究室为技术研发与标准制定中心，并探索与旷逸国际（HK1683）相关资本市场资源形成合规协同。

项目计划发行总量为 **36,900,000,000 枚 FJW（福建老酒生态通证）**，对应 **100,000,000 瓶福建老酒**：

```text
36,900,000,000 FJW / 100,000,000 bottles = 369 FJW per bottle
```

因此，项目的核心资产锚定公式为：

```text
369 FJW = 1 Bottle Redemption Unit = 1 bottle-level claim / usage / verification unit
```

FJW 不应被设计成简单的投机代币，而应被设计成“酒瓶资产凭证 + 消费权益 + DID 积分 + 生态结算凭证”的组合型工具。每一瓶老酒形成唯一 Bottle DID，每一批酒形成 Batch DID，每一个用户形成 User DID，最终构成“人、酒、仓、交易、消费、流通”的可信数据网络。

## 2. 五大核心主体

### 2.1 DID 身份系统

DID 是项目的信任根。每一个生态参与者都拥有独立 DID：

- 用户 DID：消费者、收藏者、渠道商、代理商、机构客户。
- 酒瓶 DID：每瓶酒唯一身份，绑定批次、年份、产区、仓储、质检、物流与兑换状态。
- 批次 DID：每批次酒的生产、采购、质检、入仓与审计记录。
- 仓储 DID：仓库、温湿度、保险、盘点、第三方审计。
- 企业 DID：酒厂、渠道商、WOPC、研究室、资本协同主体。

DID 的价值在于：它让福建老酒从传统消费品升级为可被链上验证的 RWA（Real World Asset）。用户买到的不是一个模糊故事，而是一个可追踪的资产单元。

### 2.2 福建老酒

福建老酒是本项目的现实资产底座。生态中每一瓶酒必须经过：

- 产地登记
- 批次登记
- 质检登记
- 仓储登记
- 防伪登记
- Bottle DID 铸造
- 可兑换或可消费状态标记

福建老酒承担三类价值：

| 价值类型 | 说明 |
| --- | --- |
| 消费价值 | 用户可兑换、饮用、礼赠、企业采购 |
| 收藏价值 | 年份、批次、限量编号、仓储记录带来溢价 |
| 数据价值 | 每瓶酒形成真实消费、流通、仓储和用户画像数据 |

### 2.4 WOPC 人工智能集团

WOPC 人工智能集团定位为生态的 AI 商业操作系统：

- AI 用户增长：通过电商、社群、直播、会员、内容推荐引流。
- AI 推荐引擎：基于用户画像推荐酒类产品、礼赠场景、收藏组合。
- AI 风控：识别刷单、羊毛党、异常套利、虚假渠道。
- AI 客服与销售：智能客服、智能导购、企业采购助手。
- AI 数据中台：分析消费数据、区域热度、库存周转、价格指数。

WOPC 不只是“卖酒渠道”，而是把福建老酒从单品销售升级为全球智能消费网络的核心引擎。

### 2.5 前沿科技研究室

- RWA-DID 标准

- AI 定价模型
- 防伪与溯源算法
- 智能合约审计与数据审计
- 跨链资产映射研究

研究室应输出年度技术报告、审计报告、储备证明报告、老酒指数报告，以形成项目的长期可信度。

## 3. 核心商业模式

本项目不采用单一“ICO 卖币”模式，而采用全球更先进的 **RWA-DID-ConsumptionFi-AI Commerce** 四层商业模型：

```text
实物老酒资产
  -> Bottle DID 确权
  -> FJW 生态通证映射
  -> AI 商业系统分发
  -> 全球用户消费 / 收藏 / 兑换 / 交易
  -> 数据沉淀
  -> 资本市场与品牌资产升级
```

### 3.1 一瓶一 DID

每瓶酒生成唯一 Bottle DID：

```text
did:fjw:bottle:{batchId}:{serialNo}
```

Bottle DID 记录：

- 产品名称
- 年份
- 容量
- 批次
- 产区
- 仓储位置
- 质检证书
- 防伪哈希
- 兑换状态
- 所属用户 DID
- 流通历史

### 3.2 369 FJW = 1 瓶权益单元

项目总发行 36.9B 枚，锚定 100M 瓶：

| 指标 | 数值 |
| --- | --- |
| 总发行量 | 36,900,000,000 FJW |
| 锚定酒瓶总数 | 100,000,000 瓶 |
| 单瓶映射 | 369 FJW |
| 单瓶基础价格叙事 | 369 美元 / 或等值法币与稳定币 |
| 最小功能单位 | 1 FJW |
| 兑换单位 | 369 FJW = 1 瓶兑换单元 |

### 3.3 三类权益

FJW 建议设计为三层权益，而不是直接承诺收益：

| 权益层 | 功能 | 合规边界 |
| --- | --- | --- |
| 消费权益 | 兑换老酒、抵扣、会员等级、品鉴活动 | 实用型权益 |
| 数据权益 | DID 积分、行为声誉、渠道等级、贡献值 | 不承诺现金收益 |
| 生态权益 | 治理建议、白名单、活动优先权、NFT 铸造资格 | 非股权、非分红 |

如需设计收益型产品，应独立设立合规 SPV 或证券型产品，只面向合格投资者，并与公开流通 FJW 严格隔离。


每次释放均应披露：

- 新增入仓瓶数
- 对应批次 DID
- 对应释放 FJW 数量
- 仓储证明
- 第三方审计或抽检报告

## 8. 全球最先进商业模式设计

### 8.1 Proof of Wine Reserve

项目应定期披露酒仓储备证明：

```text
仓库盘点数据 + 批次 DID + Bottle DID + 第三方审计 + 链上 Merkle Root
```

用户可以验证：

- 是否真的有酒
- 酒在哪个仓
- 属于哪个批次
- 是否已经被兑换
- 是否仍在保险覆盖范围内

### 8.2 AI Dynamic Commerce

WOPC AI 引擎负责将老酒从“库存”变成“动态商业资产”：

- AI 预测哪个地区更适合销售
- AI 判断用户适合消费型、收藏型还是礼赠型产品
- AI 生成企业礼品方案
- AI 推荐直播话术和销售脚本
- AI 监控价格异常和渠道套利

### 8.3 ConsumptionFi

项目不鼓励空转金融，而鼓励真实消费：

- 用户买酒获得 FJW 积分
- 用户兑换酒销毁或冻结对应兑换单元
- 用户复购提升 DID 等级
- 企业采购形成链上可验证订单
- 渠道贡献通过智能合约结算

### 8.4 RWA + NFT + DID 三层资产结构

| 层级 | 工具 | 作用 |
| --- | --- | --- |
| Fungible Layer | FJW | 生态积分、兑换单位、结算单位 |
| NFT Layer | Bottle NFT | 单瓶唯一凭证、收藏、转让、兑换 |
| DID Layer | User / Bottle / Batch DID | 身份、溯源、审计、信用 |

### 8.5 香港资本市场协同

如果 HK1683 相关合作可以依法落地，建议使用以下路径：

1. 不承诺代币对应上市公司股份。
2. 不承诺上市公司回购或兜底。
3. 先建立供应链订单、品牌授权、渠道合作或数据服务合作。
4. 再由审计机构评估真实收入、库存、数据资产和品牌价值。
5. 最后根据香港上市规则、关连交易规则、披露要求，探索资本协同。

## 9. 合规架构

建议采用“三隔离”：

```text
消费权益隔离
资产托管隔离
证券化产品隔离
```

### 9.1 消费权益层

面向普通用户，提供：

- 买酒
- 防伪
- 兑换
- 会员
- 积分
- 活动

该层避免收益承诺。

### 9.2 资产托管层

由独立主体或受托方管理：

- 酒仓
- 保险
- 审计
- 质检
- 损耗
- 储备证明

### 9.3 合格投资者层

如需资产收益分配、酒仓收益权、证券型代币、基金份额，应仅面向合格投资者，并独立文件披露：

- 风险
- 期限
- 估值方法
- 退出机制
- 法律意见
- 审计报告
