# 第三方代码检查系统与审计机构网址清单

> 项目：ZS Exchange / Stock Exchange DApp  
> 日期：2026-06-30  
> 用途：给技术团队接入自动化扫描、第三方安全审计、Web3 合约审计、漏洞赏金与董事局验收证据。

## 1. 结论

除了 Codex、Claude Code、Trae CN Solo 这类 AI 编程工具，还需要接入“机器扫描系统 + 第三方审计机构 + 可复核报告”。

建议执行顺序：

1. 先接入自动化扫描系统，1 天内拿到第一版机器报告。
2. 再做 API、前端、依赖、密钥、Docker、云配置检查。
3. 如果涉及真实资金、钱包、上链、合约，再找 Web3 安全审计机构。
4. 最后输出董事局可看的验收材料：扫描报告、修复清单、复测报告、风险声明、演示录屏。

## 8. 建议给技术团队的执行组合

### 第一层：AI 内部开发门禁

- Codex：配置、集成、仓库结构、脚本、交付文档。
- Claude Code：类型修复、安全加固、边界条件、漏洞修复。
- Trae CN Solo：前端页面、钱包页、后台页、用户流程、演示体验。

### 第二层：自动化机器扫描

- GitHub Advanced Security / CodeQL
- Semgrep
- Snyk
- Gitleaks
- Trivy
- OWASP ZAP

### 第三层：第三方专业报告

- Web2/API/云安全：奇安信、绿盟、安恒、启明星辰等。
- Web3/智能合约：OpenZeppelin、Trail of Bits、CertiK、SlowMist、PeckShield、Halborn 等。

### 第四层：董事局验收证据

- 扫描系统报告链接或 PDF。
- 第三方审计报告。
- 修复前后对比清单。
- 复测通过记录。
- 业务流程录屏。
- 管理后台、钱包流水、订单流水、数据库记录截图。

## 9. 推荐最低接入清单

如果只做 MVP，但又要有“不是 AI 自己说了算”的外部证据，最低建议接入：

| 优先级 | 工具 / 机构 | 目的 |
|---|---|---|
| P0 | GitHub Advanced Security 或 CodeQL | 代码级安全扫描 |
| P0 | Semgrep | TypeScript/Next.js/API 安全规则扫描 |
| P0 | Snyk 或 npm audit | 依赖漏洞扫描 |
| P0 | Gitleaks | 私钥、Token、JWT Secret 泄漏检查 |
| P0 | OWASP ZAP | Web/API 动态安全测试 |
| P1 | Trivy | Docker/部署镜像/IaC 检查 |
| P1 | 1 家 Web3 审计机构 | 钱包、签名、合约、链上资产安全 |
| P1 | 1 家国内安全公司 | 董事局或客户需要中文第三方安全报告时使用 |

## 10. 可直接放进董事局材料的一句话

本项目不只依赖 AI 编程工具自检。技术验收将采用“AI 开发门禁 + 自动化安全扫描 + 第三方审计机构 + 复测报告 + 现场演示证据”的组合方式，确保代码质量、资金安全、钱包安全、API 安全和生产部署风险可被独立复核。
